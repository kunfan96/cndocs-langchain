[智能体客户端协议 (ACP)](https://agentclientprotocol.com/get-started/introduction) 标准化了编码智能体与代码编辑器或 IDE 之间的通信。
使用 ACP 协议，你可以将你的自定义 Deep Agents 与任何兼容 ACP 的客户端一起使用，从而允许你的代码编辑器提供项目上下文并接收丰富的更新。

<Note>
    ACP 专为智能体-编辑器集成而设计。如果你希望你的智能体调用外部服务器托管的工具，请参阅 [模型上下文协议 (MCP)](/oss/javascript/langchain/mcp/)。
</Note>

## 快速开始

安装 ACP 集成包：

<CodeGroup>
```bash pip
pip install deepagents-acp
```

```bash uv
uv add deepagents-acp
```
</CodeGroup>

然后通过 ACP 公开一个 Deep Agent。

这将以 stdio 模式启动 ACP 服务器（它从 stdin 读取请求并将响应写入 stdout）。在实践中，你通常将其作为由 ACP 客户端（例如，你的编辑器）启动的命令运行，然后该客户端通过 stdio 与服务器通信。

<Card title="示例编码智能体" icon="brand-github" href="https://github.com/langchain-ai/deepagents/blob/main/libs/acp/examples/demo_agent.py">
  `deepagents-acp` 包包含一个示例编码智能体，带有文件系统和 shell，你可以开箱即用。
</Card>

## 客户端

Deep Agents 在任何你可以运行 ACP 智能体服务器的地方工作。一些值得注意的 ACP 客户端包括：

- [Zed](https://zed.dev/docs/ai/external-agents)
- [JetBrains IDEs](https://www.jetbrains.com/help/ai-assistant/acp.html)
- Visual Studio Code (通过 [vscode-acp](https://github.com/formulahendry/vscode-acp))
- Neovim (通过兼容 ACP 的插件)

### Zed

`deepagents` 仓库包含 [一个演示 ACP 入口点](https://github.com/langchain-ai/deepagents/blob/main/libs/acp/run_demo_agent.sh)，你可以用 [Zed](https://zed.dev/docs/ai/external-agents) 注册它：

1. 克隆 `deepagents` 仓库并安装依赖项：

```bash
git clone https://github.com/langchain-ai/deepagents.git
cd deepagents/libs/acp
uv sync --all-groups
chmod +x run_demo_agent.sh
```

2. 为演示智能体配置凭据：

```bash
cp .env.example .env
```

然后在 `.env` 中设置 `ANTHROPIC_API_KEY`。

3. 在 Zed 的 `settings.json` 中配置你的 ACP 智能体服务器命令：

```json
{
  "agent_servers": {
    "DeepAgents": {
      "type": "custom",
      "command": "/your/absolute/path/to/deepagents/libs/acp/run_demo_agent.sh"
    }
  }
}
```

4. 打开 Zed 的智能体面板并启动 DeepAgents 线程。

### Toad

如果你想作为本地开发工具运行 ACP 智能体服务器，你可以使用 [Toad](https://github.com/batrachian/toad) 来管理进程。

```bash
uv tool install -U batrachian-toad

toad acp "python path/to/your_server.py" .
# or
toad acp "uv run python path/to/your_server.py" .
```
