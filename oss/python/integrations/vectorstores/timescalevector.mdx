## 什么是 Timescale Vector？

`Timescale Vector` 可让您在 `PostgreSQL` 中高效存储和查询数百万条向量嵌入。

- 通过受 `DiskANN` 启发的索引算法，增强了 `pgvector` 在 1 亿以上向量上的搜索速度和准确性。
- 通过自动按时间分区和索引，支持快速的时序向量搜索。
- 提供熟悉的 SQL 接口，用于查询向量嵌入和关系数据。

`Timescale Vector` 是面向 AI 的云端 `PostgreSQL`，可从概念验证无缝扩展至生产环境：

- 支持在单一数据库中存储关系元数据、向量嵌入和时序数据，从而简化运维。
- 基于坚实的 PostgreSQL 基础，提供流备份与复制、高可用和行级安全等企业级特性。
- 提供企业级安全与合规，让您无后顾之忧。

## 2. 基于时间过滤的相似度搜索

Timescale Vector 的一个核心应用场景是高效的时序向量搜索。Timescale Vector 通过自动按时间对向量（及其关联元数据）进行分区来实现这一功能，使您能够按查询向量的相似度和时间两个维度高效查询向量。

基于时间的向量搜索功能适用于以下应用场景：

- 存储和检索 LLM 响应历史（例如聊天机器人）
- 查找与查询向量相似的最新嵌入（例如最新新闻）。
- 将相似度搜索限定在相关时间范围内（例如对知识库进行基于时间的提问）

为了说明如何使用 TimescaleVector 的时序向量搜索功能，我们将针对 TimescaleDB 的 git log 历史提问。我们将演示如何添加带时序 UUID 的文档，以及如何使用时间范围过滤器进行相似度搜索。

我们将定义一个辅助函数，根据时间戳为文档及其关联的向量嵌入创建 UUID。我们将使用此函数为每条 git log 条目创建 UUID。

重要提示：如果您希望向量与当前日期和时间关联以进行时序搜索，可以跳过此步骤。默认情况下，文档摄入时会自动生成 UUID。

```python
from timescale_vector import client


# Function to take in a date string in the past and return a uuid v1
def create_uuid(date_string: str):
    if date_string is None:
        return None
    time_format = "%a %b %d %H:%M:%S %Y %z"
    datetime_obj = datetime.strptime(date_string, time_format)
    uuid = client.uuid_from_time(datetime_obj)
    return str(uuid)
```

接下来，我们将定义一个元数据函数，从 JSON 记录中提取相关元数据。我们会将此函数传递给 JSONLoader。详情请参阅 [JSON 文档加载器文档](/oss/python/integrations/document_loaders/json)。

```python
# Helper function to split name and email given an author string consisting of Name Lastname <email>
def split_name(input_string: str) -> Tuple[str, str]:
    if input_string is None:
        return None, None
    start = input_string.find("<")
    end = input_string.find(">")
    name = input_string[:start].strip()
    email = input_string[start + 1 : end].strip()
    return name, email


# Helper function to transform a date string into a timestamp_tz string
def create_date(input_string: str) -> datetime:
    if input_string is None:
        return None
    # Define a dictionary to map month abbreviations to their numerical equivalents
    month_dict = {
        "Jan": "01",
        "Feb": "02",
        "Mar": "03",
        "Apr": "04",
        "May": "05",
        "Jun": "06",
        "Jul": "07",
        "Aug": "08",
        "Sep": "09",
        "Oct": "10",
        "Nov": "11",
        "Dec": "12",
    }

    # Split the input string into its components
    components = input_string.split()
    # Extract relevant information
    day = components[2]
    month = month_dict[components[1]]
    year = components[4]
    time = components[3]
    timezone_offset_minutes = int(components[5])  # Convert the offset to minutes
    timezone_hours = timezone_offset_minutes // 60  # Calculate the hours
    timezone_minutes = timezone_offset_minutes % 60  # Calculate the remaining minutes
    # Create a formatted string for the timestamptz in PostgreSQL format
    timestamp_tz_str = (
        f"{year}-{month}-{day} {time}+{timezone_hours:02}{timezone_minutes:02}"
    )
    return timestamp_tz_str


# Metadata extraction function to extract metadata from a JSON record
def extract_metadata(record: dict, metadata: dict) -> dict:
    record_name, record_email = split_name(record["author"])
    metadata["id"] = create_uuid(record["date"])
    metadata["date"] = create_date(record["date"])
    metadata["author_name"] = record_name
    metadata["author_email"] = record_email
    metadata["commit_hash"] = record["commit"]
    return metadata
```

接下来，我们将从预处理完毕的文档集合中创建一个 Timescale Vector 实例。

首先，我们定义一个集合名称，它将作为 PostgreSQL 数据库中表的名称。

我们还将定义一个时间增量，并将其传递给 `time_partition_interval` 参数，用作按时间分区数据的间隔。每个分区将包含指定时长内的数据。为简化示例，我们使用 7 天，但您可以根据实际情况选择合适的值——例如，如果您经常查询最近的向量，可以使用较小的时间增量（如 1 天）；如果查询跨越数十年的向量，则可以使用较大的时间增量（如 6 个月或 1 年）。

最后，我们创建 TimescaleVector 实例。我们将 `ids` 参数指定为上述预处理步骤中创建的元数据中的 `uuid` 字段。这样做是因为我们希望 UUID 中的时间部分反映过去的日期（即提交时间）。如果希望文档与当前日期和时间关联，可以省略 id 参数，UUID 将自动使用当前日期和时间生成。

```python
# Define collection name
COLLECTION_NAME = "timescale_commits"
embeddings = OpenAIEmbeddings()

# Create a Timescale Vector instance from the collection of documents
db = TimescaleVector.from_documents(
    embedding=embeddings,
    ids=[doc.metadata["id"] for doc in docs],
    documents=docs,
    collection_name=COLLECTION_NAME,
    service_url=SERVICE_URL,
    time_partition_interval=timedelta(days=7),
)
```

## 3. 使用 ANN 搜索索引加速查询

您可以通过在嵌入列上创建索引来加速相似度查询。建议在摄入大量数据后再创建索引。

Timescale Vector 支持以下索引：

- timescale_vector 索引 (tsv)：受 disk-ann 启发的图索引，用于快速相似度搜索（默认）。
- pgvector 的 HNSW 索引：用于快速相似度搜索的分层可导航小世界图索引。
- pgvector 的 IVFFLAT 索引：用于快速相似度搜索的倒排文件索引。

重要提示：在 PostgreSQL 中，每张表的特定列只能有一个索引。因此，如果要测试不同索引类型的性能，可以通过以下方式进行：(1) 创建多张具有不同索引的表，(2) 在同一张表中创建多个向量列并分别为每列创建不同索引，或 (3) 删除并重建同一列上的索引后比较结果。

```python
# Initialize an existing TimescaleVector store
COLLECTION_NAME = "timescale_commits"
embeddings = OpenAIEmbeddings()
db = TimescaleVector(
    collection_name=COLLECTION_NAME,
    service_url=SERVICE_URL,
    embedding_function=embeddings,
)
```

不带额外参数调用 `create_index()` 函数时，默认会使用默认参数创建 timescale_vector_index。

```python
# create an index
# by default this will create a Timescale Vector (DiskANN) index
db.create_index()
```

您也可以指定索引参数。请参阅 Timescale Vector 文档，了解不同参数及其对性能影响的详细说明。

注意：我们已设置了智能默认值，无需指定参数。但如果您想针对特定数据集进行性能优化实验，也可以自行指定参数。

```python
# drop the old index
db.drop_index()

# create an index
# Note: You don't need to specify m and ef_construction parameters as we set smart defaults.
db.create_index(index_type="tsv", max_alpha=1.0, num_neighbors=50)
```

Timescale Vector 还支持 HNSW ANN 索引算法和 ivfflat ANN 索引算法。只需在 `index_type` 参数中指定要创建的索引类型，并可选择性地指定索引参数。

```python
# drop the old index
db.drop_index()

# Create an HNSW index
# Note: You don't need to specify m and ef_construction parameters as we set smart defaults.
db.create_index(index_type="hnsw", m=16, ef_construction=64)
```

```python
# drop the old index
db.drop_index()

# Create an IVFFLAT index
# Note: You don't need to specify num_lists and num_records parameters as we set smart defaults.
db.create_index(index_type="ivfflat", num_lists=20, num_records=1000)
```

通常，我们建议使用默认的 timescale vector 索引或 HNSW 索引。

```python
# drop the old index
db.drop_index()
# Create a new timescale vector index
db.create_index()
```

## 4. 使用 Timescale Vector 的自查询检索器

为了演示 Timescale Vector 的自查询功能，我们将使用第 3 部分中相同的 gitlog 数据集。


## 5. 使用已有的 TimescaleVector 向量存储

在上述示例中，我们从文档集合创建了向量存储。但通常情况下，我们希望向已有向量存储中插入数据并进行查询。下面介绍如何初始化、向现有 TimescaleVector 向量存储的文档集合中添加文档，以及如何查询。

要使用已有的 Timescale Vector 存储，我们需要知道要查询的表名（`COLLECTION_NAME`）和云端 PostgreSQL 数据库的 URL（`SERVICE_URL`）。

```python
# Initialize the existing
COLLECTION_NAME = "timescale_commits"
embeddings = OpenAIEmbeddings()
vectorstore = TimescaleVector(
        collection_name=COLLECTION_NAME,
        service_url=SERVICE_URL,
        embedding_function=embeddings,
)
```

要向表中加载新数据，我们使用 `add_document()` 函数。该函数接收文档列表和元数据列表，元数据必须包含每个文档的唯一 ID。

如果希望文档与当前日期和时间关联，则无需创建 ID 列表，系统会为每个文档自动生成 UUID。

如果希望文档与过去的日期和时间关联，可以使用 `timescale-vector` Python 库中的 `uuid_from_time` 函数创建 ID 列表，如上方第 2 节所示。该函数接收一个 datetime 对象，并返回一个将日期和时间编码在 UUID 中的 UUID。

```python
# Add documents to a collection in TimescaleVector
ids = vectorstore.add_documents([Document(page_content="foo")])
ids
```

通过元数据删除在以下场景中特别有用：需要定期更新从特定来源、特定日期或其他元数据属性抓取的信息时。

```python
vectorstore.add_documents(
        [Document(page_content="Hello World", metadata={"source": "www.example.com/hello"})]

vectorstore.add_documents(
        [Document(page_content="Adios", metadata={"source": "www.example.com/adios"})]
)

vectorstore.delete_by_metadata({"source": "www.example.com/adios"})

vectorstore.add_documents(
```
