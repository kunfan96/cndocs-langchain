# BigtableVectorStore

本指南涵盖 `BigtableVectorStore` 集成，用于将 Google Cloud Bigtable 用作向量存储。

[Bigtable](https://cloud.google.com/bigtable) 是一个键值和宽列存储，非常适合快速访问结构化、半结构化或非结构化数据。

## 概述

`BigtableVectorStore` 使用 Google Cloud Bigtable 存储文档及其向量嵌入，用于相似性搜索和检索。它支持强大的元数据过滤以优化搜索结果。

## 初始化

初始化 `BigtableVectorStore` 涉及三个步骤：设置嵌入服务、确保 Bigtable 表已创建以及配置存储的参数。

### 1. 设置嵌入服务
首先，我们需要一个模型来为文档创建向量嵌入。在此示例中，我们将使用 Vertex AI 模型。

```python
from langchain_google_vertexai import VertexAIEmbeddings

embeddings = VertexAIEmbeddings(project=PROJECT_ID, model_name="gemini-embedding-001")
```

### 2. 初始化表
在创建 `BigtableVectorStore` 之前，必须存在具有正确列族的表。`init_vector_store_table` 辅助函数是创建和配置表的推荐方式。如果表已存在，它将不执行任何操作。

```python
from langchain_google_bigtable.vector_store import init_vector_store_table

DATA_COLUMN_FAMILY = "doc_data"

try:
    init_vector_store_table(
        project_id=PROJECT_ID,
        instance_id=INSTANCE_ID,
        table_id=TABLE_ID,
        content_column_family=DATA_COLUMN_FAMILY,
        embedding_column_family=DATA_COLUMN_FAMILY,
    )
    print(f"Table '{TABLE_ID}' is ready.")
except ValueError as e:
    print(e)
```

### 3. 配置向量存储
现在我们定义控制向量存储如何连接到 Bigtable 以及如何处理数据的参数。

#### BigtableEngine
`BigtableEngine` 对象管理客户端和异步操作。强烈建议初始化单个引擎并在多个存储之间重用它，以获得更好的性能和资源管理。

```python
from langchain_google_bigtable import BigtableEngine

engine = await BigtableEngine.async_initialize(project_id=PROJECT_ID)
```

#### 集合
`collection` 为单个 Bigtable 表中的文档提供逻辑命名空间。它用作行键的前缀，允许多个向量存储共存于同一表中而不会相互干扰。

```python
collection_name = "my_docs"
```

#### 元数据配置
创建 `BigtableVectorStore` 时，有两个可选参数用于处理元数据：

* `metadata_mappings`：这是一个 `VectorMetadataMapping` 对象列表。您**必须**为希望在搜索查询中用于过滤的任何元数据键定义映射。每个映射指定元数据字段的数据类型（`encoding`），这对于正确过滤至关重要。
* `metadata_as_json_column`：这是一个可选的 `ColumnConfig`，它告诉存储将整个元数据字典作为单个 JSON 字符串保存在特定列中。这对于高效检索文档的所有元数据（包括未在 `metadata_mappings` 中定义的字段）非常有用。**注意：** 仅存储在此 JSON 列中的字段无法用于过滤。

```python
from langchain_google_bigtable import ColumnConfig, VectorMetadataMapping, Encoding

# 定义要过滤的元数据字段的映射。
metadata_mappings = [
    VectorMetadataMapping(metadata_key="author", encoding=Encoding.UTF8),
    VectorMetadataMapping(metadata_key="year", encoding=Encoding.INT_BIG_ENDIAN),
    VectorMetadataMapping(metadata_key="category", encoding=Encoding.UTF8),
    VectorMetadataMapping(metadata_key="rating", encoding=Encoding.FLOAT),
]

# 定义用于将所有元数据存储为单个 JSON 字符串的可选列。
metadata_as_json_column = ColumnConfig(
    column_family=DATA_COLUMN_FAMILY, column_qualifier="metadata_json"
)
```

### 4. 创建 BigtableVectorStore 实例

```python
# 为您的存储配置列。
content_column = ColumnConfig(
    column_family=DATA_COLUMN_FAMILY, column_qualifier="content"
)
embedding_column = ColumnConfig(
    column_family=DATA_COLUMN_FAMILY, column_qualifier="embedding"
)
```

```python
from langchain_google_bigtable import BigtableVectorStore

vector_store = await BigtableVectorStore.create(
    project_id=PROJECT_ID,
    instance_id=INSTANCE_ID,
    table_id=TABLE_ID,
    engine=engine,
    embedding_service=embeddings,
    collection=collection_name,
    metadata_mappings=metadata_mappings,
    metadata_as_json_column=metadata_as_json_column,
    content_column=content_column,
    embedding_column=embedding_column,
)
```

## 管理向量存储

### 添加文档
您可以添加具有预定义 ID 的文档。如果添加的 `Document` 没有 `id` 属性，向量存储将自动为其生成一个 **`uuid4` 字符串**。

```python
from langchain_core.documents import Document

docs_to_add = [
    Document(
        page_content="A young farm boy, Luke Skywalker, is thrust into a galactic conflict.",
        id="doc_1",
        metadata={
            "author": "George Lucas",
            "year": 1977,
            "category": "sci-fi",
            "rating": 4.8,
        },
    ),
    Document(
        page_content="A hobbit named Frodo Baggins must destroy a powerful ring.",
        id="doc_2",
        metadata={
            "author": "J.R.R. Tolkien",
            "year": 1954,
            "category": "fantasy",
            "rating": 4.9,
        },
    ),
    # 没有预定义 ID 的文档，将生成一个 ID。
    Document(
        page_content="A group of children confront an evil entity emerging from the sewers.",
        metadata={"author": "Stephen King", "year": 1986, "category": "horror"},
    ),
    Document(
        page_content="In a distant future, the noble House Atreides rules the desert planet Arrakis.",
        id="doc_3",
        metadata={
            "author": "Frank Herbert",
            "year": 1965,
            "category": "sci-fi",
            "rating": 4.9,
        },
    ),
]

added_ids = await vector_store.aadd_documents(docs_to_add)
print(f"Added documents with IDs: {added_ids}")
```

### 更新文档
`BigtableVectorStore` 通过覆盖来处理更新。要更新文档，只需使用相同的 ID 但使用新内容或元数据再次添加它。

```python
doc_to_update = [
    Document(
        page_content="An old hobbit, Frodo Baggins, must take a powerful ring to be destroyed.",  # 更新内容
        id="doc_2",  # 相同 ID
        metadata={
            "author": "J.R.R. Tolkien",
            "year": 1954,
            "category": "epic-fantasy",
            "rating": 4.9,
        },  # 更新元数据
    )
]

await vector_store.aadd_documents(doc_to_update)
print("Document 'doc_2' has been updated.")
```

### 删除文档

```python
is_deleted = await vector_store.adelete(ids=["doc_2"])
```

## 查询向量存储

### 搜索

```python
results = await vector_store.asimilarity_search("a story about a powerful ring", k=1)
print(results[0].page_content)
```

### 带过滤器的搜索

在向量搜索运行之前应用过滤器。

#### kNN 搜索算法和过滤

默认情况下，`BigtableVectorStore` 使用 **k-最近邻 (kNN)** 搜索算法来查找数据库中与查询向量最相似的 `k` 个向量。向量存储提供过滤功能，以在执行 kNN 搜索之前缩小搜索空间，这可以使查询更快、更相关。

#### 使用 `QueryParameters` 配置查询

所有搜索设置都通过 `QueryParameters` 对象控制。该对象允许您指定过滤器以及其他重要的搜索方面：
* `algorithm`：要使用的搜索算法。默认为 `"kNN"`。
* `distance_strategy`：用于比较的度量，例如 `COSINE`（默认）或 `EUCLIDEAN`。
* `vector_data_type`：存储向量的数据类型，如 `FLOAT32` 或 `DOUBLE64`。这应与您的嵌入精度匹配。
* `filters`：定义要应用的过滤逻辑的字典。

#### 理解编码

要过滤元数据字段，您必须在 `metadata_mappings` 中使用正确的 `encoding` 定义它们，以便 Bigtable 可以正确解释数据。支持的编码包括：
* **字符串**：`UTF8`、`UTF16`、`ASCII` 用于基于文本的元数据。
* **数字**：`INT_BIG_ENDIAN` 或 `INT_LITTLE_ENDIAN` 用于整数，`FLOAT` 或 `DOUBLE` 用于小数。
* **布尔值**：`BOOL` 用于真/假值。

#### 过滤支持表

| 过滤类别 | 键 / 运算符 | 含义 |
|---|---|---|
| **行键** | `RowKeyFilter` | 将搜索范围缩小到具有特定前缀的文档 ID。 |
| **元数据键** | `ColumnQualifiers` | 检查是否存在一个或多个精确的元数据键。 |
| | `ColumnQualifierPrefix` | 检查元数据键是否以给定前缀开头。 |
| | `ColumnQualifierRegex` | 检查元数据键是否匹配正则表达式。 |
| **元数据值** | `ColumnValueFilter` | 所有基于值的条件的容器。 |
| | `==` | 相等 |
| | `!=` | 不相等 |
| | `>` | 大于 |
| | `<` | 小于 |
| | `>=` | 大于或等于 |
| | `<=` | 小于或等于 |
| | `in` | 值在列表中。 |
| | `nin` | 值不在列表中。 |
| | `contains` | 检查是否存在子字符串。 |
| | `like` | 对字符串执行正则表达式匹配。 |
| **逻辑**| `ColumnValueChainFilter` | 用于组合值条件的逻辑 AND。 |
| | `ColumnValueUnionFilter` | 用于组合值条件的逻辑 OR。 |

#### 复杂过滤器示例

此示例使用多个嵌套逻辑过滤器。它搜索满足以下条件的文档：（`category` 为 'sci-fi' 且 `year` 在 1970-2000 之间）或（`author` 为 'J.R.R. Tolkien'）或（`rating` > 4.5）。

```python
from langchain_google_bigtable.vector_store import QueryParameters

complex_filter = {
    "ColumnValueFilter": {
        "ColumnValueUnionFilter": {  # OR
            "ColumnValueChainFilter": {  # 第一个 AND 条件
                "category": {"==": "sci-fi"},
                "year": {">": 1970, "<": 2000},
            },
            "author": {"==": "J.R.R. Tolkien"},
        }
    }
}

query_params_complex = QueryParameters(filters=complex_filter)

complex_results = await vector_store.asimilarity_search(
    "a story about a hero's journey", k=5, query_parameters=query_params_complex
)

print(f"Found {len(complex_results)} documents matching the complex filter:")
for doc in complex_results:
    print(f"- ID: {doc.id}, Metadata: {doc.metadata}")
```

### 带分数的搜索
您还可以检索距离分数以及文档。

```python
results_with_scores = await vector_store.asimilarity_search_with_score(
    query="an evil entity", k=1
)
for doc, score in results_with_scores:
    print(f"* [SCORE={score:.4f}] {doc.page_content} [{doc.metadata}]")
```

### 用作检索器
向量存储可以轻松用作 RAG 应用程序中的检索器。您可以指定搜索类型（例如 `similarity` 或 `mmr`）并传递搜索时参数，如 `k` 和 `query_parameters`。

```python
# 定义一个与检索器一起使用的过滤器
retriever_filter = {"ColumnValueFilter": {"category": {"==": "horror"}}}
retriever_query_params = QueryParameters(filters=retriever_filter)

retriever = vector_store.as_retriever(
    search_type="mmr",  # 指定 MMR 用于检索
    search_kwargs={
        "k": 1,
        "lambda_mult": 0.8,
        "query_parameters": retriever_query_params,  # 传递过滤器参数
    },
)
retrieved_docs = await retriever.ainvoke("a story about a hobbit")
print(retrieved_docs[0].page_content)
```
