## 概述

AgentSystems Notary 为 AI 代理交互创建防篡改审计追踪。

## 原因

当客户、审计员、监管机构、保险公司等质疑 AI 行为时，你需要证明实际发生了什么。传统日志不起作用：你控制它们，所以第三方必须信任你没有修改它们。

防篡改日志消除了这种信任要求。

## 工作原理

原始 LLM 交互保留在你的存储中。正常运行期间没有第三方能看到它们。但每次交互的加密哈希会同时写入独立的防篡改存储（Arweave 或 AgentSystems 托管服务）。

如果发生审计或争议，你提供原始日志。审计员重新哈希它们并与存储的哈希进行比较。匹配表示日志未更改。不匹配表示篡改或损坏。

你控制你的数据，但无法在不被检测的情况下更改它。

**记录内容：**
- 到你的存储：完整原始 LLM 负载（prompts、responses、metadata、timestamps）
- 到哈希存储：SHA-256 哈希 + 元数据（例如 namespace、session ID、timestamps）

## 哈希存储选项

哈希（而非原始数据）可以写入以下任一存储选项：

| 存储 | 最佳适用 | 特性 |
|---------|----------|----------|
| [去中心化 (Arweave)](https://docs.agentsystems.ai/notary/configuration/hash-storage?utm_source=langchain-docs&utm_medium=docs) | 无供应商锁定 | 公共仅追加账本，开源验证，无需账户 |
| [`托管`](https://agentsystems.ai/notary?utm_source=langchain-docs&utm_medium=docs) | 托管合规 | 一次写入存储，验证 UI，审计签名证明 |

<Info>
[托管计划](https://agentsystems.ai/notary?utm_source=langchain-docs&utm_medium=docs) 提供符合 WORM 标准的哈希存储，托管签名，以及审计签名证明。
</Info>

## 先决条件

```shell
pip install agentsystems-notary langchain langchain-anthropic python-dotenv
```

## 示例（去中心化）

<Steps>
  <Step title="生成签名密钥">
    ```shell
    openssl genrsa -out arweave-key.pem 4096
    ```
    <Warning>
    保留此密钥。验证期间需要它来证明链上哈希的所有权。
    </Warning>

    对于生产环境，请使用 [云密钥管理服务](https://docs.agentsystems.ai/notary/configuration/signing?utm_source=langchain-docs&utm_medium=docs)。
  </Step>
  <Step title="创建 .env 文件">
    在你的项目根目录创建 `.env` 文件：
    ```
    # 用于原始负载存储的 AWS S3
    ORG_AWS_S3_BUCKET_NAME=your-bucket
    ORG_AWS_S3_ACCESS_KEY_ID=AKIA...
    ORG_AWS_S3_SECRET_ACCESS_KEY=...
    ORG_AWS_S3_REGION=us-east-1

    # 签名密钥路径
    ARWEAVE_PRIVATE_KEY_PATH=./arweave-key.pem

    # Anthropic
    ANTHROPIC_API_KEY=sk-ant-...
    ```
  </Step>
  <Step title="运行示例">
    ```python
    import os

    from agentsystems_notary import (
        LangChainNotary,
        ArweaveHashStorage,
        AwsS3StorageConfig,
        LocalKeySignerConfig,
        RawPayloadStorage,
    )
    from langchain_anthropic import ChatAnthropic
    from dotenv import load_dotenv

    load_dotenv()

    # 用于原始 LLM 负载的你的 S3 存储桶
    s3_config = AwsS3StorageConfig(
        bucket_name=os.environ["ORG_AWS_S3_BUCKET_NAME"],
        aws_access_key_id=os.environ["ORG_AWS_S3_ACCESS_KEY_ID"],
        aws_secret_access_key=os.environ["ORG_AWS_S3_SECRET_ACCESS_KEY"],
        aws_region=os.environ["ORG_AWS_S3_REGION"],
    )
    raw_payload_storage = RawPayloadStorage(storage=s3_config)

    # 用于签名的本地 RSA 密钥
    signer = LocalKeySignerConfig(
        private_key_path=os.environ["ARWEAVE_PRIVATE_KEY_PATH"],
    )

    # 用于去中心化哈希存储的 Arweave
    # 命名空间是公开的——写入账本并用于分段存储的数据
    # 命名空间应该是每个客户、代理或环境的一个匿名 ID
    # 保留你的命名空间映射记录
    arweave_storage = ArweaveHashStorage(
        namespace="tenant_a1b2c3d4",  # 参见上面的命名空间注释
        signer=signer,
    )

    # 创建 Notary 回调
    notary = LangChainNotary(
        raw_payload_storage=raw_payload_storage,
        hash_storage=[arweave_storage],
        debug=True,
    )

    # 附加到模型
    model = ChatAnthropic(
        model="claude-sonnet-4-6",
        api_key=os.environ["ANTHROPIC_API_KEY"],
        callbacks=[notary],
    )

    response = model.invoke("What is the capital of France?")
    print(response.content)
    ```
  </Step>
</Steps>

## 验证

**去中心化 (Arweave)**：从你的存储桶下载原始负载，打包它们，并使用开源 CLI 验证：

```shell
aws s3 sync s3://your-bucket/arweave/tenant_a1b2c3d4/ ./logs
zip -r logs.zip logs
npm install -g agentsystems-verify
agentsystems-verify --logs logs.zip
```

CLI 重新哈希每个负载并与存储在 Arweave 上的哈希进行比较。详见 [完整验证指南](https://docs.agentsystems.ai/notary/verification/arweave?utm_source=langchain-docs&utm_medium=docs)。

或者，[Verify UI](https://verify.agentsystems.ai?utm_source=langchain-docs&utm_medium=docs) 支持去中心化和托管验证。
