# Cala

>[Cala](https://cala.ai) 是一个用于 AI 代理的已验证知识图谱。
>它返回结构化的、有来源引用的、可追溯的事实——而不是原始的网页抓取数据。
>每个答案都包含引用、置信度分数和实体级别的来源信息，
>使其适用于需要可审计、抗幻觉检索的生产级代理。

## 安装

```bash
pip install langchain-cala
```

在 [console.cala.ai](https://console.cala.ai) 获取您的 API 密钥。

## 配置

```python
import os
os.environ["CALA_API_KEY"] = "your-api-key"
```

或者直接传递密钥：

```python
from langchain_cala import CalaRetriever

retriever = CalaRetriever(api_key="your-api-key")
```

## 检索器

`CalaRetriever` 实现了 LangChain 的 `BaseRetriever` 接口，并支持三种模式：

| 模式 | 端点 | 使用场景 |
|------|----------|----------|
| `"search"` *(默认)* | `POST /v1/knowledge/search` | 开放式自然语言问题 — 返回带引用的综合答案 |
| `"query"` | `POST /v1/knowledge/query` | 结构化点表示法查找 — 例如 `"OpenAI.founded.year"` |
| `"entities"` | `GET /v1/entities` | 在进行更深层次查询前按名称发现实体 |

### 基本搜索

```python
from langchain_cala import CalaRetriever

retriever = CalaRetriever(api_key="your-api-key")
docs = retriever.invoke("Which EU AI startups raised funding in 2024?")

for doc in docs:
    print(doc.page_content)
    print(doc.metadata)  # 包含 'verified', 'source', 'explainability'
```

### 结构化查询

```python
retriever = CalaRetriever(api_key="your-api-key", mode="query")
docs = retriever.invoke("Mistral.founded.year")
```

### 实体发现

```python
retriever = CalaRetriever(api_key="your-api-key", mode="entities")
docs = retriever.invoke("Factorial HR")
```

### 在 RAG 链中使用

```python
from langchain_cala import CalaRetriever
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

retriever = CalaRetriever(api_key="your-api-key")

prompt = ChatPromptTemplate.from_template(
    "Answer the question based only on the following context:\n\n{context}\n\nQuestion: {question}"
)

chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | ChatOpenAI()
    | StrOutputParser()
)

chain.invoke("Who are the key players in European AI infrastructure?")
```

## 关键参数

| 参数 | 类型 | 默认值 | 描述 |
|-----------|------|---------|-------------|
| `api_key` | `SecretStr` | `$CALA_API_KEY` | 您的 Cala API 密钥 |
| `mode` | `str` | `"search"` | `"search"`, `"query"`, `"entities"` 之一 |
| `k` | `int` | `5` | 返回的最大文档数量 |
| `timeout` | `int` | `30` | HTTP 请求超时时间（秒） |

## 为何选择 Cala

与网络搜索检索器不同，Cala 返回**已验证的实体数据**，并具有完整的来源可追溯性。响应中的每个文档都包含一个 `verified` 元数据字段和引用链——这使其符合欧盟 AI 法案第 13 条关于高风险 AI 部署的可解释性要求。
