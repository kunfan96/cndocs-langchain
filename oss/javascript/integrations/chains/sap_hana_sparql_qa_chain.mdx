## 设置与安装

要使用此功能，请安装 `@sap/hana-langchain` 包及其对等依赖项：

```bash
npm install @sap/hana-langchain @langchain/core@latest @langchain/classic@latest langchain@latest
```

创建到您的 SAP HANA Cloud 实例的连接。

```typescript
import * as dotenv from 'dotenv';
// 如果需要，加载环境变量
dotenv.config();

import hanaClient from "@sap/hana-client";

const connectionParams = {
  host: process.env.HANA_DB_ADDRESS,
  port: process.env.HANA_DB_PORT,
  user: process.env.HANA_DB_USER,
  password: process.env.HANA_DB_PASSWORD,
};
const client = hanaClient.createConnection(connectionParams);

// 连接到 hanaDB
await new Promise<void>((resolve, reject) => {
  client.connect((err: Error) => {
    // 此处使用箭头函数
    if (err) {
      reject(err);
    } else {
      console.log("成功连接到 SAP HANA。");
      resolve();
    }
  });
});
```

`HanaSparqlQAChain` 将以下内容绑定在一起：

1. **模式感知的 SPARQL 生成**
2. **针对 SAP HANA 的查询执行**
3. **自然语言答案格式化**

## 初始化

* 一个用于生成和解释查询的 **LLM**
* 一个 **`HanaRdfGraph`**（包含连接、`graph_uri` 和本体）

请按照此处 [HanaRdfGraph](/oss/javascript/integrations/graphs/sap_hana_rdf_graph) 的步骤了解如何创建 `HanaRdfGraph` 实例。

导入 `HanaSparqlQAChain`

```typescript
import { HanaSparqlQAChain } from "@sap/hana-langchain";
const qaChain = HanaSparqlQAChain.fromLLM({ llm, graph, allowDangerousRequests: true, verbose: true });
```

## 流程概述

1. SPARQL 生成
   - 使用 SPARQL 生成提示
   - 输入：`schema`（来自 `graph.getSchema()` 的 Turtle 格式）、`prompt`（用户问题）
2. 查询后处理
   - 从模型输出中提取 SPARQL
   - 如果缺少，则注入 `FROM <graph_uri>`
   - 确保常见前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）
3. 执行
   - `graph.query(generatedSparql)`
4. 答案制定
   - 使用 QA 提示，输入：`context`（结果）、`prompt`（原始问题）

## 提示模板

### "SPARQL 生成" 提示

`sparqlGenerationPrompt` 用于引导 LLM 根据用户问题和提供的模式生成 SPARQL 查询。

### 问答提示

`qaPrompt` 指示 LLM 仅根据数据库结果创建自然语言答案。
默认提示可在此处找到：[`prompts.ts`](https://github.com/SAP/langchainjs-integration-for-sap-hana-cloud/blob/main/src/chains/graph_qa/prompts.ts)

## 自定义提示

您可以在初始化时覆盖默认值：

```typescript
const qaChain = HanaSparqlQAChain.fromLLM({
  llm,
  graph,
  allowDangerousRequests: true,
  verbose: true,
  sparqlGenerationPrompt: YOUR_SPARQL_PROMPT,
  qaPrompt: YOUR_QA_PROMPT,
});
```

> - `sparql_generation_prompt` 必须具有输入变量：`["schema", "prompt"]`
> - `qa_prompt` 必须具有输入变量：`["context", "prompt"]`

## 示例：在“电影”知识图谱上进行问答

**先决条件**：
您必须拥有一个启用了**三元组存储**功能的 SAP HANA Cloud 实例。
详细说明请参考：[启用三元组存储](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/enable-triple-store/)<br />
加载 `kgdocu_movies` 示例数据。请参阅[知识图谱示例](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/knowledge-graph-example)。

下面我们将：

1. 实例化指向我们“电影”数据图的 `HanaRdfGraph`
2. 将其包装在由 LLM 驱动的 `HanaSparqlQAChain` 中
3. 提出自然语言问题并打印出链的响应

这演示了 LLM 如何在底层生成 SPARQL，针对 SAP HANA 执行它，并返回人类可读的答案。

首先，创建到您的 SAP HANA Cloud 实例的连接。

```typescript

  password: process.env.HANA_DB_PASSWORD,

    }

```

然后，设置知识图谱实例

```typescript
import { HanaRdfGraph, HanaRdfGraphOptions } from "@sap/hana-langchain";

const graphOptions: HanaRdfGraphOptions = {
  connection: client,
  graphUri: "kgdocu_movies",
  autoExtractOntology: true,
};

// 从源 URI 创建 Graph 实例
const graph = new HanaRdfGraph(graphOptions);

// 创建实例后需要初始化一次。
await graph.initialize(graphOptions);
```

```typescript
// 序列化图模式（可选）

  if (error) {
```

## 底层发生了什么？

1. **SPARQL 生成**
   链使用您的 Turtle 格式本体（`graph.getSchema()`）和用户问题调用 LLM，使用 `SPARQL_GENERATION_SELECT_PROMPT`。然后 LLM 发出一个针对您的模式定制的有效 `SELECT` 查询。
2. **预处理与执行**
   * **提取与清理**：从 LLM 的响应中提取原始 SPARQL 文本。
   * **注入图上下文**：如果缺少，则添加 `FROM <graph_uri>`，并确保声明了常见前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）。
   * **在 HANA 上运行**：通过 `HanaRdfGraph.query()` 在您的命名图上执行最终确定的查询。
3. **答案制定**
   返回的 CSV（或 Turtle）结果再次输入 LLM——这次使用 `SPARQL_QA_PROMPT`。LMM 生成一个简洁、人类可读的答案，严格基于检索到的数据，没有幻觉。
