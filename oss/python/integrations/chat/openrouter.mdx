## 设置

要通过 OpenRouter 访问模型，您需要创建一个 [OpenRouter 帐户](https://openrouter.ai/)，获取 API 密钥，并安装 `langchain-openrouter` 集成包。

### 安装

LangChain OpenRouter 集成位于 `langchain-openrouter` 包中：

<CodeGroup>
    ```bash pip
    pip install -U langchain-openrouter
    ```
    ```bash uv
    uv add langchain-openrouter
    ```
</CodeGroup>

### 凭据

前往 [OpenRouter 密钥页面](https://openrouter.ai/settings/keys) 注册并生成 API 密钥。完成后，设置 `OPENROUTER_API_KEY` 环境变量：

```python
import getpass
import os

if not os.getenv("OPENROUTER_API_KEY"):
    os.environ["OPENROUTER_API_KEY"] = getpass.getpass("输入您的 OpenRouter API 密钥： ")
```

要启用模型调用的自动跟踪，请设置您的 [LangSmith](/langsmith/home) API 密钥：

```python
os.environ["LANGSMITH_API_KEY"] = getpass.getpass("输入您的 LangSmith API 密钥： ")
os.environ["LANGSMITH_TRACING"] = "true"
```

## 实例化

现在我们可以实例化我们的模型对象并生成聊天补全：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    temperature=0,
    max_tokens=1024,
    max_retries=2,
    # 其他参数...
)
```

## 工具调用

OpenRouter 使用与 OpenAI 兼容的工具调用格式。您可以描述工具及其参数，并让模型返回一个 JSON 对象，其中包含要调用的工具和该工具的输入。

### 绑定工具

使用 `ChatOpenRouter.bind_tools`，您可以将 Pydantic 类、字典模式、LangChain 工具或函数作为工具传递给模型。在底层，这些会被转换为 OpenAI 工具模式，并在每次模型调用时传入。

```python
from pydantic import BaseModel, Field


class GetWeather(BaseModel):
    """获取给定位置的当前天气"""

    location: str = Field(description="城市和州，例如加利福尼亚州旧金山")


model_with_tools = model.bind_tools([GetWeather])
```

```python
ai_msg = model_with_tools.invoke(
    "旧金山的天气怎么样",
)
ai_msg
```

```text
AIMessage(content='', response_metadata={'finish_reason': 'tool_calls'}, tool_calls=[{'name': 'GetWeather', 'args': {'location': 'San Francisco, CA'}, 'id': 'call_abc123', 'type': 'tool_call'}], usage_metadata={'input_tokens': 68, 'output_tokens': 17, 'total_tokens': 85})
```

### 工具调用

AIMessage 具有 `tool_calls` 属性。这包含以标准化格式的工具调用，该格式与模型提供商无关。

```python
ai_msg.tool_calls
```

```text
[{'name': 'GetWeather',
  'args': {'location': 'San Francisco, CA'},
  'id': 'call_abc123',
  'type': 'tool_call'}]
```

### 严格模式

传递 `strict=True` 以保证模型输出与工具定义中提供的 JSON 模式完全匹配：

```python
model_with_tools = model.bind_tools([GetWeather], strict=True)
```

有关绑定工具和工具调用输出的更多信息，请前往 [工具调用](/oss/python/langchain/tools) 文档。

## 结构化输出

`ChatOpenRouter` 支持通过 [`with_structured_output`](/oss/python/langchain/models#structured-output) 方法进行结构化输出。提供两种方法：`function_calling`（默认）和 `json_schema`。

<Accordion title="单个模型调用">

使用 `with_structured_output` 生成结构化的模型响应。指定 `method="json_schema"` 以使用基于 JSON 模式的结构化输出；否则该方法默认为函数调用。

```python
from langchain_openrouter import ChatOpenRouter
from pydantic import BaseModel, Field

model = ChatOpenRouter(model="openai/gpt-4.1")

class Movie(BaseModel):
    """一部包含详细信息的电影。"""
    title: str = Field(description="电影标题")
    year: int = Field(description="电影发行年份")
    director: str = Field(description="电影导演")
    rating: float = Field(description="电影评分（满分10分）")

structured_model = model.with_structured_output(Movie, method="json_schema")  # [!code highlight]
response = structured_model.invoke("提供电影《盗梦空间》的详细信息")
response
```

```python
Movie(title='Inception', year=2010, director='Christopher Nolan', rating=8.8)
```

</Accordion>

<Accordion title="代理响应格式">

指定 `response_format` 与 [`ProviderStrategy`](/oss/python/langchain/agents#providerstrategy) 以在生成代理最终响应时启用结构化输出。

```python
from langchain.agents import create_agent
from langchain.agents.structured_output import ProviderStrategy
from pydantic import BaseModel

class Weather(BaseModel):
    temperature: float
    condition: str

def weather_tool(location: str) -> str:
    """获取某个位置的天气。"""
    return "晴朗，75 华氏度。"

agent = create_agent(
    model="openrouter:openai/gpt-4.1",
    tools=[weather_tool],
    response_format=ProviderStrategy(Weather),  # [!code highlight]
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "旧金山的天气怎么样？"}]
})

result["structured_response"]
```

```python
Weather(temperature=75.0, condition='Sunny')
```

</Accordion>

您可以传递 `strict=True` 与 `function_calling` 和 `json_schema` 方法以强制执行精确的模式遵循。`strict` 参数不支持 `json_mode`。

```python
structured_model = model.with_structured_output(Movie, method="json_schema", strict=True)
```

## 推理输出

对于支持推理的模型（例如 `anthropic/claude-sonnet-4.5`、`deepseek/deepseek-r1`），您可以通过 `reasoning` 参数启用推理令牌。有关详细信息，请参阅 [OpenRouter 推理文档](https://openrouter.ai/docs/guides/best-practices/reasoning-tokens)：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    max_tokens=16384,
    reasoning={"effort": "high", "summary": "auto"},
)

ai_msg = model.invoke("529 的平方根是多少？")

# 通过 content_blocks 访问推理内容
for block in ai_msg.content_blocks:
    if block["type"] == "reasoning":
        print(block["reasoning"])
```

有关内容块的更多信息，请参阅 [标准内容块](/oss/python/langchain/messages#standard-content-blocks) 指南。

`reasoning` 字典支持两个键：

- `effort`：控制推理令牌预算。值：`"xhigh"`、`"high"`、`"medium"`、`"low"`、`"minimal"`、`"none"`。
- `summary`：控制响应中返回的推理摘要的详细程度。值：`"auto"`、`"concise"`、`"detailed"`。

推理令牌使用情况包含在 `usage_metadata` 中：

```python
print(ai_msg.usage_metadata)
# {'input_tokens': ..., 'output_tokens': ..., 'total_tokens': ...,
#  'output_token_details': {'reasoning': ...}}
```

<Note>
    努力到预算的映射取决于模型。例如，Google Gemini 模型将努力映射到内部的 `thinkingLevel`，而不是精确的令牌预算。有关详细信息，请参阅 [OpenRouter 推理文档](https://openrouter.ai/docs/guides/best-practices/reasoning-tokens)。
</Note>

## 令牌使用情况元数据

调用后，令牌使用情况信息可在响应的 `usage_metadata` 属性上获取：

```python
ai_msg = model.invoke("给我讲个笑话。")
ai_msg.usage_metadata
```

```text
{'input_tokens': 12,
 'output_tokens': 25,
 'total_tokens': 37}
```

当底层提供商在其响应中包含详细的令牌细分时，它们会自动显示。当提供商未报告这些字段或其值为零时，这些字段将被省略。

### 推理令牌

`output_token_details.reasoning` 报告模型用于内部思维链推理的令牌数量。当使用推理模型（例如 `deepseek/deepseek-r1`、`openai/o3`）或显式启用推理时，会出现此情况：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    reasoning={"effort": "high"},
)

ai_msg = model.invoke("529 的平方根是多少？")
ai_msg.usage_metadata
```

```text
{'input_tokens': 39,
 'output_tokens': 98,
 'total_tokens': 137,
 'output_token_details': {'reasoning': 62}}
```

### 缓存的输入令牌

`input_token_details.cache_read` 报告从提供商的提示缓存中提供的输入令牌数量，而 `input_token_details.cache_creation` 报告首次调用时写入缓存的令牌。

提示缓存需要在消息内容块中显式设置 `cache_control` 断点。在要缓存的内容块上传递 `{"cache_control": {"type": "ephemeral"}}`：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(model="anthropic/claude-sonnet-4.5")

long_system = "您是一个有用的助手。" * 200
messages = [
    ("system", [{"type": "text", "text": long_system, "cache_control": {"type": "ephemeral"}}]),
    ("human", "说你好。"),
]

# 首次调用写入缓存
ai_msg = model.invoke(messages)
ai_msg.usage_metadata
```

```text
{'input_tokens': 1210,
 'output_tokens': 12,
 'total_tokens': 1222,
 'input_token_details': {'cache_creation': 1201}}
```

```python
# 第二次调用从缓存读取
ai_msg = model.invoke(messages)
ai_msg.usage_metadata
```

```text
{'input_tokens': 1210,
 'output_tokens': 12,
 'total_tokens': 1222,
 'input_token_details': {'cache_read': 1201}}
```

<Note>
    如果消息内容块上没有 `cache_control`，提供商将不会缓存提示，这些字段也不会出现。
</Note>

流式传输时，从最后一个块聚合令牌使用情况：

```python
stream = model.stream("给我讲个笑话。")
full = next(stream)
for chunk in stream:
    full += chunk
full.usage_metadata
```

```text
{'input_tokens': 12,
 'output_tokens': 25,
 'total_tokens': 37}
```

## 提供商路由

OpenRouter 上的许多模型由多个提供商提供服务。`openrouter_provider` 参数让您可以控制哪些提供商处理您的请求以及如何选择它们。

### 排序和过滤提供商

使用 `order` 设置首选提供商顺序。OpenRouter 按顺序尝试每个提供商，如果一个不可用，则回退到下一个：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    openrouter_provider={
        "order": ["Anthropic", "Google"],
        "allow_fallbacks": True,  # 默认；如果需要，回退到顺序列表之外
    },
)
```

要将请求限制为仅特定提供商，请使用 `only`。要排除某些提供商，请使用 `ignore`：

```python
# 仅使用这些提供商（不回退到其他提供商）
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"only": ["OpenAI", "Azure"]},
)

# 使用除 DeepInfra 之外的任何提供商
model = ChatOpenRouter(
    model="meta-llama/llama-4-maverick",
    openrouter_provider={"ignore": ["DeepInfra"]},
)
```

### 按成本、速度或延迟排序

默认情况下，OpenRouter 在提供商之间进行负载均衡，优先考虑较低成本。使用 `sort` 更改优先级：

```python
# 优先选择最快的提供商（最高令牌/秒）
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"sort": "throughput"},
)

# 优先选择延迟最低的提供商
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"sort": "latency"},
)
```

### 数据收集策略

如果您的用例要求提供商不存储或训练您的数据，请将 `data_collection` 设置为 `"deny"`：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    openrouter_provider={"data_collection": "deny"},
)
```

### 按量化过滤

对于开放权重模型，您可以将路由限制为特定精度级别：

```python
model = ChatOpenRouter(
    model="meta-llama/llama-4-maverick",
    openrouter_provider={"quantizations": ["fp16", "bf16"]},
)
```

### 路由参数

`route` 参数控制高级路由行为：

- `"fallback"`：启用跨提供商的自动故障转移（默认行为）。
- `"sort"`：根据 `openrouter_provider` 中配置的排序策略进行路由。

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    route="fallback",
)
```

### 组合选项

提供商选项可以组合在一起：

```python
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={
        "order": ["OpenAI", "Azure"],
        "allow_fallbacks": False,       # 严格——仅使用顺序中的提供商
        "require_parameters": True,     # 跳过不支持所有参数的提供商
        "data_collection": "deny",
    },
)
```

有关选项的完整列表，请参阅 [OpenRouter 提供商路由文档](https://openrouter.ai/docs/guides/routing/provider-selection)。

## 应用归属

OpenRouter 支持通过 HTTP 标头进行应用归属。您可以通过初始化参数或环境变量设置这些：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    app_url="https://myapp.com",   # 或 OPENROUTER_APP_URL 环境变量
    app_title="My App",            # 或 OPENROUTER_APP_TITLE 环境变量
)
```
