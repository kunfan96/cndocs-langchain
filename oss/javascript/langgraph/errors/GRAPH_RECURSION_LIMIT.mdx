---
title: GRAPH_RECURSION_LIMIT
---

你的 LangGraph [`StateGraph`](https://langchain-ai.github.io/langgraph/reference/graphs/#langgraph.graph.state.StateGraph) 在达到停止条件之前达到了最大步骤数。
这通常是由于如下代码导致的死循环：



```typescript
import { StateGraph, StateSchema } from "@langchain/langgraph";
import { z } from "zod/v4";

const State = new StateSchema({
  someKey: z.string(),
});

const builder = new StateGraph(State)
  .addNode("a", ...)
  .addNode("b", ...)
  .addEdge("a", "b")
  .addEdge("b", "a")
  ...

const graph = builder.compile();
```


然而，复杂的图可能会自然地达到默认限制。

## 故障排除

* 如果你不期望你的图进行多次迭代，那么你可能有一个循环。检查你的逻辑是否存在死循环。



* 如果你有一个复杂的图，你可以在调用图时向 `config` 对象传入一个更高的 `recursionLimit` 值，如下所示：

```typescript
await graph.invoke({...}, { recursionLimit: 100 });
```
