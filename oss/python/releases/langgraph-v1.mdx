**LangGraph v1 是一个以稳定性为重点的代理运行时发布版本。** 它保持核心图 API 和执行模型不变，同时改进了类型安全性、文档和开发者体验。

它被设计为与 [LangChain v1](/oss/python/releases/langchain-v1) 无缝配合（其 `create_agent` 基于 LangGraph 构建），这样您可以从高层开始，在需要时降至精细控制。

<CardGroup cols={1}>
    <Card title="稳定的核心 API" icon="sitemap">
        图原语（状态、节点、边）和执行/运行时模型保持不变，使升级更加简单直接。
    </Card>
    <Card title="默认可靠" icon="database">
        具有检查点、持久化、流式传输和人在回路中的持久执行继续是一等公民特性。
    </Card>
    <Card title="与 LangChain v1 无缝集成" icon="link">
        LangChain 的 `create_agent` 运行在 LangGraph 上。使用 LangChain 快速启动；降至 LangGraph 进行自定义编排。
    </Card>
</CardGroup>

升级方式：

<CodeGroup>
```bash pip
pip install -U langgraph
```
```bash uv
uv add langgraph
```
</CodeGroup>

## 弃用 `create_react_agent`

LangGraph 的 [`create_react_agent`](https://reference.langchain.com/python/langchain-classic/agents/react/agent/create_react_agent) 预构建功能已被弃用，取而代之的是 LangChain 的 [`create_agent`](https://reference.langchain.com/python/langchain/agents/factory/create_agent)。它提供了更简单的接口，并通过引入中间件提供了更大的自定义潜力。

* 有关新 [`create_agent`](https://reference.langchain.com/python/langchain/agents/factory/create_agent) API 的信息，请参阅 [LangChain v1 发布说明](/oss/python/releases/langchain-v1#create-agent)。
* 有关从 [`create_react_agent`](https://reference.langchain.com/python/langchain-classic/agents/react/agent/create_react_agent) 迁移到 [`create_agent`](https://reference.langchain.com/python/langchain/agents/factory/create_agent) 的信息，请参阅 [LangChain v1 迁移指南](/oss/python/migrate/langchain-v1#create-agent)。
