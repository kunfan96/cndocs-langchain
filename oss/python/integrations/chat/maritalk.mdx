Maritalk 是由巴西公司 [Maritaca AI](https://www.maritaca.ai) 开发的助手。Maritalk 基于特别训练以更好地理解葡萄牙语的语言模型。

本笔记本演示了如何通过两个示例使用 Maritalk 与 LangChain 集成：

1. 如何简单地使用 Maritalk 完成任务。
2. LLM + RAG：第二个示例展示了当问题的答案位于无法在 Maritalk 的 token 限制内容纳的长文档中时，如何回答该问题。为此，我们将首先使用简单的搜索器（BM25）查找文档中最相关的部分，然后将它们提供给 Maritalk 进行回答。

## API 密钥

您需要一个可以从 chat.maritaca.ai（“API 密钥”部分）获取的 API 密钥。

### 示例 1 - 宠物名字建议

让我们定义我们的语言模型 ChatMaritalk，并使用您的 API 密钥对其进行配置。

```python
from langchain_community.chat_models import ChatMaritalk
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts.chat import ChatPromptTemplate

llm = ChatMaritalk(
    model="sabia-2-medium",  # 可用模型：sabia-2-small 和 sabia-2-medium
    api_key="",  # 在此处插入您的 API 密钥
    temperature=0.7,
    max_tokens=100,
)

output_parser = StrOutputParser()

chat_prompt = ChatPromptTemplate.from_messages(
    [
        (
            "system",
            "您是一个建议宠物名字的专业助手。给定动物，您必须建议 4 个名字。",
        ),
        ("human", "我有一只 {animal}"),
    ]
)

chain = chat_prompt | llm | output_parser

response = chain.invoke({"animal": "dog"})
print(response)  # 应该回答类似于“1. Max\n2. Bella\n3. Charlie\n4. Rocky”的内容
```

### 流式生成

对于涉及长文本生成的任务，如创建一篇详细的文章或翻译一份大型文档，可以优势地在文本生成过程中分段接收响应，而不是等待完整文本。这使得应用程序更加响应和高效，特别是在生成的文本较长时。我们提供了两种满足此需求的方法：一种是同步方法，另一种是异步方法。

#### 同步

```python
from langchain.messages import HumanMessage

messages = [HumanMessage(content="为我的狗建议三个名字")]

for chunk in llm.stream(messages):
    print(chunk.content, end="", flush=True)
```

#### 异步

```python
from langchain.messages import HumanMessage


async def async_invoke_chain(animal: str):
    messages = [HumanMessage(content=f"为我的 {animal} 建议三个名字")]
    async for chunk in llm._astream(messages):
        print(chunk.message.content, end="", flush=True)


await async_invoke_chain("dog")
```

### 示例 2 - RAG + LLM：UNICAMP 2024 入学考试问题回答系统

对于此示例，我们需要安装一些额外的库：


#### 加载数据库

第一步是创建一个包含通知信息的数据库。为此，我们将从 COMVEST 网站下载通知，并将提取的文字分割成 500 字符窗口。

```python
from langchain_community.document_loaders import OnlinePDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 加载 UNICAMP 2024 通知
loader = OnlinePDFLoader(
    "https://www.comvest.unicamp.br/wp-content/uploads/2023/10/31-2023-Dispoe-sobre-o-Vestibular-Unicamp-2024_com-retificacao.pdf"
)
data = loader.load()

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500, chunk_overlap=100, separators=["\n", " ", ""]
)
texts = text_splitter.split_documents(data)
```

#### 创建搜索器

现在我们有了数据库，需要一个搜索器。对于此示例，我们将使用简单的 BM25 作为搜索系统，但这可以被任何其他搜索器（如基于嵌入的搜索）所替代。

```python
from langchain_community.retrievers import BM25Retriever

retriever = BM25Retriever.from_documents(texts)
```

#### 结合搜索系统 + LLM

现在我们有了搜索器，只需要实现一个提示来指定任务并调用链即可。

```python
from langchain_classic.chains.question_answering import load_qa_chain

prompt = """基于以下文档，请回答下面的问题。

{context}

问题：{query}
"""

qa_prompt = ChatPromptTemplate.from_messages([("human", prompt)])

chain = load_qa_chain(llm, chain_type="stuff", verbose=True, prompt=qa_prompt)

query = "入学考试的最大时限是多少？"

docs = retriever.invoke(query)

chain.invoke(
    {"input_documents": docs, "query": query}
)  # 应该输出类似于“入学考试的最大时限是 5 小时”的内容
```
