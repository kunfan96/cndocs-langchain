## 提示缓存

通过在 Amazon Bedrock 上缓存频繁重用的提示前缀，降低推理延迟和输入令牌成本。此模式对于具有长系统提示、许多工具定义或重复文档上下文的多轮代理非常有用。

支持的模型系列包括 Bedrock 上的 **Anthropic Claude** 和 **Amazon Nova**。

<Info>
    了解更多关于 [AWS Bedrock 提示缓存](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html) 策略、支持的模型和区域限制。
</Info>

### LangChain 代理

为 `createAgent` 放置 Bedrock 缓存检查点的代理中间件在 [`langchain-aws`](https://github.com/langchain-ai/langchain-aws/tree/main/libs/aws) Python 包 (`langchain_aws.middleware.prompt_caching`) 中实现。请参阅该仓库以获取安装、配置选项和可运行示例。

对于 TypeScript 中的 Bedrock 聊天模型，请从 [Bedrock 聊天模型集成](/oss/javascript/integrations/chat/bedrock) 和 [@langchain/aws](https://www.npmjs.com/package/@langchain/aws) 包开始。
