>[Parallel](https://platform.parallel.ai/) 是一个专为 LLM 和 AI 应用程序设计的实时网络搜索和内容提取平台。

`ParallelExtractTool` 提供对 Parallel Extract API 的访问，该 API 可从网页中提取干净、结构化的内容。

### 工具特性

- **干净内容提取**：从网页中提取主要内容，移除广告、导航和模板内容
- **Markdown 格式化**：返回格式化为干净 Markdown 的内容
- **批量处理**：在单个 API 调用中从多个 URL 提取
- **元数据提取**：包含标题、发布日期和其他元数据
- **内容长度控制**：配置每次提取的最大字符数
- **错误处理**：优雅处理提取失败的情况，并提供详细的错误信息
- **异步支持**：完全支持异步/等待以获得更好的性能

## 设置

集成位于 `langchain-parallel` 包中。

```python
pip install -qU langchain-parallel
```
### 凭证

前往 [Parallel](https://platform.parallel.ai) 注册并生成 API 密钥。完成后，设置 `PARALLEL_API_KEY` 环境变量：

```python
import getpass
import os

if not os.environ.get("PARALLEL_API_KEY"):
    os.environ["PARALLEL_API_KEY"] = getpass.getpass("Parallel API key:\n")
```

## 实例化

以下展示如何实例化 `ParallelExtractTool` 实例。该工具可使用 API 密钥和内容长度参数进行配置：

```python
from langchain_parallel import ParallelExtractTool

# 基本实例化 - 从环境获取 API 密钥
tool = ParallelExtractTool()

# 使用显式 API 密钥和自定义设置
tool = ParallelExtractTool(
    api_key="your-api-key",
    base_url="https://api.parallel.ai",  # 默认值
    max_chars_per_extract=5000,  # 限制内容长度
)
```

## 调用

### 直接使用参数调用

您可以使用要从中提取内容的 URL 列表调用该工具：

```python
# 从单个 URL 提取
result = tool.invoke(
    {"urls": ["https://en.wikipedia.org/wiki/Artificial_intelligence"]}
)

print(f"提取了 {len(result)} 个结果")
print(f"标题: {result[0]['title']}")
print(f"URL: {result[0]['url']}")
print(f"内容长度: {len(result[0]['content'])} 个字符")
print(f"内容预览: {result[0]['content'][:200]}...")
```

```python
# 从多个 URL 提取
result = tool.invoke(
    {
        "urls": [
            "https://en.wikipedia.org/wiki/Machine_learning",
            "https://en.wikipedia.org/wiki/Deep_learning",
            "https://en.wikipedia.org/wiki/Natural_language_processing",
        ]
    }
)

print(f"提取了 {len(result)} 个结果")
for i, item in enumerate(result, 1):
    print(f"\n{i}. {item['title']}")
    print(f"   URL: {item['url']}")
    print(f"   内容长度: {len(item['content'])} 个字符")

# 示例响应结构：
# [
#     {
#         "url": "https://example.com/article",
#         "title": "Article Title",
#         "content": "# Article Title\n\nMain content in markdown...",
#         "publish_date": "2024-01-15"  # 可选
#     }
# ]
```

### 使用 `ToolCall` 调用

我们也可以使用模型生成的 `ToolCall` 调用该工具，这种情况下将返回 `ToolMessage`：

```python
# 这通常由模型生成，但为了演示目的，我们直接创建一个工具调用。
model_generated_tool_call = {
    "args": {
        "urls": [
            "https://en.wikipedia.org/wiki/Climate_change",
            "https://en.wikipedia.org/wiki/Renewable_energy",
        ]
    },
    "id": "call_123",
    "name": tool.name,  # "parallel_extract"
    "type": "tool_call",
}

result = tool.invoke(model_generated_tool_call)
print(result)
print(f"工具名称: {tool.name}")  # parallel_extract
print(f"工具描述: {tool.description}")
```

### 异步用法

该工具支持完全的异步/等待操作，以在异步应用程序中获得更好的性能：

```python
async def extract_async():
    return await tool.ainvoke(
        {
            "urls": [
                "https://en.wikipedia.org/wiki/Python_(programming_language)",
                "https://en.wikipedia.org/wiki/JavaScript",
            ]
        }
    )


# 运行异步提取
result = await extract_async()
print(f"异步提取了 {len(result)} 个结果")
```

### 高级特性

提取工具支持使用搜索目标/查询、获取策略以及对摘录和完整内容的细粒度控制进行聚焦提取：

```python
# 使用搜索目标提取聚焦摘录
result = tool.invoke(
    {
        "urls": ["https://en.wikipedia.org/wiki/Artificial_intelligence"],
        "search_objective": "What are the main applications and ethical concerns of AI?",
        "excerpts": {"max_chars_per_result": 2000},
        "full_content": False,
    }
)

print(f"提取了聚焦摘录: {len(result[0].get('excerpts', []))} 个摘录")
print(f"内容预览: {result[0]['content'][:200]}...")

# 使用获取策略提取新鲜内容
result = tool.invoke(
    {
        "urls": ["https://en.wikipedia.org/wiki/Quantum_computing"],
        "fetch_policy": {
            "max_age_seconds": 86400,  # 1 天缓存
            "timeout_seconds": 60,
            "disable_cache_fallback": False,
        },
        "full_content": {"max_chars_per_result": 5000},
    }
)

print(f"内容长度: {len(result[0]['content'])} 个字符")
```

### 错误处理

该工具会优雅处理提取失败的 URL，将其包含在结果中并附带错误信息：

```python
# 有效和无效 URL 的混合
result = tool.invoke(
    {
        "urls": [
            "https://en.wikipedia.org/wiki/Artificial_intelligence",
            "https://this-domain-does-not-exist-12345.com/",
        ]
    }
)

for item in result:
    if "error_type" in item:
        print(f"失败: {item['url']}")
        print(f"错误: {item['content']}")
    else:
        print(f"成功: {item['url']}")
        print(f"提取了 {len(item['content'])} 个字符")
```

## 最佳实践

- **批量 URL**：在单个调用中提取多个 URL 以获得更好的性能
- **设置内容限制**：使用 `max_chars_per_extract` 控制响应大小和令牌使用量
- **处理错误**：检查结果中的 `error_type` 以识别提取失败的情况
- **使用异步以获得性能**：在异步应用程序中使用 `ainvoke()` 以获得更好的性能
- **元数据字段**：使用 `publish_date` 和其他可用元数据以获取上下文

## 响应格式

该工具返回一个字典列表，格式如下：

```python
[
    {
        "url": "https://example.com/article",
        "title": "Article Title",
        "content": "# Article Title\n\nMain content formatted as markdown...",
        "publish_date": "2024-01-15"  # 可选，如果可用
    },
    # 对于失败的提取：
    {
        "url": "https://failed-site.com",
        "title": None,
        "content": "Error: 404 Not Found",
        "error_type": "http_error"
    }
]
```
