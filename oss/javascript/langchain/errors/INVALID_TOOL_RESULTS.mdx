---
title: 无效的工具结果
---

<Note>
    目前仅在 `langchainjs`（JavaScript/TypeScript）中使用。
</Note>

当在工具调用操作期间向模型传递不匹配、不足或过多的 [`ToolMessage`](https://reference.langchain.com/javascript/langchain-core/messages/ToolMessage) 对象时，会发生此错误。

此错误源于一个基本要求：带有 `tool_calls` 的助手消息必须后跟响应每个 `tool_call_id` 的工具消息。

当模型返回带有工具调用的 [`AIMessage`](https://reference.langchain.com/javascript/langchain-core/messages/AIMessage) 时，您必须为每个工具调用提供恰好一个对应的 [`ToolMessage`](https://reference.langchain.com/javascript/langchain-core/messages/ToolMessage)，且 `tool_call_id` 值必须匹配。

## 常见原因

* **响应不足**：如果模型请求两个工具执行，但您只提供一个响应消息，模型会拒绝不完整的消息链
* **重复响应**：为同一工具调用 ID 提供多个 [`ToolMessage`](https://reference.langchain.com/javascript/langchain-core/messages/ToolMessage) 对象会导致被拒绝，ID 不匹配的情况也是如此
* **孤立的工具消息**：发送一个 [`ToolMessage`](https://reference.langchain.com/javascript/langchain-core/messages/ToolMessage) 而没有前置的包含工具调用的 [`AIMessage`](https://reference.langchain.com/javascript/langchain-core/messages/AIMessage) 违反了协议要求

以下是一个有问题的模式示例：

```javascript
// 模型请求两个工具调用
responseMessage.tool_calls // 返回 2 个调用

// 但只提供了一个 ToolMessage
chatHistory.push({
  role: "tool",
  content: toolResponse,
  tool_call_id: responseMessage.tool_calls[0].id
});

await modelWithTools.invoke(chatHistory); // 失败，出现 INVALID_TOOL_RESULTS 错误
```

## 故障排除

要解决此错误：

* **计数匹配对**：确保在前置的 [`AIMessage`](https://reference.langchain.com/javascript/langchain-core/messages/AIMessage) 中，每个工具调用都存在一个 [`ToolMessage`](https://reference.langchain.com/javascript/langchain-core/messages/ToolMessage)
* **验证 ID**：确认每个 `ToolMessage.tool_call_id` 都与实际的工具调用标识符匹配
