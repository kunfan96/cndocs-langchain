---
title: MULTIPLE_SUBGRAPHS
---

此错误发生在您[在节点内调用子图](/oss/python/langgraph/use-subgraphs#call-a-subgraph-inside-a-node)多次，且子图使用 `checkpointer=True`（延续模式）编译时。

## 故障排除

根据您的需求选择以下选项之一：

1. **不需要中断？** 使用 `checkpointer=False` 完全退出检查点：
   ```python
   subgraph = subgraph_builder.compile(checkpointer=False)
   ```

2. **需要中断但不需要跨调用持久化？** 通过省略 `checkpointer` 使用默认继承模式：
   ```python
   subgraph = subgraph_builder.compile()
   ```
   每次调用都会获得一个唯一的命名空间，因此并行执行有效。子图每次都会重新开始，但可以使用 `interrupt()`。

3. **需要跨调用持久化？** 使用 `checkpointer=True`。LangGraph 为每次调用分配一个基于位置的命名空间后缀（`calling_node`、`calling_node|1` 等）以防止冲突。对于稳定的、基于名称的命名空间，请使用唯一的节点名称包装每个子图——参见[并行子图](/oss/python/langgraph/use-subgraphs#subgraph-persistence)。
