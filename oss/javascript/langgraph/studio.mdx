title: LangSmith Studio
---

在本地使用 LangGraph 构建智能体时，可视化智能体内部发生的事情、实时与其交互以及在问题发生时调试它们是非常有帮助的。**LangSmith Studio** 是一个免费的可视化界面，用于从本地机器开发和测试你的 LangGraph 智能体。

Studio 连接到你本地运行的智能体，向你展示智能体采取的每个步骤：发送给模型的提示、工具调用及其结果以及最终输出。你可以测试不同的输入、检查中间状态并迭代智能体的行为，无需额外的代码或部署。

本页介绍如何使用你的本地 LangGraph 智能体设置 Studio。

## 前置条件

在开始之前，请确保你具备以下条件：

- **LangSmith 账户**：在 [smith.langchain.com](https://smith.langchain.com) 免费注册或登录。
- **LangSmith API 密钥**：按照[创建 API 密钥](/langsmith/create-account-api-key#create-an-api-key)指南操作。
- 如果你不想将数据[追踪](/langsmith/observability-concepts#traces)到 LangSmith，请在应用程序的 `.env` 文件中设置 `LANGSMITH_TRACING=false`。禁用追踪后，不会有数据离开你的本地服务器。

## 设置本地 Agent 服务器

### 1. 安装 LangGraph CLI

[LangGraph CLI](/langsmith/cli) 提供了一个本地开发服务器（也称为 [Agent Server](/langsmith/agent-server)），将你的智能体连接到 Studio。



```shell
npx @langchain/langgraph-cli
```



### 2. 准备你的智能体

如果你已经有一个 LangGraph 智能体，可以直接使用它。这个示例使用一个简单的邮件智能体：



```typescript title="agent.ts"
import { StateSchema, MessagesValue, GraphNode, StateGraph, START, END } from "@langchain/langgraph";

const State = new StateSchema({
  messages: MessagesValue,
});

const mockLlm: GraphNode<typeof State> = (state) => {
  return { messages: [{ role: "ai", content: "hello world" }] };
};

export const agent = new StateGraph(State)
  .addNode("mock_llm", mockLlm)
  .addEdge(START, "mock_llm")
  .addEdge("mock_llm", END)
  .compile();
```



### 3. 环境变量

Studio 需要 LangSmith API 密钥来连接你的本地智能体。在项目根目录创建一个 `.env` 文件，并从 [LangSmith](https://smith.langchain.com/settings) 添加你的 API 密钥。

<Warning>
    确保你的 `.env` 文件不会被提交到版本控制系统，如 Git。
</Warning>

```bash .env
LANGSMITH_API_KEY=lsv2...
```

### 4. 创建 LangGraph 配置文件

LangGraph CLI 使用配置文件来定位你的智能体并管理依赖项。在应用程序目录中创建一个 `langgraph.json` 文件：



```json title="langgraph.json"
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.ts:agent"
  },
  "env": ".env"
}
```



<Info>
有关配置文件中 JSON 对象每个键的详细说明，请参阅 [LangGraph 配置文件参考](/langsmith/cli#configuration-file)。
</Info>

此时，项目结构将如下所示：



```bash
my-app/
├── src
│   └── agent.ts
├── .env
├── package.json
└── langgraph.json
```



### 5. 安装依赖



```shell
yarn install
```



### 6. 在 Studio 中查看你的智能体

启动开发服务器将你的智能体连接到 Studio：



```shell
npx @langchain/langgraph-cli dev
```



<Warning>
Safari 会阻止 `localhost` 连接到 Studio。要解决此问题，请使用 `--tunnel` 标志运行上述命令以通过安全隧道访问 Studio。你需要通过在 Studio UI 中点击 **Connect to a local server** 手动将隧道 URL 添加到允许的来源中。请参阅[故障排除指南](/langsmith/troubleshooting-studio#safari-connection-issues)了解步骤。
</Warning>

服务器运行后，你的智能体可以通过 `http://127.0.0.1:2024` 的 API 和 `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024` 的 Studio UI 访问：

<Frame>
![Studio UI 中的智能体视图](/oss/images/studio_create-agent.png)
</Frame>

Studio 连接到你的本地智能体后，你可以快速迭代智能体的行为。运行测试输入，检查完整的执行轨迹，包括提示、工具参数、返回值以及令牌/延迟指标。当出现问题时，Studio 会捕获异常及周围状态，帮助你了解发生了什么。

开发服务器支持热重载——在代码中更改提示或工具签名，Studio 会立即反映这些变化。从任何步骤重新运行对话线程以测试你的更改，无需从头开始。这种工作流程可以从简单的单工具智能体扩展到复杂的多节点图。
