>Teradata Vector Store 旨在企业数据平台内高效存储、索引和搜索高维向量嵌入。

本指南将向您展示如何快速启动并运行 TeradataVectorStore，用于您的语义搜索和 RAG 应用。无论您是 Teradata 的新手，还是希望为现有数据工作流添加 AI 功能，本指南都将引导您了解所需的一切知识。

**TeradataVectorStore 的特别之处？**
- 基于企业级 Teradata Vantage 平台构建。
- 与您现有的数据仓库无缝集成。
- 支持多种向量搜索算法，适用于不同用例。
- 可从原型扩展到生产工作负载。

## 设置

在开始之前，您需要安装必要的软件包。TeradataVectorStore 是 `langchain-teradata` 软件包的一部分，该软件包还包含其他用于 LangChain 的 Teradata 集成。

**初次接触 Teradata？** 请参考：
- [Teradata VantageCloud Lake](https://www.teradata.com/platform/vantagecloud)
- 开始使用 [VantageCloud Lake](https://docs.teradata.com/r/Lake-Getting-Started-with-VantageCloud-Lake/)

### 安装

    ```python pip
    pip install langchain-teradata
    ```

### 凭据

**连接到 Teradata：** `create_context()` 函数建立您与 Teradata Vantage 系统的连接。这是 teradataml（以及 TeradataVectorStore）知道连接到哪个数据库并进行身份验证的方式。

**您需要：**
- **hostname**：您的 Teradata 系统地址
- **username/password**：您的数据库凭据
- **base_url**：您的 Teradata 系统的 API 端点
- **pat_token**：用于 API 身份验证的个人访问令牌
- **pem_file**：用于安全连接的 SSL 证书文件

**更多信息** 请查看 [Teradata Vector Store 用户指南](https://docs.teradata.com/r/Enterprise_IntelliFlex_VMware/Teradata-Vector-Store-User-Guide/Setting-up-Vector-Store/Required-Privileges) 以获取详细的设置说明。

**有关 teradataml 的信息** 请参考 [TeradataML 用户指南](https://docs.teradata.com/r/Enterprise_IntelliFlex_VMware/Teradata-Package-for-Python-User-Guide/Introduction-to-Teradata-Package-for-Python)

```python
import os
from getpass import getpass
from teradataml import create_context

os.environ['TD_HOST'] = getpass(prompt='hostname: ')
os.environ['TD_USERNAME'] = getpass(prompt='username: ')
os.environ['TD_PASSWORD'] = getpass(prompt='password: ')
os.environ['TD_BASE_URL'] = getpass(prompt='base_url: ')
os.environ['TD_PAT_TOKEN'] = getpass(prompt='pat_token: ')
os.environ['TD_PEM_FILE'] = getpass(prompt='pem_file: ')
create_context()
```

## 实例化

**初始化您的嵌入**

**TeradataVectorStore 支持三种类型的嵌入对象：**
1. **字符串标识符**（例如 "amazon.titan-embed-text-v1"）
2. **TeradataAI 对象**
3. **LangChain 嵌入对象** - 与 LangChain 兼容的嵌入模型对象

```python
# 初始化嵌入
from langchain_aws import BedrockEmbeddings
embeddings = BedrockEmbeddings(model_id="amazon.titan-embed-text-v1", region_name="us-west-2")
```

**创建您的第一个向量存储**

让我们从一些示例文档开始，创建一个向量存储。`from_documents()` 方法是最直接的入门方式之一 - 只需传入您的文档，其余工作由 TeradataVectorStore 处理。

**底层发生了什么：**
- 您的文档被转换为 Teradataml Dataframe 并传递给向量存储
- 为每个 Document 对象生成并存储嵌入
- 自动创建索引以实现快速的相似性搜索和聊天操作

```python
from langchain_teradata import TeradataVectorStore
from langchain_core.documents import Document
# 关于不同主题的示例文档
docs = [
    Document(page_content="Teradata 为企业提供可扩展的数据分析解决方案。"),
    Document(page_content="机器学习模型需要高质量的训练数据才能表现良好。"),
    Document(page_content="向量数据库支持超越关键词匹配的语义搜索功能。"),
    Document(page_content="LangChain 简化了使用大型语言模型构建应用程序的过程。"),
    Document(page_content="数据仓库已发展到支持实时分析和 AI 工作负载。")
]

# 创建向量存储
vs = TeradataVectorStore.from_documents(
    name="my_knowledge_base",
    documents=docs,
    embedding=embeddings
)

print("向量存储创建成功！")
```

创建向量存储后，最好验证一切是否设置正确。TeradataVectorStore 提供了有用的方法来监控您的操作并了解幕后发生的情况。

**为什么要检查状态？**
- **操作跟踪**：查看向量存储创建的确切阶段。
- **故障排除**：快速识别设置过程中是否出现问题。
- **进度监控**：对于大型数据集，跟踪嵌入生成进度。
- **验证**：确认您的向量存储已准备好进行查询。

```python
# 检查存储的状态。
vs.status()
```

想看看向量存储内部的实际内容吗？`get_details()` 方法为您提供设置的全面概述 - 可将其视为向量存储的“仪表板”。

**您将看到：**
- **对象清单**：您已添加的表或文档数量。
- **搜索参数**：当前算法设置（HNSW、K-means 等）
- **配置详情**：嵌入维度、距离度量和索引选项。
- **性能设置**：Top-k 值、相似性阈值和其他查询参数。

```python
vs.get_details()
```

## 管理向量存储

### 向向量存储添加项目

TeradataVectorStore 的最佳功能之一是扩展知识库非常容易。随着业务增长和文档增多，您可以持续添加它们，而无需从头开始重建一切。

**实际场景：**
- 在创建新产品文档时添加。
- 包含新的研究论文或行业报告。
- 纳入客户反馈和支持文档。
- 更新最新的政策或程序变更。

**企业优势：** 由于一切都在 Teradata 上运行，您可以轻松地从现有表、数据仓库或实时数据流中添加数据，而无需复杂的数据移动。

```python
# 添加更多文档
additional_docs = [
    Document(page_content="检索增强生成结合了搜索和语言模型的力量。"),
    Document(page_content="Teradata 的向量功能支持结构化和非结构化数据分析。")
]

vs.add_documents(documents=additional_docs)
print("已向向量存储添加更多知识！")
```

```python
# 检查新存储的状态。
vs.status()
```

## 查询向量存储

一旦您的向量存储创建完毕并添加了相关文档，您很可能希望在运行链或代理时对其进行查询。

### 直接查询

现在让我们在向量存储中搜索信息。与传统的关键词搜索不同，向量搜索理解您问题背后的含义。询问“AI 应用程序”，它可能会返回关于“机器学习模型”的结果，因为它理解这些概念是相关的。

**相似性搜索的工作原理：**
- 您的问题被转换为向量嵌入（就像您的文档一样）。
- TeradataVectorStore 计算您的问题与存储文档之间的相似性分数。
- 返回最相关的结果，并按相似性排序。

```python
# 提问
question = "什么是向量数据库？"
results = vs.similarity_search(question=question, return_type = "json")

print("找到相关信息：")
for result in results.similar_objects:
    print(f" {result}")
```

### 通过转换为检索器进行查询

您还可以将向量存储转换为检索器，以便在链中更轻松地使用。

```python
# 为您的 RAG 管道创建检索器
retriever = vs.as_retriever(search_type="similarity")

# 测试检索器
retrieved_docs = retriever.invoke("告诉我 Teradata 的功能")

print("为 RAG 检索到的文档：")
for doc in retrieved_docs:
    print(f"- {doc.page_content}")
```

## 用于检索增强生成的用法

`ask()` 结合了向量搜索和语言模型生成的力量。它不只是返回原始文档块，而是提供连贯、上下文相关的答案。

**两步过程：**
1. **检索**：从您的向量存储中查找最相关的文档。
2. **生成**：使用这些文档作为上下文来生成自然语言响应。

**为什么这很强大：** 您的 AI 响应基于您的实际数据，减少了幻觉并确保了准确性。这就像拥有一位真正阅读过您公司文档的知识渊博的助手！

```python
# 获取全面答案
response = vs.ask(question="使用向量数据库有哪些好处？")
print("AI 响应：")
print(response)
```

检索增强生成 (RAG) 是为大多数现代 AI 助手和聊天机器人提供支持的技术。TeradataVectorStore 与 LangChain 无缝集成，使构建 RAG 应用变得简单。

**一个好的 RAG 应用具备什么：**
- **相关检索**：您的向量存储找到正确的信息。
- **上下文生成**：语言模型有效地使用该信息。
- **来源透明度**：用户可以看到答案的来源。

**如何与 TeradataVectorStore 一起工作：**
- 您可以使用您的向量存储作为检索器来获取最相关的文档，然后将这些文档传递给 LangChain 工作流中的 RAG 链。
- 这为您提供了构建自定义管道的灵活性，同时利用 Teradata 强大的向量搜索功能。

现在，让我们构建一个完整的 RAG 管道，将您的 TeradataVectorStore 检索器与语言模型相结合。这展示了 RAG 的全部力量 - 从您的向量存储中检索相关信息并使用它来生成知情的响应。

**此管道中发生的情况：**

- 检索：您的向量存储为问题查找最相关的文档。
- 上下文准备：这些文档成为语言模型的上下文。
- 生成：语言模型根据您的实际数据生成答案。
- 输出解析：为您的应用程序准备干净、格式化的响应。

**实际应用：**

- 客户支持：使用您的产品文档回答问题。
- 研究协助：查询您组织的知识库。
- 合规性：确保响应基于批准的公司信息。

```python
from langchain_core.runnables import RunnablePassthrough
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain.chat_models import init_chat_model

#示例：简单的 RAG 链
# 初始化聊天模型
llm = init_chat_model("anthropic.claude-3-5-sonnet-20240620-v1:0", 
                      model_provider="bedrock_converse", 
                      region_name="<输入区域>",
                      aws_access_key_id = "<输入 AWS 访问密钥>" ,
                      aws_secret_access_key = "<输入 AWS 密钥>"
                      )


# 为 LLM 创建提示模板，使用检索到的上下文格式化其响应
prompt = PromptTemplate.from_template(
    "使用以下上下文回答问题。\n上下文：\n{context}\n\n问题：{question}\n答案："
)

# 构建 RAG 链：检索上下文、格式化提示、生成答案并解析输出
rag_chain = (
    {
        "context": retriever,
        "question": RunnablePassthrough()
    }
    | prompt
    | llm
    | StrOutputParser()
)

# 使用示例问题调用 RAG 链并打印响应
response = rag_chain.invoke("向量存储的好处")
print(response)
```

## 处理不同的数据类型

TeradataVectorStore 的灵活性在处理不同类型的数据源时真正显现出来。根据您的起点，您可以选择最合适的方法。

**选择您的起点：**
- **有 PDF 文档？** 使用 `from_documents()` 和文件路径
- **处理数据库表？** 使用 `from_datasets()` 和 DataFrame
- **已有嵌入？** 使用 `from_embeddings()` 直接导入它们

### 从 PDF 文件
```python
# 基于文件的向量存储（来自 PDF）
pdf_vs = TeradataVectorStore.from_documents(
    name="pdf_knowledge",
    documents="path/to/your/document.pdf",  # 或 PDF 路径列表
    embedding=embeddings
)
```

### 从数据库表
```python
# 基于内容的存储（来自现有表）
from teradataml import DataFrame
table_data = DataFrame('your_table_name')

table_vs = TeradataVectorStore.from_datasets(
    name="table_knowledge", 
    data=table_data,
    data_columns=["text_column"],
    embedding=embeddings
)
```

### 从预计算的嵌入
```python
# 如果您已有嵌入
embedding_vs = TeradataVectorStore.from_embeddings(
    name="embedding_store",
    data=your_embedding_data,
    data_columns="embedding_column"
)
```

***注意*** <br />
处理表（和嵌入表）时，`data_columns` 参数是必需的。这会告诉 TeradataVectorStore 哪些列包含您要转换为嵌入的文本内容。可以将其视为将服务指向正确的信息。

例如，如果您的表有 id、title、description 和 category 等列，您可以指定 data_columns=["description"] 仅嵌入描述文本，或 data_columns=["title", "description"] 组合两个字段。

下面是一个使用 `teradatagenai` 加载示例表并基于其创建内容存储的小示例。对于 data_columns，我们将传递 "rev_text" 列，该列将用于生成嵌入。

```python
from teradatagenai import load_data

# 将示例数据加载到 Teradata
load_data("byom", "amazon_reviews_25")

# 从 Teradata 表创建向量存储
td_vs = TeradataVectorStore.from_datasets(
    name="table_store_amazon",
    data="amazon_reviews_25",
    data_columns="rev_text",
    embedding=embeddings)
```

```python
# 检查新存储的状态
td_vs.status()
```
