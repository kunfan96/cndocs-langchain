本指南将引导您开始使用 [Isaacus](/oss/python/integrations/providers/isaacus) 的 LangChain 集成来生成法律嵌入。

## 1. 设置您的账户

前往 [Isaacus 平台](https://platform.isaacus.com/accounts/signup/) 创建新账户。

注册完成后，[添加支付方式](https://platform.isaacus.com/billing/) 以领取您的[免费额度](https://docs.isaacus.com/pricing/credits)。

添加支付方式后，[创建新的 API 密钥](https://platform.isaacus.com/users/api-keys/)。

请妥善保管您的 API 密钥。创建后您将无法再次查看它。但请放心，您随时可以生成一个新的。

## 2. 安装 isaacus API 客户端

账户设置完成后，请安装 [Isaacus LangChain](https://pypi.org/project/langchain-isaacus/) 集成包。

<CodeGroup>
```bash pip
pip install langchain-isaacus
```

```bash uv
uv add langchain-isaacus
```
</CodeGroup>

## 3. 嵌入文档

安装好 API 客户端后，让我们开始嵌入第一个法律查询和文档。

首先，您需要**使用 API 密钥初始化客户端**。您可以通过设置 `ISAACUS_API_KEY` 环境变量或直接传递该密钥来实现，本示例采用直接传递的方式。

我们将使用 [Kanon 2 Embedder](https://isaacus.com/blog/introducing-kanon-2-embedder)，截至 2025 年 10 月 20 日，它是 [Massive Legal Embedding Benchmark](https://isaacus.com/blog/introducing-mleb) 上全球最准确的法律嵌入模型。

```python
from langchain_isaacus import IsaacusEmbeddings

# 为 Kanon 2 Embedder 创建 Isaacus API 客户端。
client = IsaacusEmbeddings(
    model="kanon-2-embedder",
    api_key="PASTE_YOUR_API_KEY_HERE",
    # dimensions=1792, # 您可以选择指定一个较低的维度。
)
```
接下来，让我们获取一个要嵌入的法律文档。在本示例中，我们将使用 [GitHub 的服务条款](https://github.com/terms)。

```python
import isaacus

tos = isaacus.Isaacus().get(path="https://examples.isaacus.com/github-tos.md", cast_to=str)
```

我们的目标是根据相关搜索查询检索 GitHub 服务条款。

为此，我们将首先使用 API 客户端的 `.embed_documents()` 方法对文档进行嵌入。使用此方法表明我们正在嵌入文档（而非搜索查询），这对于确保我们的嵌入向量针对检索任务进行了优化（而非分类或句子相似度等其他任务）至关重要。

```python
document_embedding = client.embed_documents(texts=[tos])[0]
```

现在，让我们嵌入两个搜索查询：一个与文档明显相关，另一个明显不相关。这次我们将使用 API 客户端的 `.embed_query()` 方法，该方法表明我们正在嵌入搜索查询。

```python
relevant_query_embedding = client.embed_query(text="What are GitHub's billing policies?")
irrelevant_query_embedding = client.embed_query(text="What are Microsoft's billing policies?")
```

为了评估查询与文档的相关性，我们可以计算它们各自的嵌入向量与文档嵌入向量之间的余弦相似度。

余弦相似度用于衡量两组数字的相似程度（具体而言，是内积空间中两个向量之间夹角的余弦值）。理论上，其取值范围为 $$-1$$ 到 $$1$$，其中 $$1$$ 表示向量完全相同，$$0$$ 表示向量正交（即完全不相似），$$-1$$ 表示向量完全相反。然而在实际应用中，文本嵌入向量的余弦相似度通常介于 $$0$$ 到 $$1$$ 之间（因为它们通常为非负值）。

Isaacus 的嵌入模型经过优化，使其生成的嵌入向量的余弦相似度大致对应于原始文本在语义上的相似程度。然而，与 Isaacus 的通用分类器不同，Isaacus 嵌入模型的得分并未校准为概率解释，仅作为相似度的相对度量，因此它们在排序搜索结果时最为有用。

为方便起见，我们的 Python 示例使用了 [`numpy`](https://numpy.org/) 的 `dot` 函数来计算嵌入向量的点积（由于所有嵌入向量均已进行 L2 归一化，点积等价于余弦相似度）。如果您愿意，也可以使用其他库来计算嵌入向量的余弦相似度（例如通过 [`torch`](https://pytorch.org/) 的 `torch.nn.functional.cosine_similarity`），或者自行编写实现代码。

```python
import numpy as np

relevant_similarity = np.dot(relevant_query_embedding, document_embedding)
irrelevant_similarity = np.dot(irrelevant_query_embedding, document_embedding)

print(f"Similarity of relevant query to the document: {relevant_similarity * 100:.2f}")
print(f"Similarity of irrelevant query to the document: {irrelevant_similarity * 100:.2f}")
```

输出结果应类似于以下内容：
```
Similarity of relevant query to the document: 52.87
Similarity of irrelevant query to the document: 24.86
```

正如您所看到的，相关查询与文档的相似度分数远高于不相关查询，这表明我们的嵌入模型已成功捕捉到了文本的语义含义。

以上就是全部内容！您已成功使用 LangChain 结合 Isaacus API 完成了法律文档和查询的嵌入。
