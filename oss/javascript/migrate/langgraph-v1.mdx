sidebarTitle: LangGraph v1

LangGraph v1 是针对代理运行时的稳定性发布，保留了核心图 API 与执行模型，同时改进了类型安全、文档与开发 ergonomics（开发体验）。

它与 [LangChain v1](/oss/javascript/releases/langchain-v1) 无缝协作（LangChain 的 `createAgent` 即运行在 LangGraph 之上），你可以先使用 LangChain 快速上手，在需要时再降到 LangGraph 进行精细编排。

要升级：

<CodeGroup>
```bash npm
npm install @langchain/langgraph @langchain/core
```
```bash pnpm
pnpm add @langchain/langgraph @langchain/core
```
```bash yarn
yarn add @langchain/langgraph @langchain/core
```
```bash bun
bun add @langchain/langgraph @langchain/core
```
</CodeGroup>

有关完整变更列表，请参见 [迁移指南](/oss/javascript/migrate/langgraph-v1)。

## `createReactAgent` 的弃用

LangGraph 的 `createReactAgent` 预构建已弃用，推荐使用 LangChain 的 `createAgent`。新 API 更简洁并通过 middleware 提供更强的自定义能力。

## 类型化中断（Typed interrupts）

`StateGraph` 构造函数现在接受中断类型的映射，可更精确地约束图中可使用的中断类型（示例见原文）。

## 前端 SDK 增强

### 事件流编码

底层 `toLangGraphEventStream` 辅助已移除。SDK 原生处理流式响应，你可以通过向 `graph.stream` 传入 `encoding` 参数选择传输格式，方便在 SSE 与普通 JSON 间切换而无需修改 UI 逻辑。

### `useStream` 的自定义传输

React 的 `useStream` 钩子现在支持可插拔的传输层，便于在不改动 UI 代码情况下自定义网络层（示例见原文）。

## 报告问题

若在 v1 中发现任何问题，请在 GitHub 上以 `v1` 标签报告。
