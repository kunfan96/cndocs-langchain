此页面解释了 [LangChain 深度代理](/oss/python/deepagents/overview) 与 [Claude Agent SDK](https://platform.anthropic.com/docs/en/agent-sdk/overview) 的区别。两者都是构建自定义代理的框架，但它们在执行环境、部署和供应商耦合方面做出了不同的权衡。

<Info>
    深度代理已在生产环境中由 [OpenSWE](https://github.com/langchain-ai/open-swe) 和 [LangSmith Fleet](/langsmith/fleet/index) 使用。
</Info>

## 一目了然

| | **深度代理** | **Claude Agent SDK** |
| --- | --- | --- |
| **代理运行的位置** | 在沙盒内，或在沙盒外远程执行命令 | 沙盒内 |
| **执行后端** | 可插拔：[本地、虚拟文件系统、远程沙盒或自定义](/oss/python/deepagents/backends) | 执行沙盒的本地文件系统 |
| **模型提供商** | 任何（Anthropic, OpenAI, Google 等100+） | Claude (Anthropic, Bedrock, Vertex, Azure) |
| **部署** | [通过 LangSmith 管理云](/oss/python/langgraph/deploy) 使用 `deepagents deploy`，或自托管一个[独立镜像](/langsmith/deploy-standalone-server) 通过 `langgraph build` | 自托管。您构建服务器、认证和流式传输层。[Claude 管理代理](https://platform.claude.com/docs/en/managed-agents/overview) 是单独的产品 |
| **多租户** | [内置](/oss/python/deepagents/going-to-production#multi-tenancy): 作用域线程、每个用户沙盒、RBAC | 自行构建 |
| **许可证** | MIT | MIT (Claude Code 本身是专有的) |

## 主要差异

### 代理和执行环境

有 [两种模式将代理连接到沙盒](https://www.langchain.com/blog/the-two-patterns-by-which-agents-connect-sandboxes): 将代理 *运行* 在沙盒内，或在外部运行并 **使用沙盒作为工具**。

Claude Agent SDK 只支持第一种。您的代理在沙盒内部运行，并执行针对沙盒本地文件系统的工具命令。Anthropic 的托管模型 [Claude 管理代理](https://platform.claude.com/docs/en/managed-agents/overview) 使用解耦模式，这反映了生产代理架构的发展方向。

深度代理支持这两种模式，并允许您选择一个 [后端](/oss/python/deepagents/backends#quickstart) 将它们连接起来。实际上这意味着您可以：

- 在沙盒内部运行代理（与 Claude Agent SDK 相同的模型）。
- 在长生命周期容器中运行代理并 [使用远程沙盒作为工具](https://www.langchain.com/blog/the-two-patterns-by-which-agents-connect-sandboxes)，通过网络执行命令。
- 为测试换入虚拟文件系统，或为您的基础设施自定义后端。

### 多租户

当您将应用程序生产化时，通常会向许多最终用户公开它，并且必须为每个用户提供隔离的环境。

在 Claude Agent SDK 中，SDK 将代理绑定到其沙盒。为了给每个用户提供一个独立的执行环境，您必须构建一个 API 包装器，每次启动一个新的沙盒，跟踪哪个沙盒属于谁，然后在其后销毁它。

深度代理直接处理此问题：在框架中为用户或助手配置沙盒 [生命周期](/oss/python/deepagents/going-to-production#lifecycle)，包含作用域线程、运行历史记录和 [RBAC](/langsmith/rbac)。如果您使用 [LangSmith 沙盒](/langsmith/sandbox-auth-proxy)，您还可以获得一个认证代理，以便最终用户可以从沙盒调用第三方 API 而无需为每个用户提供凭证。

### 生产代理服务器

要将 [自托管 Claude Agent SDK](https://code.claude.com/docs/en/agent-sdk/hosting) 应用程序暴露给最终用户，您需要编写自己的 HTTP/WebSocket 或 SSE 服务器来调用代理、流式传输令牌并管理对话线程。该服务器是您构建、操作和保护的。

深度代理部署包括一个 [代理服务器](/langsmith/agent-server)，提供流式端点、线程管理、运行历史记录、Webhook 和 [认证](/langsmith/auth)。

### 管理云或自托管

Claude Agent SDK 部署是 [自托管的](https://code.claude.com/docs/en/agent-sdk/hosting)。SDK 和 [Claude 管理代理](https://platform.claude.com/docs/en/managed-agents/overview) 是单独的产品。针对 SDK 编写的代码不会直接部署到管理提供程序。

深度代理在无需更改代码的情况下运行两种模式：

- **托管：** 通过 `deepagents deploy` 部署到 [LangSmith 管理云](/oss/python/langgraph/deploy)。
- **自托管：** 运行 `langgraph build` 来生成一个您可以部署到任何地方的 [独立 Docker 镜像](/langsmith/deploy-standalone-server)。

### 语言模型

Claude Agent SDK 执行将模型、后端和部署捆绑在一起，并在三者之间优化支持。

使用深度代理，您可以选择模型提供商、执行后端和部署目标。通过选择此框架，您可以保留对模型和基础设施的最大灵活性。

## 总结

- **选择深度代理** 如果您希望获得模型和基础设施的灵活性、内置多租户部署选项，并且可以无需更改代码直接运行托管或自托管。
- **选择 Claude Agent SDK** 如果您致力于使用 Claude 并希望自行构建 API、认证和多租户层。
