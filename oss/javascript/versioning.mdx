title: 版本控制
---

我们的 OSS 版本号遵循 [语义化版本控制 (Semantic Versioning)](https://semver.org/) 定义的格式：`MAJOR.MINOR.PATCH`。

- **Major (主要版本)**：需要更改代码的重大 API 更新。
- **Minor (次要版本)**：保持向后兼容的新功能和改进。
- **Patch (补丁版本)**：错误修复和小的改进。

例如：

- `1.0.0`：首个具有生产就绪 API 的稳定版本
- `1.1.0`：以向后兼容的方式添加了新功能
- `1.0.1`：向后兼容的错误修复

## API 稳定性

我们通过如下方式传达 API 的稳定性：

### 稳定 API

所有没有特殊前缀的 API 都被视为稳定且可用于生产。我们保持稳定功能的向后兼容性，仅在主要版本中引入重大更改。

### Beta API

标记为 `beta` 的 API 功能已完整，但可能会根据用户反馈进行小的更改。它们可安全用于生产，但在未来的版本中可能需要进行小的调整。

### Alpha API

标记为 `alpha` 的 API 是实验性的，可能会发生重大变化。在生产环境中使用这些 API 时请务必谨慎。

### 已弃用 API

标记为 `deprecated` 的 API 将在未来的主要版本中移除。如果可能，我们会指定计划移除的版本。要处理弃用：

1. 切换到推荐的替代 API
2. 遵循迁移指南（随主要版本发布）
3. 使用自动迁移工具（如果可用）

### 内部 API

某些 API 通过以下几种方式明确标记为“内部”：

- 一些文档提到了内部结构并将其称为内部结构。如果文档说明某物是内部的，它可能会发生变化。
- 以前导下划线 (**`_`**) 为前缀的函数、方法和其他对象。这是 Python 中表示私有的标准约定；如果任何方法以单个 **`_`** 开头，则它是内部 API。
    - **例外**：某些方法以 `_` 为前缀，但不包含实现。这些方法 *旨在* 由提供实现的子类覆盖。此类方法通常属于 LangChain **公共 API** 的一部分。

## 发布周期

<AccordionGroup>
    <Accordion title="主要版本">
        主要版本（例如 `1.0.0` → `2.0.0`）可能包括：

        - 重大的 API 更改
        - 移除已弃用的功能
        - 重大的架构改进

        我们提供：

        - 详细的迁移指南
        - 可行时的自动迁移工具
        - 上一主要版本的延长支持期
    </Accordion>

    <Accordion title="次要版本">
        次要版本（例如 `1.0.0` → `1.1.0`）包括：

        - 新功能和能力
        - 性能改进
        - 新的可选参数
        - 向后兼容的增强
    </Accordion>

    <Accordion title="补丁版本">
        补丁版本（例如 `1.0.0` → `1.0.1`）包括：

        - 错误修复
        - 安全更新
        - 文档改进
        - 没有 API 更改的性能优化
    </Accordion>
</AccordionGroup>

## 版本支持策略

- **最新主要版本**：全面支持并积极开发（ACTIVE 状态）
- **上一主要版本**：在下一个主要版本发布后 12 个月内提供安全更新和关键错误修复（MAINTENANCE 状态）
- **更旧的版本**：仅社区支持

### 长期支持 (LTS) 版本

LangChain 和 LangGraph 1.0 均被指定为 LTS 版本：

- 版本 1.0 将保持 ACTIVE (活跃) 状态，直到版本 2.0 发布
- 版本 2.0 发布后，版本 1.0 将进入 MAINTENANCE (维护) 模式至少 1 年
- LTS 版本遵循语义化版本控制 (semver)，允许在次要版本之间安全升级
- 旧版本（LangChain 0.3 和 LangGraph 0.4）处于 MAINTENANCE (维护) 模式，直到 2026 年 12 月

## 检查您的版本

要检查您安装的版本：



<CodeGroup>

```javascript LangChain
import { version } from "langchain/package.json";
console.log(version);
```

```javascript LangGraph
import { version } from "@langchain/langgraph/package.json";
console.log(version);
```
</CodeGroup>


## 升级



<CodeGroup>

```bash LangChain
# 升级到最新版本
npm update langchain @langchain/core

# 安装特定版本
npm install langchain@1.0.0 @langchain/core@1.0.0
```

```bash LangGraph
# 升级到最新版本
npm update @langchain/langgraph

# 安装特定版本
npm install @langchain/langgraph@1.0.0
```
</CodeGroup>


## 预发布版本

我们偶尔会发布 alpha 和 beta 版本以进行早期测试：

- **Alpha** (例如 `1.0.0a1`)：早期预览，预计有重大变化
- **Beta** (例如 `1.0.0b1`)：功能完整，可能有小的变化
- **Release Candidate (候选版本)** (例如 `1.0.0rc1`)：稳定发布前的最终测试
