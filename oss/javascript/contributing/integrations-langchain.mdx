**集成是 LangChain 的核心组件。**

LangChain 为多种不同组件（语言模型、向量存储等）提供了标准接口，这些组件在构建 LLM 应用时至关重要。实现新的集成有助于扩展 LangChain 的生态系统，并使您的服务能被数百万开发者发现。

<Warning>
    新集成**不接受**作为 PR 提交到任何 `langchain-ai` 仓库。所有新集成必须作为独立包发布到 PyPI（例如 `langchain-yourprovider`）。您应该向 `langchain-ai` 仓库提交的唯一 PR 是为您已发布的包添加文档。
</Warning>

## 为什么实现 LangChain 集成？

<Card title="可发现性" icon="search">
    LangChain 是构建 LLM 应用最常用的框架，每月下载量超过 2 亿次。
</Card>

<Card title="互操作性" icon="refresh">
    LangChain 组件暴露了标准接口，允许开发者轻松地相互替换它们。如果您实现了 LangChain 集成，任何使用不同组件的开发者都能轻松地替换为您的组件。
</Card>

<Card title="最佳实践" icon="star">
    通过其标准接口，LangChain 组件鼓励并促进最佳实践（流式传输、异步等），从而改善开发者体验和应用性能。
</Card>

## 要集成的组件

虽然任何组件都可以集成到 LangChain 中，但我们更鼓励以下特定类型的集成：

**集成这些 ✅**：

- [**聊天模型**](/oss/javascript/integrations/chat)：最活跃使用的组件类型
- [**工具/工具包**](/oss/javascript/integrations/tools)：启用代理功能
- [**检索器**](/oss/javascript/integrations/retrievers)：RAG 应用的核心
- [**嵌入模型**](/oss/javascript/integrations/embeddings)：向量操作的基础
- [**向量存储**](/oss/javascript/integrations/vectorstores)：语义搜索的必需品
- [**中间件**](/oss/javascript/integrations/middleware)：通过钩子扩展代理行为
- [**沙盒**](/oss/javascript/deepagents/sandboxes)：使用 Deep Agents 安全运行代码

<Accordion title="第三方沙盒集成附加标准">
请注意，我们仅在以下情况下展示第三方沙盒集成：

- 该集成由提供沙盒的公司编写和维护。
- **或** 该集成被广泛使用，意味着该集成在 PyPI 或 npm 上必须有每日至少 10,000 次下载才能被考虑展示。
</Accordion>

**不集成这些 ❌**：

- **LLM（文本补全模型）**：已弃用，推荐使用[聊天模型](/oss/javascript/integrations/chat)
- [**文档加载器**](/oss/javascript/integrations/document_loaders)：维护负担高
- [**键值存储**](/oss/javascript/integrations/stores)：使用有限
- **文档转换器**：利基用例
- **模型缓存**：基础设施问题
- **图**：复杂抽象
- **消息历史**：存储抽象
- **回调**：系统级组件
- **聊天加载器**：需求有限
- **适配器**：边缘情况工具

## 如何贡献集成

<Steps>
    <Step title="实现您的包">
        <Card title="如何实现 LangChain 集成" icon="link" href="/oss/javascript/contributing/implement-langchain" arrow />
    </Step>

    <Step title="通过标准测试">
        如果适用，为您的集成实现 LangChain 的[标准测试](/oss/javascript/contributing/standard-tests-langchain)套件支持，并成功运行它们。
    </Step>

    <Step title="发布集成">
        <Card title="如何发布集成" icon="upload" href="/oss/javascript/contributing/publish-langchain" arrow />
    </Step>

    <Step title="添加文档">
        提交 PR 以将您的集成文档添加到官方 LangChain 文档中。

        <Accordion title="集成文档指南" icon="book">
            集成只有在其文档有用时才有价值。为确保用户获得一致的体验，所有新集成都需要文档。我们为每种集成类型提供了一个标准起点模板，供您复制和修改。

            在向 LangChain [文档仓库](https://github.com/langchain-ai/docs) 提交新 PR 时，在 `src/oss/python/integrations/<component_type>/integration_name.mdx` 下的相关目录中创建一个新文件，使用相应的模板文件：

            - [聊天模型](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx)
            - [工具和工具包](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx)
            - [中间件](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/middleware/TEMPLATE.mdx)
            - 检索器 - 即将推出
            - 文本分割器 - 即将推出
            - 嵌入模型 - 即将推出
            - [向量存储](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx)
            - 文档加载器 - 即将推出
            - 键值存储 - 即将推出
        </Accordion>
    </Step>

    <Step title="联合营销" icon="speakerphone">
        （可选）与 LangChain 团队联系进行联合[营销](/oss/javascript/contributing/comarketing)。
    </Step>
</Steps>
