`SpiceDBRetriever` 是一个 LangChain `BaseRetriever`，它使用 SpiceDB 授权包装任何现有检索器。它遵循后过滤授权模式：首先基于语义搜索检索文档，然后根据用户权限进行过滤。

## 安装

```bash
pip install langchain-spicedb
```

## 设置

<Tip>
    此检索器需要运行中的 SpiceDB 实例。请参阅 [SpiceDB 提供程序页面](/oss/python/integrations/providers/spicedb) 获取设置说明。
</Tip>

### 环境设置

```python
import os

# SpiceDB 连接详情
os.environ["SPICEDB_ENDPOINT"] = "localhost:50051"
os.environ["SPICEDB_TOKEN"] = "sometoken"
```

## 初始化

```python
from langchain_spicedb import SpiceDBRetriever
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

# 创建基础检索器（任何向量存储都适用）
vectorstore = FAISS.from_documents(documents, OpenAIEmbeddings())
base_retriever = vectorstore.as_retriever()

# 使用 SpiceDB 授权包装
auth_retriever = SpiceDBRetriever(
    base_retriever=base_retriever,
    subject_id="alice",
    spicedb_endpoint="localhost:50051",
    spicedb_token="sometoken",
    resource_type="article",
    subject_type="user",
    permission="view",
    resource_id_key="article_id",
)
```

### 参数

- **base_retriever** (BaseRetriever): 要包装授权的基础检索器（必需）
- **subject_id** (str): 要检查权限的用户 ID（必需）
- **spicedb_endpoint** (str): SpiceDB 服务器地址（默认："localhost:50051"）
- **spicedb_token** (str): SpiceDB 认证的预共享密钥（默认："sometoken"）
- **resource_type** (str): SpiceDB 资源类型，例如 "document"、"article"（默认："document"）
- **subject_type** (str): SpiceDB 主体类型，例如 "user"（默认："user"）
- **permission** (str): 要检查的权限，例如 "view"、"edit"（默认："view"）
- **resource_id_key** (str): 文档元数据中包含资源 ID 的键（默认："resource_id"）
- **fail_open** (bool): 如果为 True，则在错误时允许访问；如果为 False，则在错误时拒绝访问（默认：False）
- **use_tls** (bool): 是否对 SpiceDB 连接使用 TLS（默认：False）

<Warning>
  **所有参数对于 SpiceDB 做出访问决策都是必需的。** 虽然有些有默认值，但您应显式设置它们以匹配您的 SpiceDB 模式。
</Warning>

## 用法

### 基本 RAG 流程

```python
from langchain_spicedb import SpiceDBRetriever
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 设置向量存储
documents = [...]  # 您的带元数据的文档
vectorstore = FAISS.from_documents(documents, OpenAIEmbeddings())
base_retriever = vectorstore.as_retriever()

# 使用授权包装
auth_retriever = SpiceDBRetriever(
    base_retriever=base_retriever,
    subject_id="alice",
    spicedb_endpoint="localhost:50051",
    spicedb_token="sometoken",
    resource_type="article",
    subject_type="user",
    permission="view",
    resource_id_key="article_id",
)

# 构建 RAG 链
prompt = ChatPromptTemplate.from_messages([
    ("system", "仅基于提供的上下文回答。"),
    ("human", "问题：{question}\n\n上下文：\n{context}")
])

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

chain = (
    {"context": auth_retriever | format_docs, "question": lambda x: x}
    | prompt
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

# 使用授权查询
answer = await chain.ainvoke("什么是 SpiceDB？")
print(answer)
```

## 文档元数据要求

文档**必须**在其元数据中包含资源 ID：

```python
from langchain_core.documents import Document

# 正确：带资源 ID 的文档
doc = Document(
    page_content="SpiceDB 是一个开源授权系统...",
    metadata={
        "article_id": "doc123",  # 必须与 resource_id_key 参数匹配
        "title": "SpiceDB 简介",
        "author": "AuthZed",
    }
)

# 检索器将根据用户是否在 SpiceDB 中拥有查看 article:doc123 的权限来过滤此文档
```

<Warning>
  如果文档元数据中缺少资源 ID，它将被过滤掉（视为未授权）。
</Warning>

## 授权流程

检索器遵循以下流程：

1. **语义搜索**：基础检索器执行语义搜索并返回前 K 个文档
2. **提取资源 ID**：从文档元数据中提取资源 ID
3. **批量权限检查**：在单个 SpiceDB API 调用中检查所有权限
4. **过滤**：仅返回用户有权查看的文档
5. **指标**：跟踪授权率、延迟和被拒绝的资源

```mermaid
graph LR
    A[用户查询] --> B[基础检索器]
    B --> C[前 K 个文档]
    C --> D[提取资源 ID]
    D --> E[SpiceDB 批量检查]
    E --> F[按权限过滤]
    F --> G[已授权文档]
```

## 性能

检索器使用 SpiceDB 的原生 `CheckBulkPermissionsRequest` API 以获得最佳性能：

- **单次 API 调用**：所有权限在一次请求中检查，而不是 N 次单独调用
- **高效**：显著快于单独的权限检查
- **可扩展**：高效处理数百个文档

### 性能示例

```python
# 检索 100 个文档
base_docs = await base_retriever.ainvoke("query")
print(f"检索到：{len(base_docs)} 个文档")  # 100

# 使用 SpiceDB 过滤（单次 API 调用）
auth_docs = await auth_retriever.ainvoke("query")
print(f"已授权：{len(auth_docs)} 个文档")  # 例如，25

# 所有 100 次权限检查在约 50 毫秒内完成（单次批量请求）
# 而 100 次单独请求需要约 5000 毫秒
```

## 错误处理

### 失败关闭（默认）

默认情况下，检索器失败关闭——如果检查权限时出错，文档将被过滤掉：

```python
auth_retriever = SpiceDBRetriever(
    base_retriever=base_retriever,
    subject_id="alice",
    spicedb_endpoint="localhost:50051",
    spicedb_token="sometoken",
    resource_type="article",
    resource_id_key="article_id",
    permission="view",
    fail_open=False,  # 默认 - 出错时拒绝
)
```

### 失败开放

用于开发或特定用例：

```python
auth_retriever = SpiceDBRetriever(
    base_retriever=base_retriever,
    subject_id="alice",
    spicedb_endpoint="localhost:50051",
    spicedb_token="sometoken",
    resource_type="article",
    resource_id_key="article_id",
    permission="view",
    fail_open=True,  # 出错时允许访问
)
```

## 完整示例：多用户 RAG

```python
import os
from langchain_spicedb import SpiceDBRetriever
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.documents import Document

# 设置
os.environ["OPENAI_API_KEY"] = "your-api-key"

# 创建带元数据的文档
documents = [
    Document(
        page_content="SpiceDB 是一个开源授权系统。",
        metadata={"article_id": "doc1", "title": "SpiceDB 简介"}
    ),
    Document(
        page_content="LangChain 是一个用于 LLM 应用的框架。",
        metadata={"article_id": "doc2", "title": "LangChain 简介"}
    ),
    Document(
        page_content="授权对于 RAG 系统至关重要。",
        metadata={"article_id": "doc3", "title": "RAG 安全"}
    ),
]

# 创建向量存储
vectorstore = FAISS.from_documents(documents, OpenAIEmbeddings())
base_retriever = vectorstore.as_retriever()

# 用于每个请求创建检索器的工厂函数
def create_user_rag_chain(subject_id: str):
    """为特定用户创建 RAG 链（在生产中按请求调用）。"""
    auth_retriever = SpiceDBRetriever(
        base_retriever=base_retriever,
        subject_id=subject_id,
        spicedb_endpoint="localhost:50051",
        spicedb_token="sometoken",
        resource_type="article",
        resource_id_key="article_id",
        permission="view",
    )

    prompt = ChatPromptTemplate.from_messages([
        ("system", "仅基于提供的上下文回答。"),
        ("human", "问题：{question}\n\n上下文：\n{context}")
    ])

    def format_docs(docs):
        if not docs:
            return "未找到已授权的文档。"
        return "\n\n".join(doc.page_content for doc in docs)

    return (
        {"context": auth_retriever | format_docs, "question": lambda x: x}
        | prompt
        | ChatOpenAI(model="gpt-4o-mini")
        | StrOutputParser()
    )

# 查询不同用户（每个请求创建自己的链）
question = "什么是 SpiceDB？"

alice_answer = await create_user_rag_chain("alice").ainvoke(question)
print(f"Alice 的答案：{alice_answer}")

bob_answer = await create_user_rag_chain("bob").ainvoke(question)
print(f"Bob 的答案：{bob_answer}")

# 在生产 Web 应用中：
# @app.post("/query")
# async def query(question: str, user_id: str):
#     chain = create_user_rag_chain(user_id)
#     return await chain.ainvoke(question)

# 不同用户看到不同的文档并获得不同的答案
```

## API 参考

### SpiceDBRetriever

**继承自**: `BaseRetriever`

**方法**：
- `invoke(query: str) -> List[Document]`：同步检索已授权文档
- `ainvoke(query: str) -> List[Document]`：异步检索已授权文档
- `with_config(subject_id: str, **kwargs) -> SpiceDBRetriever`：使用更新的配置创建新检索器

**属性**：
- `base_retriever`：包装的检索器
- `subject_id`：当前用户 ID
- `spicedb_endpoint`：SpiceDB 服务器地址
- `resource_type`：权限的资源类型
- `permission`：正在检查的权限
