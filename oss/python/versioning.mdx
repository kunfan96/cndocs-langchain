title: 版本管理
---

我们的开源软件版本号遵循以下格式：`MAJOR.MINOR.PATCH`，由 [Semantic Versioning](https://semver.org/) 定义。

- **Major（主版本）**：需要修改代码的破坏性 API 更新。
- **Minor（次版本）**：保持向后兼容的新功能和改进。
- **Patch（补丁版本）**：错误修复和小改进。

例如：

- `1.0.0`：首个稳定版本，具有生产就绪的 API
- `1.1.0`：以向后兼容的方式添加新功能
- `1.0.1`：向后兼容的错误修复

## API 稳定性

我们通过以下方式传达 API 的稳定性：

### 稳定 API

所有没有特殊前缀的 API 都被视为稳定的，可用于生产环境。我们为稳定功能保持向后兼容性，仅在主版本更新时引入破坏性变更。

### Beta API

标记为 `beta` 的 API 功能完整，但可能会根据用户反馈进行小幅更改。它们可以安全地用于生产环境，但在未来版本中可能需要小幅调整。

### Alpha API

标记为 `alpha` 的 API 是实验性的，可能会发生重大变更。在生产环境中使用时请谨慎。

### 已弃用 API

标记为 `deprecated` 的 API 将在未来的主版本中被移除。在可能的情况下，我们会指定计划移除的版本。处理弃用的方法：

1. 切换到推荐的替代 API
2. 遵循迁移指南（随主版本一起发布）
3. 在可用时使用自动迁移工具

### 内部 API

某些 API 通过以下几种方式明确标记为"内部"：

- 某些文档会提及内部 API 并标明其性质。如果文档说某些内容是内部的，它可能会更改。
- 以下划线 (**`_`**) 为前缀的函数、方法和其他对象。这是 Python 表示某些内容为私有的标准约定；如果任何方法以单个 **`_`** 开头，它就是内部 API。
    - **例外：** 某些方法以 `_` 为前缀，但不包含实现。这些方法*旨在*被提供实现的子类覆盖。此类方法通常是 LangChain **公共 API** 的一部分。

## 发布周期

<AccordionGroup>
    <Accordion title="主版本发布">
        主版本发布（例如 `1.0.0` → `2.0.0`）可能包括：

        - 破坏性 API 变更
        - 移除已弃用的功能
        - 重大架构改进

        我们提供：

        - 详细的迁移指南
        - 在可能的情况下提供自动迁移工具
        - 为上一个主版本提供延长支持期
    </Accordion>

    <Accordion title="次版本发布">
        次版本发布（例如 `1.0.0` → `1.1.0`）包括：

        - 新功能和能力
        - 性能改进
        - 新的可选参数
        - 向后兼容的增强功能
    </Accordion>

    <Accordion title="补丁版本发布">
        补丁版本发布（例如 `1.0.0` → `1.0.1`）包括：

        - 错误修复
        - 安全更新
        - 文档改进
        - 不涉及 API 变更的性能优化
    </Accordion>
</AccordionGroup>

## 版本支持政策

- **最新主版本**：完全支持，活跃开发中（ACTIVE 状态）
- **上一个主版本**：在下一个主版本发布后 12 个月内提供安全更新和关键错误修复（MAINTENANCE 状态）
- **更早版本**：仅社区支持

### 长期支持 (LTS) 版本

LangChain 和 LangGraph 1.0 都被指定为 LTS 版本：

- 1.0 版本将保持 ACTIVE 状态，直到 2.0 版本发布
- 2.0 版本发布后，1.0 版本将进入 MAINTENANCE 模式至少 1 年
- LTS 版本遵循语义化版本（semver），允许在次版本之间安全升级
- 旧版本（LangChain 0.3 和 LangGraph 0.4）处于 MAINTENANCE 模式，直到 2026 年 12 月

有关发布状态和支持时间表的详细信息，请参阅[发布政策](/oss/python/release-policy)。

## 检查您的版本

检查已安装的版本：

<CodeGroup>

```python LangChain
import langchain_core
print(langchain_core.__version__)
```

```python LangGraph
import langgraph
print(langgraph.__version__)
```

</CodeGroup>




## 升级

<CodeGroup>

```bash LangChain
# Upgrade to the latest version
pip install -U langchain-core langchain

# Upgrade to a specific version
pip install langchain-core==1.0.0
```

```bash LangGraph
# Upgrade to the latest version
pip install -U langgraph

# Upgrade to a specific version
pip install langgraph==1.0.0
```
</CodeGroup>




## 预发布版本

我们偶尔会发布 alpha 和 beta 版本供早期测试：

- **Alpha**（例如 `1.0.0a1`）：早期预览，预计会有重大变更
- **Beta**（例如 `1.0.0b1`）：功能完整，可能有小幅变更
- **Release Candidate**（例如 `1.0.0rc1`）：稳定版发布前的最终测试
