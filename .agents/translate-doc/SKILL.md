# 文档翻译技能（Translate Document Skill）

## 使用说明（Instructions）

当用户希望将**大型英文文档**翻译成其他语言（如中文），或将英文文档润色为更高质量的英文时，调用此技能。

调用本技能时，请遵循以下规则：

1. **分段处理（Chunking）**
   - 自动将长文本切分为多个 chunk（建议 1500–2500 字符）。
   - 避免在 Markdown 结构、代码块、标题中间切断。
   - 保持原始顺序。

2. **翻译要求（Translation Requirements）**
   - 必须逐句翻译，不得总结、删减或跳过内容。
   - 必须保留所有格式：Markdown、列表、标题、表格、代码块等。
   - 在所有 chunk 中保持术语一致性。
   - 若提供术语表（glossary），必须严格遵循。
   - 根据用户要求应用翻译风格：`直译 literal`、`自然 natural`、`学术 academic`、`技术 technical`、`英文润色 polished English`。

3. **术语表处理（Glossary Handling）**
   - 若用户提供术语表，必须严格使用。
   - 若未提供术语表，需自动推断并保持术语一致性。

4. **输出要求（Output）**
   - 输出的结构必须与原文完全一致。
   - 不得添加解释、注释或额外内容。
   - 不得合并或拆分段落，除非原文如此。

5. **大型文档工作流（Large Document Workflow）**
   - 第一步：预处理并切分文档。
   - 第二步：逐段翻译，每段使用相同规则。
   - 第三步：合并所有翻译后的 chunk。
   - 第四步：进行一致性检查，统一术语与风格。

6. **错误处理（Error Handling）**
   - 若文本为空，要求用户提供内容。
   - 若缺少目标语言，要求用户指定。

## 指南（Guidelines）

- 保留专业术语和技术细节，例如"langchain"和"Python"。
- 不得总结或省略内容。
- 不得改变文档结构。
- 必须保留 Markdown 格式。
- 必须在所有 chunk 中保持术语一致性。
- 若提供术语表，必须优先使用术语表。
- 对于超长文档，可多次处理。
- 除非用户要求，否则不得进行创造性改写。
