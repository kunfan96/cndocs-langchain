>[Optimum Intel](https://github.com/huggingface/optimum-intel?tab=readme-ov-file#optimum-intel) 是 🤗 Transformers 和 Diffusers 库与 Intel 提供的各种工具和库之间的接口，旨在加速在 Intel 架构上的端到端流水线。

>[Intel® Extension for Transformers](https://github.com/intel/intel-extension-for-transformers?tab=readme-ov-file#intel-extension-for-transformers)（ITREX）是一个创新工具包，旨在通过在包括 Intel Gaudi2、Intel CPU 和 Intel GPU 在内的各种 Intel 平台上以最佳性能运行基于 Transformer 的模型，从而全方位加速 GenAI/LLM。

本页介绍如何在 LangChain 中使用 optimum-intel 和 ITREX。

## Optimum-intel

所有与 [optimum-intel](https://github.com/huggingface/optimum-intel.git) 和 [IPEX](https://github.com/intel/intel-extension-for-pytorch) 相关的功能。

### 安装

使用以下命令安装 optimum-intel 和 ipex：

<CodeGroup>
```bash pip
pip install "optimum[neural-compressor]"
pip install intel_extension_for_pytorch
```

```bash uv
uv add "optimum[neural-compressor]"
uv add intel_extension_for_pytorch
```
</CodeGroup>

请按照以下指定的安装说明进行操作：

* 按照 [optimum-intel 安装指南](https://github.com/huggingface/optimum-intel) 中的说明安装 optimum-intel。
* 按照 [IPEX 安装指南](https://intel.github.io/intel-extension-for-pytorch/index.html#installation?platform=cpu&version=v2.2.0%2Bcpu) 中的说明安装 IPEX。

## Intel® extension for transformers (ITREX)
（ITREX）是一个创新工具包，用于加速 Intel 平台上的基于 Transformer 的模型，特别是在第 4 代 Intel Xeon Scalable 处理器 Sapphire Rapids（代号 Sapphire Rapids）上效果显著。

量化是一种通过用更少的位数来表示这些权重来降低其精度的过程。仅权重量化专门专注于对神经网络的权重进行量化，同时保持其他组件（如激活值）的原始精度。

随着大语言模型（LLMs）日益普及，对于能够兼顾现代架构计算需求并保持准确性的新型改进量化方法的需求不断增长。与 [常规量化](https://github.com/intel/intel-extension-for-transformers/blob/main/docs/quantization.md)（如 W8A8）相比，仅权重量化可能是平衡性能与准确性的更好折衷方案，因为下文将指出，部署 LLM 的瓶颈在于内存带宽，而通常仅权重量化能带来更好的准确性。

在此，我们将介绍如何使用 ITREX 为 Transformers 大语言模型引入嵌入模型和仅权重量化。仅权重量化是深度学习中用于降低神经网络内存和计算需求的一种技术。在深度神经网络的背景下，模型参数（也称为权重）通常使用浮点数表示，这会消耗大量内存并需要密集的计算资源。

所有与 [intel-extension-for-transformers](https://github.com/intel/intel-extension-for-transformers) 相关的功能。

### 安装

安装 intel-extension-for-transformers。有关系统要求和其他安装提示，请参阅 [安装指南](https://github.com/intel/intel-extension-for-transformers/blob/main/docs/installation.md)

<CodeGroup>
```bash pip
pip install intel-extension-for-transformers
```

```bash uv
uv add intel-extension-for-transformers
```
</CodeGroup>

安装其他必需的依赖包。

<CodeGroup>
```bash pip
pip install -U torch onnx accelerate datasets
```

```bash uv
uv add torch onnx accelerate datasets
```
</CodeGroup>

## 配置参数详解

以下是 `WeightOnlyQuantConfig` 类的详细说明。

#### weight_dtype (string)：权重数据类型，默认为 "nf4"。
我们支持将权重量化为以下存储数据类型（WeightOnlyQuantConfig 中的 weight_dtype）：
* **int8**：使用 8 位数据类型。
* **int4_fullrange**：与常规 int4 范围 [-7,7] 相比，使用 int4 范围的 -8 值。
* **int4_clip**：裁剪并将值保留在 int4 范围内，其余设为零。
* **nf4**：使用归一化浮点 4 位数据类型。
* **fp4_e2m1**：使用常规浮点 4 位数据类型。“e2”表示指数部分使用 2 位，“m1”表示尾数部分使用 1 位。

#### compute_dtype (string)：计算数据类型，默认为 "fp32"。
尽管这些技术将权重存储为 4 位或 8 位，但计算仍在 float32、bfloat16 或 int8 中进行（WeightOnlyQuantConfig 中的 compute_dtype）：
* **fp32**：使用 float32 数据类型进行计算。
* **bf16**：使用 bfloat16 数据类型进行计算。
* **int8**：使用 8 位数据类型进行计算。

#### llm_int8_skip_modules (list of module's name)：跳过量化的模块列表，默认为无。
这是一个需要跳过量化的模块列表。

#### scale_dtype (string)：缩放因子数据类型，默认为 "fp32"。
目前仅支持 "fp32"（float32）。

#### mse_range (boolean)：是否从范围 [0.805, 1.0, 0.005] 中搜索最佳裁剪范围，默认为 false。
#### use_double_quant (boolean)：是否对缩放因子进行量化，默认为 false。
暂不支持。
#### double_quant_dtype (string)：为双重量化预留。
#### double_quant_scale_dtype (string)：为双重量化预留。
#### group_size (int)：量化时的分组大小。
#### scheme (string)：权重量化到的格式。默认为 "sym"。
* **sym**：对称。
* **asym**：非对称。
#### algorithm (string)：用于提高准确性的算法。默认为 "RTN"
* **RTN**：四舍五入到最近值（RTN）是一种我们可以非常直观理解的量化方法。
* **AWQ**：仅保护 1% 的关键权重即可大幅降低量化误差。关键权重通道是通过观察每个通道的激活值和权重分布来选择的。关键权重在量化前也会乘以一个较大的缩放因子以进行保护。
* **TEQ**：一种可训练的等效变换，可在仅权重量化中保留 FP32 精度。
