> PebbloRetrievalQA 是一个检索链，具有身份和语义强制功能，用于针对向量数据库进行问答。

本笔记本介绍如何使用身份和语义强制（拒绝主题/实体）来检索文档。有关 Pebblo 及其 SafeRetriever 功能的更多详细信息，请访问 [Pebblo 文档](https://daxa-ai.github.io/pebblo/retrieval_chain/)。

### 步骤

1. **加载文档：**
   我们将加载带有授权和语义元数据的文档到内存中的 Qdrant 向量存储中。此向量存储将用作 PebbloRetrievalQA 中的检索器。

   > **注意：** 建议在摄取端使用 [PebbloSafeLoader](https://daxa-ai.github.io/pebblo/rag) 作为加载带有身份验证和语义元数据文档的对应工具。`PebbloSafeLoader` 保证文档的安全高效加载，同时维护元数据的完整性。

2. **测试强制机制：**
   我们将分别测试身份和语义强制。对于每个用例，我们将定义一个特定的 "ask" 函数，其中包含所需的上下文（*auth_context* 和 *semantic_context*），然后提出问题。

## 设置

### 依赖项

在此演练中，我们将使用 OpenAI LLM、OpenAI 嵌入和 Qdrant 向量存储。

```python
pip install -qU langchain langchain_core langchain-community langchain-openai qdrant_client
```

### 身份感知的数据摄取

这里我们使用 Qdrant 作为向量数据库；但是，您可以使用任何受支持的向量数据库。

**PebbloRetrievalQA 链支持以下向量数据库：**

- Qdrant
- Pinecone
- Postgres（利用 pgvector 扩展）

**加载带有授权和语义信息的向量数据库：**

在此步骤中，我们将源文档的授权和语义信息捕获到向量数据库条目中每个块的元数据内的 `authorized_identities`、`pebblo_semantic_topics` 和 `pebblo_semantic_entities` 字段中。

*注意：要使用 PebbloRetrievalQA 链，必须始终将授权和语义元数据放在指定字段中。这些字段必须包含字符串列表。*

```python
from langchain_community.vectorstores.qdrant import Qdrant
from langchain_core.documents import Document
from langchain_openai.embeddings import OpenAIEmbeddings
from langchain_openai.llms import OpenAI

llm = OpenAI()
embeddings = OpenAIEmbeddings()
collection_name = "pebblo-identity-and-semantic-rag"

page_content = """
**ACME Corp 财务报告**

**概述：**
ACME Corp 是并购行业的领先企业，呈现其截至 2020 年 12 月 31 日的财年财务报告。
尽管经济环境充满挑战，ACME Corp 展现了稳健的业绩和战略增长。

**财务亮点：**
收入飙升至 5000 万美元，较上年增长 15%，这得益于成功的交易完成和向新市场的扩张。
净利润达到 1200 万美元，利润率高达 24%。

**关键指标：**
总资产激增至 8000 万美元，增长 20%，凸显了 ACME Corp 强劲的财务状况和资产基础。
此外，公司保持了 0.5 的保守债务权益比率，确保了可持续的财务稳定性。

**未来展望：**
ACME Corp 对未来保持乐观，计划利用全球并购领域的新兴机会。
公司致力于为股东创造价值，同时保持道德的商业实践。

**银行账户详情：**
如需咨询或交易，请参考 ACME Corp 的美国银行账户：
账户号码：123456789012
银行名称：虚构美国银行
"""

documents = [
    Document(
        **{
            "page_content": page_content,
            "metadata": {
                "pebblo_semantic_topics": ["financial-report"],
                "pebblo_semantic_entities": ["us-bank-account-number"],
                "authorized_identities": ["finance-team", "exec-leadership"],
                "page": 0,
                "source": "https://drive.google.com/file/d/xxxxxxxxxxxxx/view",
                "title": "ACME Corp Financial Report.pdf",
            },
        }
    )
]

vectordb = Qdrant.from_documents(
    documents,
    embeddings,
    location=":memory:",
    collection_name=collection_name,
)

print("Vectordb loaded.")
```

```text
Vectordb loaded.
```

## 使用身份强制进行检索

PebbloRetrievalQA 链使用 SafeRetrieval 来强制执行上下文片段仅从用户授权的文档中检索。
为此，Gen-AI 应用程序需要为此检索链提供授权上下文。
此 *auth_context* 应填充访问 Gen-AI 应用程序的用户的身份和授权组。

以下是 `PebbloRetrievalQA` 的示例代码，其中包含来自访问 RAG 应用程序的用户的 `user_auth`（用户授权列表，可能包括其用户 ID 和所属组），传递到 `auth_context` 中。

```python
from langchain_community.chains import PebbloRetrievalQA
from langchain_community.chains.pebblo_retrieval.models import AuthContext, ChainInput

# 初始化 PebbloRetrievalQA 链
qa_chain = PebbloRetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectordb.as_retriever(),
    app_name="pebblo-identity-rag",
    description="使用 PebbloRetrievalQA 的身份强制应用程序",
    owner="ACME Corp",
)


def ask(question: str, auth_context: dict):
    """
    向 PebbloRetrievalQA 链提问
    """
    auth_context_obj = AuthContext(**auth_context) if auth_context else None
    chain_input_obj = ChainInput(query=question, auth_context=auth_context_obj)
    return qa_chain.invoke(chain_input_obj.dict())
```

### 1. 授权用户提问

我们摄取了授权身份 `["finance-team", "exec-leadership"]` 的数据，因此具有授权身份/组 `finance-team` 的用户应收到正确答案。

```python
auth = {
    "user_id": "finance-user@acme.org",
    "user_auth": [
        "finance-team",
    ],
}

question = "分享 ACME Corp 2020 年的财务表现"
resp = ask(question, auth)
print(f"Question: {question}\n\nAnswer: {resp['result']}")
```

```text
Question: 分享 ACME Corp 2020 年的财务表现

Answer:
收入：5000 万美元（较上年增长 15%）
净利润：1200 万美元（利润率 24%）
总资产：8000 万美元（增长 20%）
债务权益比率：0.5
```

### 2. 未授权用户提问

由于用户的授权身份/组 `eng-support` 未包含在授权身份 `["finance-team", "exec-leadership"]` 中，我们不应收到答案。

```python
auth = {
    "user_id": "eng-user@acme.org",
    "user_auth": [
        "eng-support",
    ],
}

question = "分享 ACME Corp 2020 年的财务表现"
resp = ask(question, auth)
print(f"Question: {question}\n\nAnswer: {resp['result']}")
```

```text
Question: 分享 ACME Corp 2020 年的财务表现

Answer:  I don't know.
```

### 3. 使用 PromptTemplate 提供额外指令

您可以使用 PromptTemplate 向 LLM 提供额外指令以生成自定义响应。

```python
from langchain_core.prompts import PromptTemplate

prompt_template = PromptTemplate.from_template(
    """
使用提供的上下文回答问题。
如果未提供上下文，请仅说“抱歉，该信息不可用，或访问受限。”。

问题：{question}
"""
)

question = "分享 ACME Corp 2020 年的财务表现"
prompt = prompt_template.format(question=question)
```

#### 3.1 授权用户提问

```python
auth = {
    "user_id": "finance-user@acme.org",
    "user_auth": [
        "finance-team",
    ],
}
resp = ask(prompt, auth)
print(f"Question: {question}\n\nAnswer: {resp['result']}")
```

## 使用语义强制进行检索

PebbloRetrievalQA 链使用 SafeRetrieval 来确保上下文片段仅从符合提供的语义上下文的文档中检索。
为此，Gen-AI 应用程序必须为此检索链提供语义上下文。
此 `semantic_context` 应包括应为访问 Gen-AI 应用程序的用户拒绝的主题和实体。

以下是 PebbloRetrievalQA 的示例代码，包含 `topics_to_deny` 和 `entities_to_deny`。这些被传递到链输入的 `semantic_context` 中。

```python
from typing import List, Optional

from langchain_community.chains import PebbloRetrievalQA
from langchain_community.chains.pebblo_retrieval.models import (
    ChainInput,
    SemanticContext,
)

# 初始化 PebbloRetrievalQA 链
qa_chain = PebbloRetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectordb.as_retriever(),
    app_name="pebblo-semantic-rag",
    description="使用 PebbloRetrievalQA 的语义强制应用程序",
    owner="ACME Corp",
)


def ask(
    question: str,
    topics_to_deny: Optional[List[str]] = None,
    entities_to_deny: Optional[List[str]] = None,
):
    """
    向 PebbloRetrievalQA 链提问
    """
    semantic_context = dict()
    if topics_to_deny:
        semantic_context["pebblo_semantic_topics"] = {"deny": topics_to_deny}
    if entities_to_deny:
        semantic_context["pebblo_semantic_entities"] = {"deny": entities_to_deny}

    semantic_context_obj = (
        SemanticContext(**semantic_context) if semantic_context else None
    )
    chain_input_obj = ChainInput(query=question, semantic_context=semantic_context_obj)
    return qa_chain.invoke(chain_input_obj.dict())
```

### 1. 无语义强制

由于未应用语义强制，系统应返回答案，而不因上下文关联的语义标签排除任何上下文。

```python
topic_to_deny = []
entities_to_deny = []
question = "分享 ACME Corp 2020 年的财务表现"
resp = ask(question, topics_to_deny=topic_to_deny, entities_to_deny=entities_to_deny)
print(
    f"Topics to deny: {topic_to_deny}\nEntities to deny: {entities_to_deny}\n"
    f"Question: {question}\nAnswer: {resp['result']}"
)
```

```text
Topics to deny: []
Entities to deny: []
Question: 分享 ACME Corp 2020 年的财务表现
Answer:
ACME Corp 2020 年收入增长 15% 至 5000 万美元，净利润 1200 万美元，资产基础强劲，达 8000 万美元。公司还保持了 0.5 的保守债务权益比率。
```

### 2. 拒绝 financial-report 主题

数据已摄取主题：`["financial-report"]`。
因此，拒绝 `financial-report` 主题的应用程序不应收到答案。

```python
topic_to_deny = ["financial-report"]
entities_to_deny = []
question = "分享 ACME Corp 2020 年的财务表现"
resp = ask(question, topics_to_deny=topic_to_deny, entities_to_deny=entities_to_deny)
print(
    f"Topics to deny: {topic_to_deny}\nEntities to deny: {entities_to_deny}\n"
    f"Question: {question}\nAnswer: {resp['result']}"
)
```

```text
Topics to deny: ['financial-report']
Entities to deny: []
Question: 分享 ACME Corp 2020 年的财务表现
Answer:

不幸的是，我无法访问 ACME Corp 2020 年的财务表现。
```

由于实体 `us-bank-account-number` 被拒绝，系统不应返回答案。

```python
topic_to_deny = []

)
```
