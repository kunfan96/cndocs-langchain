[聊天模型](/oss/javascript/langchain/models) 是使用一系列[消息](/oss/javascript/langchain/messages)作为输入并返回消息作为输出的语言模型 <Tooltip tip="不遵循聊天模型接口的旧模型，而是使用接受字符串输入并返回字符串输出的接口。这些模型通常在其名称中不包含 'Chat' 前缀或包含 'LLM' 后缀。">(与纯文本相对)</Tooltip>。

## 安装与使用

<Tip>
    请参阅[此部分获取安装 LangChain 包的一般说明](/oss/javascript/langchain/install)。
</Tip>

<AccordionGroup>
    <Accordion title="OpenAI">
        安装：

        <CodeGroup>
            ```bash npm
            npm i @langchain/openai
            ```
            ```bash yarn
            yarn add @langchain/openai
            ```
            ```bash pnpm
            pnpm add @langchain/openai
            ```
        </CodeGroup>

        添加环境变量：

        ```bash
        OPENAI_API_KEY=your-api-key
        ```

        实例化模型：

        ```typescript
        import { ChatOpenAI } from "@langchain/openai";

        const model = new ChatOpenAI({ model: "gpt-4.1-mini" });
        ```
        ```javascript
        await model.invoke("Hello, world!")
        ```
    </Accordion>

## 聊天补全 API

某些模型提供商提供与 OpenAI（旧版）[聊天补全 API](https://platform.openai.com/docs/guides/completions) 兼容的端点。在这种情况下，您可以使用 [`ChatOpenAI`](/oss/javascript/integrations/chat/openai) 并自定义 `base_url` 来连接到这些端点。请注意，基于聊天补全 API 构建的功能可能无法被 `ChatOpenAI` 完全支持；在这种情况下，如果可用，请考虑使用提供商特定的类。

<Accordion title="示例：OpenRouter">
    要使用 OpenRouter，您需要注册一个帐户并获取 [API 密钥](https://openrouter.ai/docs/api/reference/authentication)。

    ```typescript
    import { ChatOpenAI } from "@langchain/openai";

    const model = new ChatOpenAI({
        model: "...", // 指定 OpenRouter 上可用的模型
        configuration: {
        apiKey: "OPENROUTER_API_KEY",
        baseURL: "https://openrouter.ai/api/v1",
        }
    });
    ```

    请参阅 [OpenRouter 文档](https://openrouter.ai/docs/quickstart) 了解更多详情。
</Accordion>
