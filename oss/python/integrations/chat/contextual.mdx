## 配置

要访问 Contextual 模型，您需要创建一个 Contextual AI 账户、获取 API 密钥，并安装 `langchain-contextual` 集成包。

### 凭据

前往 [app.contextual.ai](https://app.contextual.ai) 注册 Contextual 并生成 API 密钥。完成此操作后，请设置 CONTEXTUAL_AI_API_KEY 环境变量：

```python
import getpass
import os

if not os.getenv("CONTEXTUAL_AI_API_KEY"):
    os.environ["CONTEXTUAL_AI_API_KEY"] = getpass.getpass(
        "请输入您的 Contextual API 密钥: "
    )
```

如果您希望在调用模型时自动进行跟踪，还可以通过取消注释下方代码设置 [LangSmith](/langsmith/home) API 密钥：

```python
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = getpass.getpass("请输入您的 LangSmith API 密钥: ")
```

### 安装

LangChain Contextual 集成位于 `langchain-contextual` 包中：

```python
pip install -qU langchain-contextual
```

## 实例化

现在我们可以实例化我们的模型对象并生成聊天完成。

聊天客户端可以通过以下附加设置进行初始化：

| 参数 | 类型 | 描述 | 默认值 |
|-----------|------|-------------|---------|
| temperature | 可选[float] | 采样温度，影响响应的随机性。注意较高的温度值可能会降低接地性。 | 0 |
| top_p | 可选[float] | 核心抽样的参数，它是温度的一种替代方案，也会影响响应的随机性。注意较高的 top_p 值可能会降低接地性。 | 0.9 |
| max_new_tokens | 可选[int] | 模型在响应中可以生成的最大标记数。最小值为1，最大值为2048。 | 1024 |

```python
from langchain_contextual import ChatContextual

llm = ChatContextual(
    model="v1",  # 默认为 `v1`
    api_key="",
    temperature=0,  # 默认为 0
    top_p=0.9,  # 默认为 0.9
    max_new_tokens=1024,  # 默认为 1024
)
```

## 调用

Contextual Grounded 语言模型在调用 `ChatContextual.invoke` 方法时接受附加的 `kwargs`.

这些附加输入包括：

| 参数 | 类型 | 描述 |
|-----------|------|-------------|
| knowledge | list[str] | 必填：生成响应时 Contextual 语言模型可以使用的知识源字符串列表。 |
| system_prompt | 可选[str] | 可选：模型在生成响应时应遵循的指令。请注意，我们不保证模型会严格按照这些指令行事。 |
| avoid_commentary | 可选[bool] | 可选（默认为 `False`）：标志以指示模型是否应在响应中避免提供额外评论。评论具有对话性质，并且包含不可验证的声明；因此，评论并不严格基于可用上下文。然而，评论可能会提供有用的信息，从而提高响应的实用性。 |

```python
# 包含系统提示（可选）
system_prompt = "您是一个使用所有提供的知识尽可能准确地回答用户查询的帮助助手。"

# 在数组中提供自己的知识库字符串
knowledge = [
    "世界上有两种狗：好狗和最好的狗。",
    "世界上有两种猫：好猫和最好的猫。",
]

# 创建您的消息
messages = [
    ("human", "世界上的猫有哪些类型？"),
]

# 通过提供知识字符串、可选系统提示调用 GLM，如果要关闭 GLM 的评论，请将 `avoid_commentary` 参数设置为 True
ai_msg = llm.invoke(
    messages, knowledge=knowledge, system_prompt=system_prompt, avoid_commentary=True
)

print(ai_msg.content)
```

## 链接

我们可以通过输出解析器链 Contextual 模型。

```python
from langchain_core.output_parsers import StrOutputParser

chain = llm | StrOutputParser

chain.invoke(
    messages, knowledge=knowledge, systemp_prompt=system_prompt, avoid_commentary=True
)
```
