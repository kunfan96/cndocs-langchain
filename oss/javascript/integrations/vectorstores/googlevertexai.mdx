<Tip>
**兼容性**

仅在 Node.js 上可用。
</Tip>

Google Vertex AI Matching Engine "提供了业界领先的高规模低延迟向量数据库。这些向量数据库通常被称为向量相似度匹配或近似最近邻 (ANN) 服务。"

## 设置

<Warning>
**此模块期望端点和已部署的索引已经创建，因为**

创建时间接近一小时。要了解更多信息，请参阅 LangChain Python 文档 [创建索引并将其部署到端点](https://python.langchain.com/docs/integrations/vectorstores/google_vertex_ai_vector_search/#deploy-index-to-the-endpoint)。
</Warning>

在运行此代码之前，您应确保在 Google Cloud 仪表板中为相关项目启用了 Vertex AI API，并且您已使用以下方法之一向 Google Cloud 进行身份验证：

- 您已登录到有权访问该项目的帐户（使用 `gcloud auth application-default login`）。
- 您正在使用有权访问该项目的服务帐户运行的机器上。
- 您已下载了有权访问该项目的服务帐户的凭据，并将 `GOOGLE_APPLICATION_CREDENTIALS` 环境变量设置为该文件的路径。

使用以下命令安装身份验证库：

```bash npm
npm install @langchain/community @langchain/core google-auth-library
```
Matching Engine 不存储实际文档内容，仅存储嵌入。因此，您需要一个文档存储。以下示例使用 Google Cloud Storage，这需要：

```bash npm
npm install @google-cloud/storage
```
## 用法

### 初始化引擎

创建 `MatchingEngine` 对象时，您需要一些关于匹配引擎配置的信息。您可以从 Matching Engine 的云控制台获取此信息：

- 索引的 ID
- 索引端点的 ID

您还需要一个文档存储。虽然 `InMemoryDocstore` 适用于初始测试，但您可能希望使用类似 [GoogleCloudStorageDocstore](https://reference.langchain.com/javascript/langchain-community/stores/doc/gcs/GoogleCloudStorageDocstore) 的东西来更持久地存储它。

```typescript
import { MatchingEngine } from "@langchain/community/vectorstores/googlevertexai";
import { Document } from "@langchain/classic/document";
import { SyntheticEmbeddings } from "@langchain/classic/embeddings/fake";
import { GoogleCloudStorageDocstore } from "@langchain/community/stores/doc/gcs";

const embeddings = new SyntheticEmbeddings({
  vectorSize: Number.parseInt(
    process.env.SYNTHETIC_EMBEDDINGS_VECTOR_SIZE ?? "768",
    10
  ),
});

const store = new GoogleCloudStorageDocstore({
  bucket: process.env.GOOGLE_CLOUD_STORAGE_BUCKET!,
});

const config = {
  index: process.env.GOOGLE_VERTEXAI_MATCHINGENGINE_INDEX!,
  indexEndpoint: process.env.GOOGLE_VERTEXAI_MATCHINGENGINE_INDEXENDPOINT!,
  apiVersion: "v1beta1",
  docstore: store,
};

const engine = new MatchingEngine(embeddings, config);
```
### 添加文档

```typescript
const doc = new Document({ pageContent: "this" });
await engine.addDocuments([doc]);
```
文档中的任何元数据都会转换为 Matching Engine 的 "allow list" 值，这些值可在查询期间用于过滤。

```typescript
const documents = [
  new Document({
    pageContent: "this apple",
    metadata: {
      color: "red",
      category: "edible",
    },
  }),
  new Document({
    pageContent: "this blueberry",
    metadata: {
      color: "blue",
      category: "edible",
    },
  }),
  new Document({
    pageContent: "this firetruck",
    metadata: {
      color: "red",
      category: "machine",
    },
  }),
];

// 添加所有文档
await engine.addDocuments(documents);
```
文档假定也具有可用的 "id" 参数。如果未设置，则将分配一个 ID 并作为 Document 的一部分返回。

### 查询文档

执行简单的 k-最近邻搜索，返回所有结果，可以使用任何标准方法：

```typescript
const results = await engine.similaritySearch("this");
```
### 使用过滤器/限制查询文档

我们可以根据为文档设置的元数据来限制返回的文档。如果我们只想将结果限制为红色的文档，可以这样做：

```typescript
import { Restriction } from `@langchain/community/vectorstores/googlevertexai`;

const redFilter: Restriction[] = [
  {
    namespace: "color",
    allowList: ["red"],
  },
];
const redResults = await engine.similaritySearch("this", 4, redFilter);
```
如果我们想做更复杂的事情，比如红色的但不可食用的东西：

```typescript
const filter: Restriction[] = [
  {
    namespace: "color",
    allowList: ["red"],
  },
  {
    namespace: "category",
    denyList: ["edible"],
  },
];
const results = await engine.similaritySearch("this", 4, filter);
```
### 删除文档

使用 ID 删除文档。

```typescript
import { IdDocument } from `@langchain/community/vectorstores/googlevertexai`;

const oldResults: IdDocument[] = await engine.similaritySearch("this", 10);
const oldIds = oldResults.map( doc => doc.id! );
await engine.delete({ids: oldIds});
```
