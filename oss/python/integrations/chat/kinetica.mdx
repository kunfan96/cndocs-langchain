[Kinetica](https://www.kinetica.com/) 是一个内置支持文本生成 SQL 功能的数据库。

此笔记本演示了如何使用 Kinetica 将自然语言转换为 SQL，简化数据检索过程。本示例旨在展示文本生成的工作流程，而不是 LLM 的能力。

## 概述

在 Kinetica 语言模型工作流中，您可以在数据库中创建一个提供用于推断所需信息的上下文，包括表、注释、规则和样本。调用 `ChatKinetica.load_messages_from_context()` 可以从数据库检索这些上下文信息，以便用于创建对话提示。

对话提示由一个 [`SystemMessage`](https://reference.langchain.com/python/langchain-core/messages/system/SystemMessage) 和一系列包含样本的 `HumanMessage`/`AIMessage` 对组成。样本是问题/SQL 的配对。您可以向此列表中添加样本对，但其目的是不用于典型的自然语言对话。

当您从对话提示创建一个链并执行时，Kinetica 语言模型将生成 SQL。可选地，可以使用 `KineticaSqlOutputParser` 执行 SQL 并返回结果数据框。

目前支持两种用于生成 SQL 的 LLM：

1. **Kinetica SQL-GPT**：基于 OpenAI ChatGPT API。
2. **Kinetica SqlAssist**：专为集成到 Kinetica 数据库而设计，可以在客户现场安全运行。

对于本示例我们将使用 **SqlAssist**。有关更多信息，请参阅 [Kinetica 文档站点](https://docs.kinetica.com/7.1/sql-gpt/concepts/)。

## 先决条件

要开始，您需要一个 Kinetica 数据库实例。如果没有，可以获取一个[免费开发实例](https://cloud.kinetica.com/trynow)。

您需要安装以下包...

```python
pip install -qU langchain-kinetica faker
```

## 数据库连接

必须在以下环境变量中设置数据库连接信息。如果您使用虚拟环境，可以在项目的 `.env` 文件中设置这些值：

* `KINETICA_URL`: 数据库连接 URL（例如 `http://localhost:9191`）
* `KINETICA_USER`: 数据库用户
* `KINETICA_PASSWD`: 安全密码。

如果可以创建一个 `KineticaChatLLM` 实例，则表示已成功连接。

```python
from langchain_kinetica import ChatKinetica

kinetica_llm = ChatKinetica()

# 我们将要创建的测试表
table_name = "demo.user_profiles"

# 我们将要创建的 LLM 上下文
kinetica_ctx = "demo.test_llm_ctx"
```

### 创建一些虚假用户资料

我们将使用 `faker` 包来创建包含 100 个虚假资料的数据框。

```python
from collections.abc import Generator

import pandas as pd
from faker import Faker

Faker.seed(5467)
faker = Faker(locale="en-US")


def profile_gen(count: int) -> Generator:
    for p_id in range(count):
        rec = dict(id=p_id, **faker.simple_profile())
        rec["birthdate"] = pd.Timestamp(rec["birthdate"])
        yield rec


load_df = pd.DataFrame.from_records(data=profile_gen(100), index="id")
print(load_df.head())
```

### 创建 LLM 上下文

您可以使用 Kinetica Workbench UI 创建一个上下文，或者手动创建它并使用 `CREATE OR REPLACE CONTEXT` 语法。

这里我们从 SQL 语法中引用了我们创建的表来创建一个上下文。

```python
from gpudb import GPUdbSamplesClause, GPUdbSqlContext, GPUdbTableClause

table_ctx = GPUdbTableClause(table=table_name, comment="Contains user profiles.")

samples_ctx = GPUdbSamplesClause(
    samples=[
        (
            "How many users born after 1970 are there?",
            f"""
            select count(1) as num_users
                from {table_name}
                where birthdate > '1970-01-01';
            """,
        )
    ]
)

context_sql = GPUdbSqlContext(
    name=kinetica_ctx, tables=[table_ctx], samples=samples_ctx
).build_sql()

print(context_sql)
count_affected = kinetica_llm.kdbc.execute(context_sql)
count_affected
```

## 使用 LangChain 进行推断

在下面的示例中，我们将从之前创建的表和 LLM 上下文创建一个链。此链将生成 SQL 并返回结果数据作为数据框。

### 从 Kinetica 数据库加载对话提示

`load_messages_from_context()` 函数将从数据库检索上下文并将其转换为用于创建 `ChatPromptTemplate` 的聊天消息列表。

```python
from langchain_core.prompts import ChatPromptTemplate

# 从数据库加载上下文
ctx_messages = kinetica_llm.load_messages_from_context(kinetica_ctx)

# 添加输入提示。这是问题将被替换的地方。
ctx_messages.append(("human", "{input}"))

# 创建提示模板。
prompt_template = ChatPromptTemplate.from_messages(ctx_messages)
print(prompt_template.pretty_repr())
```

```text

CREATE TABLE demo.user_profiles AS
(

    birthdate TIMESTAMP NOT NULL

COMMENT ON TABLE demo.user_profiles IS 'Contains user profiles.';

================================== Ai Message ==================================

select count(1) as num_users
    from demo.user_profiles
    where birthdate > '1970-01-01';

================================ Human Message =================================

{input}
```

### 创建链

此链的最后一个元素是 `KineticaSqlOutputParser`，它将执行 SQL 并返回数据框。这是可选的，如果我们不使用它，则只会返回 SQL。

```python
from langchain_kinetica import (
    KineticaSqlOutputParser,
    KineticaSqlResponse,
)

chain = prompt_template | kinetica_llm | KineticaSqlOutputParser(kdbc=kinetica_llm.kdbc)
```

### 生成 SQL

我们创建的链将接受一个问题作为输入，并返回包含生成的 SQL 和数据的 `KineticaSqlResponse`。问题必须与用于创建提示模板的 LLM 上下文相关。

```python
# 这里您必须提出一个与提示模板中提供的 LLM 上下文相关的问题。
response: KineticaSqlResponse = chain.invoke(
    {"input": "What users were born after 1990?"}
)

print(f"SQL: {response.sql}")
print(response.dataframe.head())
```
