[Llamafile](https://github.com/Mozilla-Ocho/llamafile) 允许你通过单个文件分发和运行 LLM。

Llamafile 通过将 [llama.cpp](https://github.com/ggerganov/llama.cpp) 与 [Cosmopolitan Libc](https://github.com/jart/cosmopolitan) 结合到一个框架中来实现这一功能，该框架将所有 LLM 的复杂性简化为一个单文件可执行程序（称为“llamafile”），可在大多数计算机上本地运行，无需安装。

## 设置

1. 下载你想要使用的模型的 llamafile。你可以在 [HuggingFace](https://huggingface.co/models?other=llamafile) 上找到许多 llamafile 格式的模型。在本指南中，我们将下载一个较小的模型 `TinyLlama-1.1B-Chat-v1.0.Q5_K_M`。注意：如果你没有安装 `wget`，可以直接通过此 [链接](https://huggingface.co/jartine/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/TinyLlama-1.1B-Chat-v1.0.Q5_K_M.llamafile?download=true) 下载模型。

```bash
wget https://huggingface.co/jartine/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/TinyLlama-1.1B-Chat-v1.0.Q5_K_M.llamafile
```

2. 使 llamafile 具备可执行权限。首先，如果尚未打开终端，请打开它。**如果你使用的是 MacOS、Linux 或 BSD，** 你需要使用 `chmod` 授予计算机执行此新文件的权限（见下文）。**如果你使用的是 Windows，** 请将文件名末尾添加 `.exe` 进行重命名（模型文件应命名为 `TinyLlama-1.1B-Chat-v1.0.Q5_K_M.llamafile.exe`）。

```bash
chmod +x TinyLlama-1.1B-Chat-v1.0.Q5_K_M.llamafile  # 在 MacOS、Linux 或 BSD 系统上运行
```

3. 以“服务器模式”运行 llamafile：

```bash
./TinyLlama-1.1B-Chat-v1.0.Q5_K_M.llamafile --server --nobrowser
```

现在你可以调用 llamafile 的 REST API 了。默认情况下，llamafile 服务器监听 [localhost:8080](http://localhost:8080)。你可以在 [llama.cpp 服务器 API 端点 README](https://github.com/Mozilla-Ocho/llamafile/blob/main/llama.cpp/server/README.md#api-endpoints) 中找到完整的服务器文档。你可以直接通过 REST API 与 llamafile 交互，但此处我们将展示如何使用 LangChain 与其交互。

## 用法

```python
from langchain_community.llms.llamafile import Llamafile

llm = Llamafile()

llm.invoke("Tell me a joke")
```

```text
'? \nI\'ve got a thing for pink, but you know that.\n"Can we not talk about work anymore?" - What did she say?\nI don\'t want to be a burden on you.\nIt\'s hard to keep a good thing going.\nYou can\'t tell me what I want, I have a life too!'
```

若要流式传输 Token，请使用 `.stream(...)` 方法：

```python
query = "Tell me a joke"

for chunks in llm.stream(query):
    print(chunks, end="")

print()
```
