## 设置

要访问 Moonshot 模型，你需要一个 Moonshot 账户、一个 API 密钥以及 `langchain-moonshot` 集成包。

### 凭证

前往 [Moonshot 控制台](https://platform.moonshot.ai/console/api-keys) 创建 API 密钥。完成后，设置 `MOONSHOT_API_KEY` 环境变量。

```python
import getpass
import os

if not os.getenv("MOONSHOT_API_KEY"):
    os.environ["MOONSHOT_API_KEY"] = getpass.getpass("Enter your Moonshot API key: ")
```

默认情况下，该包使用 Moonshot 的国际端点（`https://api.moonshot.ai/v1`）。若要改用中国端点，请设置：

```python
import os

os.environ["MOONSHOT_API_BASE"] = "https://api.moonshot.cn/v1"
```

要启用模型调用的自动追踪，请设置你的 [LangSmith](/langsmith/home) API 密钥：

```python
import getpass
import os

os.environ["LANGSMITH_API_KEY"] = getpass.getpass("Enter your LangSmith API key: ")
os.environ["LANGSMITH_TRACING"] = "true"
```

## 实例化

现在我们可以实例化模型对象并生成响应：

```python
from langchain_moonshot import ChatMoonshot

llm = ChatMoonshot(
    model="kimi-k2.5",
    thinking=False,
    temperature=0.6,
    max_retries=2,
    # prompt_cache_key="docs-example-cache",
    # safety_identifier="docs-example-user",
    # max_completion_tokens=1024,
)
```

<Note>
    对于 `kimi-k2.5`，如果设置了 `temperature`，当 `thinking=True` 时必须为 `1.0`，当 `thinking=False` 时必须为 `0.6`。省略 `thinking`（或将其设置为 `None`）在验证目的上被视为启用 thinking。要使用 `temperature=0.6`，请显式设置 `thinking=False`。
</Note>

## 调用

```python
messages = [
    ("system", "You are a concise bilingual assistant."),
    ("human", "Summarize why Moonshot reasoning models are useful in two bullet points."),
]

ai_msg = llm.invoke(messages)

print(ai_msg.text)
print(ai_msg.usage_metadata)
```

## 推理输出

`ChatMoonshot` 在非流式和流式响应中均保留 Moonshot 的 `reasoning_content` 字段。

```python
reasoning_llm = ChatMoonshot(
    model="kimi-k2.5",
    thinking=True,
    temperature=1.0,
)

ai_msg = reasoning_llm.invoke(
    "Explain in two bullet points why reasoning models are useful."
)

print(ai_msg.text)
print(ai_msg.additional_kwargs.get("reasoning_content"))
```

## 流式传输

要在流式传输时恢复用量元数据，请设置 `stream_usage=True`：

```python
streaming_llm = ChatMoonshot(
    model="kimi-k2.5",
    thinking=True,
    temperature=1.0,
    stream_usage=True,
)

full = None
for chunk in streaming_llm.stream("Explain streaming output in two short bullet points."):
    if full is None:
        full = chunk
    else:
        full += chunk

    if chunk.text:
        print(chunk.text, end="")

    reasoning = chunk.additional_kwargs.get("reasoning_content")
    if reasoning:
        print(f"\n[reasoning] {reasoning}", end="")

print()
print(full.usage_metadata if full is not None else None)
```

## 工具调用

Moonshot 支持通过 `bind_tools` 进行 LangChain 工具调用：

```python
from langchain.messages import ToolMessage
from langchain.tools import tool


@tool
def add(a: int, b: int) -> int:
    """相加两个整数。"""
    return a + b


@tool
def multiply(a: int, b: int) -> int:
    """相乘两个整数。"""
    return a * b


llm_with_tools = ChatMoonshot(
    model="kimi-k2.5",
    thinking=False,
    temperature=0.6,
).bind_tools([add, multiply])

messages = [
    ("system", "Use the provided math tools before answering."),
    ("human", "Add 17 and 25, multiply 12 by 13, then summarize the results."),
]

response = llm_with_tools.invoke(messages)
print(response.tool_calls)

if response.tool_calls:
    tools_map = {"add": add, "multiply": multiply}
    tool_results = []
    for tool_call in response.tool_calls:
        result = tools_map[tool_call["name"]].invoke(tool_call["args"])
        tool_results.append(
            ToolMessage(content=str(result), tool_call_id=tool_call["id"])
        )

    final_response = llm_with_tools.invoke([*messages, response, *tool_results])
    print(final_response.text)
```

<Note>
    对于 `thinking=True` 的 `kimi-k2.5`，`tool_choice` 必须为 `"auto"` 或 `"none"`。不支持强制工具选择（指定函数名）。
</Note>

## 结构化输出

Moonshot 通过 LangChain 的 `with_structured_output(...)` 支持结构化输出。此包中 Moonshot 不暴露独立的 `json_schema`  steering 路径，因此 `method="json_schema"` 有意降级为 `function_calling`。

```python
from pydantic import BaseModel, Field


class WeatherAnswer(BaseModel):
    city: str = Field(description="City name")
    summary: str = Field(description="One-sentence weather summary")


structured_llm = ChatMoonshot(
    model="kimi-k2.5",
    thinking=False,
    temperature=0.6,
).with_structured_output(WeatherAnswer)

result = structured_llm.invoke("Summarize today's weather in Shanghai.")
print(result)
```

## 多模态输入

支持视觉的 Moonshot 模型接受 OpenAI 风格的 `image_url` 内容块：

```python
from langchain.messages import HumanMessage

vision_llm = ChatMoonshot(
    model="moonshot-v1-32k-vision-preview",
)

message = HumanMessage(
    content=[
        {"type": "text", "text": "Describe the image and mention one concrete detail."},
        {
            "type": "image_url",
            "image_url": {"url": "data:image/png;base64,<your-base64-image>"},
        },
    ]
)

response = vision_llm.invoke([message])
print(response.text)
```

## Moonshot 特定说明

- `ChatMoonshot` 是一个独立的 LangChain 集成包，用于 Moonshot AI 聊天模型，构建于 `langchain-openai` 之上。
- 该包暴露的 Moonshot 特定请求控制包括 `thinking`、`prompt_cache_key`、`safety_identifier` 和 `max_completion_tokens`。
- `kimi-k2.5` 的验证比通用的 OpenAI 兼容聊天模型更严格。
- 对于 `kimi-k2.5`，`top_p` 必须保持 `0.95`，`n` 必须保持 `1`，且 `presence_penalty` 和 `frequency_penalty` 都必须保持 `0.0`。
- 当 `thinking=True` 时，Moonshot 内置的 `$web_search` 会被 `kimi-k2.5` 拒绝。
