**集成是 LangChain 的核心组件。**

LangChain 为多种不同的组件（语言模型、向量存储等）提供了标准接口，这些组件在构建 LLM 应用程序时至关重要。贡献集成有助于扩展 LangChain 的生态系统，并让数百万开发者能够发现您的服务。

## 为什么要向 LangChain 贡献集成？

<Card title="可发现性" icon="search">
    LangChain 是用于构建 LLM 应用程序的最常用框架，每月下载量超过 2000 万次。
</Card>

<Card title="互操作性" icon="refresh">
    LangChain 组件公开了标准接口，允许开发者轻松地互相交换它们。如果您实现了 LangChain 集成，任何使用不同组件的开发者都将能够轻松地换用您的组件。
</Card>

<Card title="最佳实践" icon="star">
    通过其标准接口，LangChain 组件鼓励并促进最佳实践（流式传输、异步等），从而改善开发者体验和应用程序性能。
</Card>

## 要集成的组件

虽然任何组件都可以集成到 LangChain 中，但我们更鼓励特定类型的集成：

**集成这些 ✅**:
- [**聊天模型**](/oss/javascript/integrations/chat): 最活跃使用的组件类型
- [**工具/工具包**](/oss/javascript/integrations/tools): 启用代理功能
- [**检索器**](/oss/javascript/integrations/retrievers): RAG 应用程序的核心
- [**嵌入模型**](/oss/javascript/integrations/text_embedding): 向量操作的基础
- [**向量存储**](/oss/javascript/integrations/vectorstores): 语义搜索必不可少
- [**沙盒**](/oss/javascript/deepagents/sandboxes): 使用深度代理安全地运行代码

<Accordion title="额外的第三方沙盒集成标准">
请注意，只有在以下情况下，我们才会推介第三方沙盒集成：

- 该集成由提供沙盒的公司编写和维护。
- **或者** 该集成被广泛使用，这意味着该集成必须在 PyPI 或 npm 上有至少 10,000 次的日下载量才能被考虑推介。
</Accordion>

**不要这些 ❌**:
- **LLM（文本补全模型）**: 已弃用，取而代之的是 [聊天模型](/oss/javascript/integrations/chat)
- [**文档加载器**](/oss/javascript/integrations/document_loaders): 维护负担重
- [**键值存储**](/oss/javascript/integrations/stores): 使用有限
- **文档转换器**: 小众用例
- **模型缓存**: 基础设施问题
- **图**: 复杂的抽象
- **消息历史**: 存储抽象
- **回调**: 系统级组件
- **聊天加载器**: 需求有限
- **适配器**: 边缘情况实用程序

## 如何贡献集成

<Steps>
    <Step title="确认资格">
        验证您的集成是否在我们目前接受的 [受鼓励组件](#components-to-integrate) 列表中。
    </Step>

    <Step title="实施您的包">
        <Card title="如何实现 LangChain 集成" icon="link" href="/oss/javascript/contributing/implement-langchain" arrow />
    </Step>

    <Step title="通过标准测试">
        如果适用，为您的集成实现对 LangChain [标准测试](/oss/javascript/contributing/standard-tests-langchain) 套件的支持并成功运行它们。
    </Step>

    <Step title="发布集成">
        <Card title="如何发布集成" icon="upload" href="/oss/javascript/contributing/publish-langchain" arrow />
    </Step>

    <Step title="添加文档">
        打开一个 PR，为您的集成向官方 LangChain 文档添加文档。

        <Accordion title="集成文档指南" icon="book">
            集成的作用取决于其文档。为了确保用户获得一致的体验，所有新集成都需要文档。我们为每种类型的集成提供了一个标准的起点模板，供您复制和修改。

            在向 LangChain [docs 仓库](https://github.com/langchain-ai/docs) 提交的新 PR 中，在 `src/oss/python/integrations/<component_type>/integration_name.mdx` 下的相关目录中创建一个新文件，使用适当的模板文件：

            - [聊天模型](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx)
            - [工具和工具包](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx)
            - [检索器](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/retrievers/TEMPLATE.mdx)
            - 文本分割器 - 即将推出
            - 嵌入模型 - 即将推出
            - [向量存储](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx)
            - 文档加载器 - 即将推出
            - 键值存储 - 即将推出

            对于参考文档，请在仓库上打开一个 issue，以便维护者可以添加它们。
        </Accordion>
    </Step>

    <Step title="联合营销" icon="speakerphone">
        (可选) 与 LangChain 团队接洽以进行联合 [联合营销](/oss/javascript/contributing/comarketing)。
    </Step>
</Steps>
