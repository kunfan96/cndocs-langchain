Oracle Cloud Infrastructure (OCI) 生成式 AI 是一项完全托管的服务，提供一系列最先进的、可定制的大型语言模型（LLM），涵盖广泛的应用场景，并通过单一 API 提供服务。
使用 OCI 生成式 AI 服务，您可以访问开箱即用的预训练模型，或者基于自有数据在专用 AI 集群上创建并托管您自己的微调自定义模型。有关该服务和 API 的详细文档可在 [OCI 生成式 AI 文档](https://docs.oracle.com/en-us/iaas/Content/generative-ai/home.htm) 和 [OCI 生成式 AI API 参考](https://docs.oracle.com/en-us/iaas/api/#/en/generative-ai/20231130/) 中查阅。

本笔记本介绍了如何在 LangChain 中使用 OCI 的生成式 AI 完整模型。

## 环境配置

确保已安装 oci SDK 和 langchain-community 包

## 使用方法

```python
from langchain_community.llms.oci_generative_ai import OCIGenAI

llm = OCIGenAI(
    model_id="cohere.command",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="MY_OCID",
    model_kwargs={"temperature": 0, "max_tokens": 500},
)

response = llm.invoke("Tell me one fact about earth", temperature=0.7)
print(response)
```

#### 结合提示词模板进行链式调用

```python
from langchain_core.prompts import PromptTemplate

llm = OCIGenAI(
    model_id="cohere.command",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="MY_OCID",
    model_kwargs={"temperature": 0, "max_tokens": 500},
)

prompt = PromptTemplate(input_variables=["query"], template="{query}")
llm_chain = prompt | llm

response = llm_chain.invoke("what is the capital of france?")
print(response)
```

#### 流式输出

```python
llm = OCIGenAI(
    model_id="cohere.command",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="MY_OCID",
    model_kwargs={"temperature": 0, "max_tokens": 500},
)

for chunk in llm.stream("Write me a song about sparkling water."):
    print(chunk, end="", flush=True)
```

## 身份验证

LlamaIndex 支持的认证方法与其他 OCI 服务使用的认证方法等效，并遵循__[标准 SDK 认证](https://docs.oracle.com/en-us/iaas/Content/API/Concepts/sdk_authentication_methods.htm)__方法，具体包括 API Key、会话令牌（session token）、实例主体（instance principal）和资源主体（resource principal）。

API Key 是上述示例中使用的默认认证方法。以下示例演示如何使用不同的认证方法（会话令牌）：

```python
llm = OCIGenAI(
    model_id="cohere.command",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="MY_OCID",
    auth_type="SECURITY_TOKEN",
    auth_profile="MY_PROFILE",  # 替换为您的配置文件名称
    auth_file_location="MY_CONFIG_FILE_LOCATION",  # 替换为包含配置文件名称配置的所在位置
)
```

## 专用 AI 集群

要访问托管在专用 AI 集群中的模型，请__[创建端点](https://docs.oracle.com/en-us/iaas/api/#/en/generative-ai-inference/20231130/)*，其分配的 OCID（目前以‘ocid1.generativeaiendpoint.oc1.us-chicago-1’为前缀）将用作您的模型 ID。

当访问托管在专用 AI 集群中的模型时，您需要使用两个额外的必需参数（“provider”和“context_size”）来初始化 OCIGenAI 接口。

```python
llm = OCIGenAI(
    model_id="ocid1.generativeaiendpoint.oc1.us-chicago-1....",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="DEDICATED_COMPARTMENT_OCID",
    auth_profile="MY_PROFILE",  # 替换为您的配置文件名称
    auth_file_location="MY_CONFIG_FILE_LOCATION",  # 替换为包含配置文件名称配置的所在位置
    provider="MODEL_PROVIDER",  # 例如："cohere" 或 "meta"
    context_size="MODEL_CONTEXT_SIZE",  # 例如：128000
)
```
