`TitanML` 通过我们的训练、压缩和推理优化平台，帮助企业构建并部署更优质、更小、更经济且更快的自然语言处理（NLP）模型。

我们的推理服务器 [Titan Takeoff](https://docs.titanml.co/docs/intro) 支持通过单条命令在本地硬件上部署大语言模型（LLM）。大多数嵌入模型开箱即用，如果您在使用特定模型时遇到问题，请通过 [hello@titanml.co](mailto:hello@titanml.co) 联系我们。

## 示例用法

以下是一些帮助您开始使用 Titan Takeoff Server 的实用示例。运行这些命令前，请确保 Takeoff Server 已在后台启动。更多信息请参阅 [启动 Takeoff 的文档页面](https://docs.titanml.co/docs/Docs/launching/)。

```python
import time

from langchain_community.embeddings import TitanTakeoffEmbed
```

### 示例 1

基本用法，假设 Takeoff 正在您的机器上使用默认端口运行（例如 localhost:3000）。

```python
embed = TitanTakeoffEmbed()
output = embed.embed_query(
    "What is the weather in London in August?", consumer_group="embed"
)
print(output)
```

### 示例 2

使用 TitanTakeoffEmbed Python 包装器启动读取器。如果在首次启动 Takeoff 时尚未创建任何读取器，或者您想添加其他读取器，可以在初始化 TitanTakeoffEmbed 对象时完成。只需将想要启动的模型列表作为 `models` 参数传入即可。

您可以使用 `embed.query_documents` 一次性嵌入多个文档。预期的输入是字符串列表，而不是 `embed_query` 方法所期望的单个字符串。

```python
# 嵌入模型的模型配置，您可以在其中指定以下参数：
#   model_name (str)：要使用的模型名称
#   device (str)：用于推理的设备，cuda 或 cpu
#   consumer_group (str)：将读取器放入的消费者组
embedding_model = {
    "model_name": "BAAI/bge-large-en-v1.5",
    "device": "cpu",
    "consumer_group": "embed",
}
embed = TitanTakeoffEmbed(models=[embedding_model])

# 模型需要时间启动，所需时间取决于模型大小和网络连接速度
time.sleep(60)

prompt = "What is the capital of France?"
# 我们指定了 "embed" 消费者组，因此需要将请求发送到相同的消费者组，以便命中我们的嵌入模型而非其他模型
output = embed.embed_query(prompt, consumer_group="embed")
print(output)
```
