**集成是 LangChain 的核心组成部分。**

LangChain 为构建 LLM 应用程序时至关重要的多种不同组件（语言模型、向量存储等）提供了标准接口。贡献集成有助于扩展 LangChain 生态系统，并使您的服务能够被数百万开发者发现。

## 为什么要向 LangChain 贡献集成？

<Card title="可发现性" icon="search">
    LangChain 是构建 LLM 应用程序最常用的框架，每月下载量超过 2000 万次。
</Card>

<Card title="互操作性" icon="refresh">
    LangChain 组件提供了标准接口，使开发者能够轻松地相互替换。如果您实现了 LangChain 集成，任何使用其他组件的开发者都能轻松将您的组件替换进来。
</Card>

<Card title="最佳实践" icon="star">
    通过标准接口，LangChain 组件鼓励并促进最佳实践（流式处理、异步等），从而改善开发者体验和应用程序性能。
</Card>

## 可集成的组件

虽然任何组件都可以集成到 LangChain 中，但我们更鼓励特定类型的集成：

**推荐集成 ✅**：
- [**聊天模型**](/oss/python/integrations/chat)：使用最活跃的组件类型
- [**工具/工具包**](/oss/python/integrations/tools)：启用智能体功能
- [**检索器**](/oss/python/integrations/retrievers)：RAG 应用程序的核心
- [**嵌入模型**](/oss/python/integrations/text_embedding)：向量操作的基础
- [**向量存储**](/oss/python/integrations/vectorstores)：语义搜索的必备组件
- [**沙盒**](/oss/python/deepagents/sandboxes)：使用 deep agents 安全运行代码

<Accordion title="第三方沙盒集成的额外标准">
请注意，我们仅在以下情况下展示第三方沙盒集成：

- 该集成由提供沙盒的公司编写和维护。
- **或者**该集成被广泛使用，即在 PyPI 或 npm 上每日下载量至少达到 10,000 次，才会被考虑展示。
</Accordion>

**不推荐 ❌**：
- **LLMs（文本补全模型）**：已被[聊天模型](/oss/python/integrations/chat)所取代
- [**文档加载器**](/oss/python/integrations/document_loaders)：维护负担较重
- [**键值存储**](/oss/python/integrations/stores)：使用量有限
- **文档转换器**：使用场景较为小众
- **模型缓存**：基础设施相关问题
- **图**：抽象层次复杂
- **消息历史**：存储抽象
- **回调**：系统级组件
- **聊天加载器**：需求有限
- **适配器**：边缘案例工具

## 如何贡献集成

<Steps>
    <Step title="确认资格">
        确认您的集成在我们目前接受的[推荐组件](#components-to-integrate)列表中。
    </Step>

    <Step title="实现您的包">
        <Card title="如何实现 LangChain 集成" icon="link" href="/oss/python/contributing/implement-langchain" arrow />
    </Step>

    <Step title="通过标准测试">
        如果适用，请为您的集成实现 LangChain 的[标准测试](/oss/python/contributing/standard-tests-langchain)套件并成功运行。
    </Step>

    <Step title="发布集成">
        <Card title="如何发布集成" icon="upload" href="/oss/python/contributing/publish-langchain" arrow />
    </Step>

    <Step title="添加文档">
        提交 PR 将您的集成文档添加到官方 LangChain 文档中。

        <Accordion title="集成文档指南" icon="book">
            集成的价值取决于其文档质量。为确保用户体验一致，所有新集成都需要提供文档。我们为每种集成类型提供了标准的起始模板供您复制和修改。

            在 LangChain [文档仓库](https://github.com/langchain-ai/docs) 的新 PR 中，使用相应的模板文件在 `src/oss/python/integrations/<component_type>/integration_name.mdx` 路径下创建新文件：

            - [聊天模型](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx)
            - [工具和工具包](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx)
            - [检索器](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/retrievers/TEMPLATE.mdx)
            - 文本分割器 - 即将推出
            - 嵌入模型 - 即将推出
            - [向量存储](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx)
            - 文档加载器 - 即将推出
            - 键值存储 - 即将推出

            对于参考文档，请在仓库中提交 issue，维护人员将负责添加。
        </Accordion>
    </Step>

    <Step title="联合营销" icon="speakerphone">
        （可选）与 LangChain 团队合作进行[联合营销](/oss/python/contributing/comarketing)。
    </Step>
</Steps>
