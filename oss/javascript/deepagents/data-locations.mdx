Deep Agents CLI 将数据存储在两个目录层次结构中：

- **`~/.deepagents/`** — Deep Agents 特定数据（智能体记忆、技能、会话）
- **`~/.agents/`** — 工具无关数据（跨 AI CLI 工具共享的技能）

## 目录结构

```text
~/.deepagents/
├── sessions.db              # 用于对话检查点的 SQLite 数据库
├── history.jsonl            # 命令输入历史记录
└── {agent}/                 # 每个智能体的目录（默认："agent"）
    ├── AGENTS.md            # 用户对智能体指令的自定义
    ├── skills/              # 用户级技能
    │   └── {skill-name}/
    │       └── SKILL.md
    └── agents/              # 自定义子智能体定义
        └── {subagent-name}/
            └── AGENTS.md

~/.agents/                   # 工具无关的别名（跨 AI CLI 共享）
└── skills/                  # 可供任何兼容工具使用的技能
    └── {skill-name}/
        └── SKILL.md

{project}/                   # 项目级（在 git 仓库根目录中）
├── AGENTS.md                # 项目指令（根级别）
└── .deepagents/
│   ├── AGENTS.md            # 项目指令（首选位置）
│   ├── skills/              # 项目特定技能
│   │   └── {skill-name}/
│   │       └── SKILL.md
│   └── agents/              # 项目特定子智能体
│       └── {subagent-name}/
│           └── AGENTS.md
└── .agents/                 # 工具无关的项目技能
    └── skills/
        └── {skill-name}/
            └── SKILL.md
```

## 什么去哪里

| 数据 | 位置 | 读/写 | 备注 |
|------|----------|------------|-------|
| **会话** | `~/.deepagents/sessions.db` | R/W | SQLite 检查点数据库 |
| **输入历史** | `~/.deepagents/history.jsonl` | R/W | JSON-lines，上下箭头调用 |
| **基本指令** | 包 `default_agent_prompt.md` | R | 不可变，随 CLI 升级更新 |
| **用户自定义** | `~/.deepagents/{agent}/AGENTS.md` | R/W | 附加到基本指令 |
| **项目指令** | `.deepagents/AGENTS.md` 或 `AGENTS.md` | R | 如果存在则同时加载 |
| **用户技能** | `~/.deepagents/{agent}/skills/` | R/W | 智能体特定技能 |
| **共享技能** | `~/.agents/skills/` | R | 工具无关，跨 CLI |
| **项目技能** | `.deepagents/skills/` 或 `.agents/skills/` | R | 项目范围 |
| **自定义子智能体** | `~/.deepagents/{agent}/agents/` | R/W | 用户定义的子智能体 |
| **项目子智能体** | `.deepagents/agents/` | R | 项目定义的子智能体 |

## 优先级规则

当同一项目存在于多个位置时，**优先级高的完全获胜**（无合并）。

### 技能

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/skills/` — 用户 deepagents
2. `~/.agents/skills/` — 用户工具无关
3. `.deepagents/skills/` — 项目 deepagents
4. `.agents/skills/` — 项目工具无关 *(最高)*

### 子智能体

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/agents/` — 用户级
2. `.deepagents/agents/` — 项目级 *(最高)*

### 指令

所有指令来源都 **组合**（不被覆盖）：

1. 包基本提示 *(始终加载)*
2. `~/.deepagents/{agent}/AGENTS.md` *(附加)*
3. `.deepagents/AGENTS.md` *(附加)*
4. 项目根目录的 `AGENTS.md` *(附加)*

## `.deepagents` vs `.agents`

| 目录 | 目的 | 何时使用 |
|-----------|---------|-------------|
| `.deepagents/` | Deep Agents CLI 特定 | 使用 CLI 特定功能及其技能和配置 |
| `.agents/` | 工具无关 | 你想跨不同 AI CLI 工具共享的技能 |

<Tip>
使用 `.agents/skills/` 存放适用于任何 AI 编码助手的技能。
使用 `.deepagents/skills/` 存放依赖于 Deep Agents 特定工具或约定的技能。
</Tip>

## 清理

| 需求 | 操作 |
|------|--------|
| 重置所有数据 | `rm -rf ~/.deepagents` |
| 仅清除会话 | `rm ~/.deepagents/sessions.db` |
| 清除输入历史 | `rm ~/.deepagents/history.jsonl` |
| 重置智能体指令 | `deepagents reset --agent {name}` |
| 删除技能 | `rm -rf ~/.deepagents/{agent}/skills/{skill-name}` |

<Warning>
删除 `~/.deepagents/sessions.db` 将删除所有对话历史记录和检查点。
此操作无法撤消。
</Warning>
