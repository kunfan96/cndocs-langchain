当你准备好将你的 LangChain 代理部署到生产环境时，LangSmith 提供了一个为代理工作负载设计的托管平台。传统的托管平台是为无状态、短期的 web 应用程序构建的，而 LangGraph 是**专门为需要持久状态和后台执行的有状态、长时间运行的代理而构建的**。LangSmith 处理基础设施、扩展和运维问题，因此你可以直接从你的仓库部署。

## 前提条件

在开始之前，确保你有以下内容：

- 一个 [GitHub 账户](https://github.com/)
- 一个 [LangSmith 账户](https://smith.langchain.com/)（免费注册）

## 部署你的代理

### 1. 在 GitHub 上创建仓库

你的应用程序代码必须位于 GitHub 仓库中才能在 LangSmith 上部署。公共和私有仓库都受支持。对于这个快速入门，首先通过遵循 [本地服务器设置指南](/oss/javascript/langchain/studio#setup-local-agent-server) 确保你的应用程序与 LangGraph 兼容。然后，将你的代码推送到仓库。
