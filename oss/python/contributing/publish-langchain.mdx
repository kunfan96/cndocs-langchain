**使您的集成可供社区使用。**

<Warning>
**请勿向 LangChain 或 Deep Agents 仓库提交集成 PR。**

新的集成应发布为 **独立的 PyPI 包**，在您自己的 GitHub 组织或账户下（例如 `langchain-yourservice`），而不是作为对 [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain) 仓库的 PR。

主仓库仅包含由 LangChain 团队维护的第一方集成的一小部分（如 OpenAI、Anthropic 和 Ollama）。
</Warning>

现在您的包已经实现并经过测试，您可以发布它并在文档中添加说明，使社区能够发现它。

## 发布您的包

<Info>
    本指南假设您已经实现了该包并为其编写了测试。如果没有，请参阅 [实现指南](/oss/python/contributing/implement-langchain) 和 [测试指南](/oss/python/contributing/standard-tests-langchain)。
</Info>

为了本文档的目的，我们将使用 PyPI 作为包注册表。您可以选择发布到其他注册表；具体步骤会有所不同。

### 设置凭证

首先，请确保您有一个 PyPI 账户：

<AccordionGroup>
    <Accordion title="如何创建 PyPI Token" icon="key">
        <Steps>
            <Step title="创建账户">
                访问 [PyPI 网站](https://pypi.org/) 并创建一个账户
            </Step>

            <Step title="验证邮箱">
                单击 PyPI 发送到您邮箱的链接以验证您的邮箱地址
            </Step>

            <Step title="启用双因素认证 (2FA)">
                转到您的账户设置并点击“生成恢复代码”以启用 2FA。为了生成 API token，您 **必须** 启用 2FA
            </Step>

            <Step title="生成 token">
                转到您的账户设置并 [生成一个新的 API token](https://pypi.org/manage/account/token/)
            </Step>
        </Steps>
    </Accordion>
</AccordionGroup>

## 添加文档

要在 [集成标签页](/oss/python/integrations/providers/overview) 下为您的包添加文档，您需要创建相关的文档页面并在 [LangChain 文档仓库](https://github.com/langchain-ai/docs) 中打开一个 PR。

### 编写文档

根据您构建的集成类型，您可能需要创建不同类型的文档页面。LangChain 提供了不同类型的集成模板以帮助您开始编写文档。

<CardGroup>
    <Card title="聊天模型" icon="message" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx" arrow/>
    <Card title="工具/工具包" icon="tool" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx" arrow/>
    <Card title="中间件" icon="plug" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/middleware/TEMPLATE.mdx" arrow/>
    <Card title="向量存储" icon="database" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx" arrow/>
</CardGroup>

<Tip>
    要引用现有文档，您可以查看 [集成列表](/oss/python/integrations/providers/overview) 并找到与您类似的项目。

    要查看某个文档页面的原始 Markdown，请在页面右上角的“复制页面”旁边的下拉按钮中选择“以 Markdown 查看”。
</Tip>

### 向文档仓库提交 PR

在个人 GitHub 账户下为 [LangChain 文档仓库](https://github.com/langchain-ai/docs) 创建一个分支，并克隆到本地。为此集成创建一个新的分支。复制模板并使用您喜欢的 Markdown 编辑器进行修改。请务必参考和遵循 [文档指南](/oss/python/contributing/documentation) 来编写您的文档。

<Info>
    本 PR 仅涉及 **文档**。您的集成包本身应在其自己的 GitHub 组织或账户下的独立包中发布到 PyPI。
</Info>

<Warning>
    我们可能会拒绝或要求修改以下情况的 PR：
    - CI 检查失败
    - 存在严重的语法错误或拼写错误
    - 不正确地使用了 [Mintlify 组件](/oss/python/contributing/documentation#mintlify-components)
    - 缺少 [前言信息](/oss/python/contributing/documentation#page-structure)
    - 当适用时缺少 [本地化](/oss/python/contributing/documentation#localization)
    - 代码示例无法运行或存在错误
    - 不符合 **质量标准**（见下文）
</Warning>

请耐心等待我们处理大量 PR。我们将尽快审查您的 PR 并提供反馈或合并它。**不要反复标记维护者关于您的 PR 的问题。**

<Note>
    如果您的 PR 包含 AI 生成的内容，请遵循我们的 [LLM 合理使用政策](/oss/python/contributing/overview#acceptable-uses-of-llms)。
</Note>
