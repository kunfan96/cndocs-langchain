title: PaddleOCR-VL
---

[PaddleOCR](https://www.paddleocr.com) 是由百度开发的一款功能强大且轻量级的 OCR 工具包，可将图像和 PDF 与 LLM 连接起来。它支持超过 100 种语言，并将文档内容转换为结构化的、可供 AI 使用的数据。

此集成通过 `PaddleOCRVLLoader` 文档加载器提供了 PaddleOCR 的大模型文档解析能力。

## 概述

### 集成详情

| 类 | 包 | 本地支持 | 可序列化 | JS 支持 |
| :--- | :--- | :---: | :---: |  :---: |
| `PaddleOCRVLLoader` | `langchain-paddleocr` | ✅ | ❌ | ❌ |

### 加载器特性

| 来源 | 文档延迟加载 | 原生异步支持 |
| :---: | :---: | :---: |
| `PaddleOCRVLLoader` | ✅ | ❌ |

`PaddleOCRVLLoader` 使您能够：

- 使用百度 PaddleOCR-VL 系列模型（例如 PaddleOCR-VL、PaddleOCR-VL-1.5）从 PDF 和图像文件中提取文本和布局信息
- 处理来自本地文件或远程 URL 的文档

## 先决条件

要使用 PaddleOCR-VL 加载器，您需要：

1. **API 访问权限**：访问 PaddleOCR-VL API 端点
2. **身份验证**：API 的访问令牌（可以直接提供或通过 `PADDLEOCR_ACCESS_TOKEN` 环境变量提供）

API URL 和访问令牌均可在 [PaddleOCR 官方网站](https://www.paddleocr.com) 上获取。只需点击 **API** 按钮，并从提供的 API 调用示例中复制 URL 和令牌。

## 设置

```python
pip install langchain-paddleocr
```

## 初始化

基本初始化需要 API 端点 URL 和文件路径：

```python
from langchain_paddleocr import PaddleOCRVLLoader
from pydantic import SecretStr

loader = PaddleOCRVLLoader(
    file_path="path/to/document.pdf",
    api_url="your-api-endpoint",
    access_token=SecretStr("your-access-token")  # 如果使用环境变量，此项可选
)
```

通过环境变量进行身份验证：

```bash
export PADDLEOCR_ACCESS_TOKEN="your-access-token"
```

然后初始化时无需 `access_token` 参数：

```python
loader = PaddleOCRVLLoader(
    file_path="path/to/document.pdf",
    api_url="your-api-endpoint"
)
```

### 高级配置

加载器支持众多配置选项，用于微调文档处理：

```python
loader = PaddleOCRVLLoader(
    file_path=["document1.pdf", "document2.jpg"],  # 多个文件
    api_url="your-api-endpoint",

    access_token=None,  # 可选：用于 API 身份验证的 SecretStr
    file_type="pdf",  # 可选："pdf" 或 "image"，或 None 表示自动检测

    use_doc_orientation_classify=False,  # 启用文档方向分类
    use_doc_unwarping=False,  # 启用文档去扭曲
    use_layout_detection=None,  # 启用布局检测（None = 使用服务默认值）
    use_chart_recognition=None,  # 启用图表识别（None = 使用服务默认值）
    use_seal_recognition=None,  # 启用印章识别（None = 使用服务默认值）
    use_ocr_for_image_block=None,  # 在图像块上运行 OCR（None = 使用服务默认值）

    layout_threshold=None,  # 检测阈值（None = 使用服务默认值）
    layout_nms=None,  # 应用非最大抑制（None = 使用服务默认值）
    layout_unclip_ratio=None,  # 布局未裁剪比率（None = 使用服务默认值）
    layout_merge_bboxes_mode=None,  # 合并布局边界框的模式（None = 使用服务默认值）
    layout_shape_mode=None,  # 布局形状模式（None = 使用服务默认值）

    prompt_label=None,  # VLM 的提示标签（None = 使用服务默认值）
    format_block_content=None,  # 格式化块内容（None = 使用服务默认值）
    repetition_penalty=None,  # VLM 采样的重复惩罚（None = 使用服务默认值）
    temperature=None,  # VLM 采样的温度（None = 使用服务默认值）
    top_p=None,  # VLM 的 Top-p 采样值（None = 使用服务默认值）
    min_pixels=None,  # 预处理中允许的最小像素数（None = 使用服务默认值）
    max_pixels=None,  # 预处理中允许的最大像素数（None = 使用服务默认值）
    max_new_tokens=None,  # VLM 生成的最大令牌数（None = 使用服务默认值）

    merge_layout_blocks=None,  # 跨列合并布局块（None = 使用服务默认值）
    markdown_ignore_labels=None,  # Markdown 中要忽略的布局标签（None = 使用服务默认值）
    vlm_extra_args=None,  # 附加的 VLM 配置参数（None = 使用服务默认值）

    prettify_markdown=None,  # 美化 Markdown 输出（None = 使用服务默认值）
    show_formula_number=None,  # 在 Markdown 中包含公式编号（None = 使用服务默认值）
    restructure_pages=None,  # 跨页面重构结果（None = 使用服务默认值）
    merge_tables=None,  # 跨页面合并表格（None = 使用服务默认值）
    relevel_titles=None,  # 重新调整标题级别（None = 使用服务默认值）
    visualize=None,  # 包含可视化结果（None = 使用服务默认值）

    additional_params=None,  # 附加的 API 参数
    timeout=300,  # 请求超时时间（秒）
)
```

## 基本用法

### 加载文档

```python
# 加载单个文档
loader = PaddleOCRVLLoader(
    file_path="https://arxiv.org/pdf/2408.09869",
    api_url="your-api-endpoint"
)
docs = loader.load()

# 检查结果
for doc in docs[:2]:
    print(f"内容: {doc.page_content[:200]}...")
    print(f"来源: {doc.metadata['source']}")
    print("---")
```

### 处理多种文件类型

加载器会根据扩展名自动检测文件类型：

```python
# 混合文件类型 - 自动检测
files = [
    "document.pdf",      # PDF 文件
    "image.jpg",         # 图像文件
    "https://example.com/report.pdf"  # 远程 PDF
]

loader = PaddleOCRVLLoader(file_path=files, api_url="your-api-endpoint")
```

支持的图像格式：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.tiff`、`.tif`、`.webp`
支持的文档格式：`.pdf`

## 高级功能

### 访问原始 API 响应

加载器在文档元数据中包含完整的 API 响应：

```python
docs = loader.load()
first_doc = docs[0]

# 访问原始 API 响应以进行高级处理
raw_response = first_doc.metadata["paddleocr_vl_raw_response"]
print(f"布局结果数量: {len(raw_response['result']['layoutParsingResults'])}")
```

### 错误处理

加载器提供详细的错误消息以供故障排除：

```python
try:
    docs = loader.load()
except ValueError as e:
    print(f"处理失败: {e}")
    # 常见问题：无效的 API 端点、身份验证错误、不支持的文件类型
```

## 最佳实践

### 错误处理

- **网络超时**：为大型文档设置适当的 `timeout` 参数
- **身份验证**：使用环境变量进行安全的令牌管理
- **文件验证**：在处理前验证文件的可访问性

## 故障排除

### 常见问题

1. **身份验证错误**：确保设置了 `PADDLEOCR_ACCESS_TOKEN` 或提供了 `access_token`
2. **文件类型错误**：验证文件扩展名和可访问性
3. **API 连接问题**：检查端点 URL 和网络连接

### 调试模式

要进行详细调试，请检查原始 API 响应：

```python
docs = loader.load()
if docs:
    raw_response = docs[0].metadata.get("paddleocr_vl_raw_response")
    print("API 响应结构:", raw_response.keys())
```
