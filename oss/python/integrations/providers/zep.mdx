> 从聊天记录中回忆、理解并提取数据。为个性化 AI 体验提供动力。

>[Zep](https://www.getzep.com) 是 AI 助手应用的长期记忆服务。
> 借助 Zep，您可以为 AI 助手提供回忆过去对话的能力，无论这些对话多么久远，
> 同时还能减少幻觉、延迟和成本。

## Zep 工作原理

Zep 持久化并回忆聊天记录，并自动从这些聊天记录中生成摘要和其他产物。
它还会嵌入消息和摘要，使您能够从过去的对话中搜索 Zep 以获取相关上下文。
Zep 异步完成所有这些操作，确保这些操作不会影响用户的聊天体验。
数据持久化到数据库，允许您在增长需求时进行扩展。

Zep 还提供了一个简单易用的文档向量搜索抽象，称为文档集合。
这旨在补充 Zep 的核心记忆功能，但并非设计为通用向量数据库。

Zep 允许您更有目的地构建提示：
- 自动添加一些最近的消息，数量可根据您的应用进行定制；
- 上述消息之前的近期对话摘要；
- 和/或从整个会话中提取的上下文相关摘要或消息。
- 和/或来自 Zep 文档集合的相关业务数据。

## 什么是 Zep Cloud？
[Zep Cloud](https://www.getzep.com) 是一项托管服务，其核心是 Zep 开源版。
除了 Zep 开源版的记忆管理功能外，Zep Cloud 还提供：
- **事实提取**：自动从对话构建事实表，无需预先定义数据模式。
- **对话分类**：即时准确地对聊天对话进行分类。了解用户意图和情感，对用户进行细分等。基于语义上下文路由链，并触发事件。
- **结构化数据提取**：使用您定义的模式快速从聊天对话中提取业务数据。了解您的助手接下来应询问什么以完成其任务。

## 检索器

Zep 的记忆检索器是一个 LangChain 检索器，使您能够从 Zep 会话中检索消息并使用它们来构建提示。

该检索器支持搜索单个消息和对话摘要。后者对于为 LLM 提供丰富但简洁的相关过去对话上下文非常有用。

Zep 的记忆检索器支持相似性搜索和[最大边际相关性 (MMR) 重新排序](https://help.getzep.com/working-with-search#how-zeps-mmr-re-ranking-works)。MMR 搜索有助于确保检索到的消息多样化且彼此不太相似。

请参阅[使用示例](/oss/python/integrations/retrievers/zep_cloud_memorystore)。

```python
from langchain_community.retrievers import ZepCloudRetriever
```

## 向量存储

Zep 的[文档向量存储 API](https://help.getzep.com/document-collections) 使您能够使用向量相似性搜索存储和检索文档。Zep 不要求您理解距离函数、嵌入类型或索引最佳实践。您只需传入分块的文档，Zep 会处理其余部分。

Zep 支持相似性搜索和[最大边际相关性 (MMR) 重新排序](https://help.getzep.com/working-with-search#how-zeps-mmr-re-ranking-works)。
MMR 搜索有助于确保检索到的文档多样化且彼此不太相似。

```python
from langchain_community.vectorstores import ZepCloudVectorStore
```

请参阅[使用示例](/oss/python/integrations/vectorstores/zep_cloud)。
