`ZeroxPDFLoader` 是一个利用 [Zerox](https://github.com/getomni-ai/zerox) 库的文档加载器。Zerox 将 PDF 文档转换为图像，使用视觉能力语言模型进行处理，并生成结构化的 Markdown 表示。此加载器支持异步操作，并提供页面级文档提取。

### 集成详情

| 类 | 包 | 本地 | 可序列化 | JS 支持 |
| :--- | :--- | :---: | :---: |  :---: |
| [`ZeroxPDFLoader`](https://reference.langchain.com/python/langchain-community/document_loaders/pdf/ZeroxPDFLoader) | [`langchain_community`](https://reference.langchain.com/python/langchain-community) | ❌ | ❌ | ❌ |

### 加载器特性

| 来源 | 文档延迟加载 | 原生异步支持 |
| :---: | :---: | :---: |
| `ZeroxPDFLoader` | ✅ | ❌ |

## 设置

### 凭据

需要在环境变量中设置适当的凭据。加载器支持多种不同的模型和模型提供商。请参阅下面的 _使用_ 标题查看一些示例，或查看 [Zerox 文档](https://github.com/getomni-ai/zerox) 获取支持的模型的完整列表。

### 安装

要使用 `ZeroxPDFLoader`，您需要安装 `zerox` 包。同时确保已安装 `langchain-community`。

```bash
pip install zerox langchain-community
```

## 初始化

`ZeroxPDFLoader` 通过将每一页转换为图像并异步处理，实现使用视觉能力语言模型进行 PDF 文本提取。要使用此加载器，您需要指定一个模型并为 Zerox 配置任何必要的环境变量，例如 API 密钥。

如果您在 Jupyter Notebook 等环境中工作，可能需要使用 `nest_asyncio` 处理异步代码。您可以按如下方式设置：

```python
import nest_asyncio
nest_asyncio.apply()
```

```python
import os

# 使用 nest_asyncio（仅在 Jupyter Notebook 内部需要）
import nest_asyncio
from langchain_community.document_loaders.pdf import ZeroxPDFLoader

nest_asyncio.apply()

# 指定要处理的 PDF 的 URL 或文件路径
# 在本例中，我们使用来自网络的 PDF
file_path = "https://assets.ctfassets.net/f1df9zr7wr1a/soP1fjvG1Wu66HJhu3FBS/034d6ca48edb119ae77dec5ce01a8612/OpenAI_Sacra_Teardown.pdf"

# 为视觉模型设置必要的环境变量
os.environ["OPENAI_API_KEY"] = (
    "zK3BAhQUmbwZNoHoOcscBwQdwi3oc3hzwJmbgdZ"  ## 您的 API 密钥
)

# 使用所需模型初始化 ZeroxPDFLoader
loader = ZeroxPDFLoader(file_path=file_path, model="azure/gpt-4.1-mini")
```

## 延迟加载

加载器总是延迟获取结果。`.load()` 方法等同于 `.lazy_load()`

## API 参考

### `ZeroxPDFLoader`

此类加载器使用文件路径和模型类型初始化，并支持通过 `zerox_kwargs` 进行自定义配置以处理 Zerox 特定参数。

**参数**:

- `file_path` (Union[str, Path]): PDF 文件的路径。
- `model` (str): 用于处理的视觉能力模型，格式为 `<provider>/<model>`。
  一些有效值的示例：
  - `model = "gpt-4.1-mini" ## openai 模型`
  - `model = "azure/gpt-4.1-mini"`
  - `model = "gemini/gpt-4.1-mini"`
  - `model="claude-3-opus-20240229"`
  - `model = "vertex_ai/gemini-2.5-flash"`
  - 有关更多详细信息，请参阅 [Zerox 文档](https://github.com/getomni-ai/zerox)
  - 默认为 `"gpt-4.1-mini"`。
- `**zerox_kwargs` (dict): 其他 Zerox 特定参数，例如 API 密钥、端点等。
  - 请参阅 [Zerox 文档](https://github.com/getomni-ai/zerox)

**方法**:

- `lazy_load`: 生成 [`Document`](https://reference.langchain.com/python/langchain-core/documents/base/Document) 实例的迭代器，每个实例表示 PDF 的一页，并包含页码和来源等元数据。

请参阅 [完整的 API 文档](https://reference.langchain.com/python/langchain-community/document_loaders/pdf/ZeroxPDFLoader)

## 注意事项

- **模型兼容性**: Zerox 支持一系列视觉能力模型。请参阅 [Zerox 的 GitHub 文档](https://github.com/getomni-ai/zerox) 获取支持的模型列表和配置详情。
- **环境变量**: 确保设置所需的环境变量，例如 `API_KEY` 或端点详细信息，如 Zerox 文档中指定。
- **异步处理**: 如果在 Jupyter Notebook 中遇到与事件循环相关的错误，您可能需要如设置部分所示应用 `nest_asyncio`。

## 故障排除

- **RuntimeError: This event loop is already running**: 在 Jupyter 等环境中使用 `nest_asyncio.apply()` 防止异步循环冲突。
- **配置错误**: 验证 `zerox_kwargs` 是否与您选择的模型的预期参数匹配，以及是否设置了所有必要的环境变量。
